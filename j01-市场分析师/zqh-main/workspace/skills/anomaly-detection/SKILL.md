# Anomaly Detection

## 触发条件

- 市场份额单周下降超过 15% 或连续 2 周下降超过 10%。
- 客单价较 8 周移动均值偏离超过 2 个标准差。
- 转化率环比下降超过 20%，且流量未同步下降。
- 毛利率环比下降超过 5 个百分点。
- 渠道 ROI 连续 2 周低于目标线 20%。
- 高价值用户占比月环比下降超过 10%。

## 输入

- 指标时间序列。
- 阈值规则。
- 节假日、促销、价格调整、渠道活动、竞品动作等事件表。
- 数据口径变更记录。

## 步骤

1. 检查数据完整性和口径变化。
2. 计算环比、同比、移动均值、标准差或阈值偏离。
3. 判断告警等级。
4. 关联近期业务事件和外部信号。
5. 给出可能原因排序。
6. 输出排查路径和建议动作。

## 输出

- 告警等级。
- 异常指标、时间、幅度。
- 可能原因排序。
- 排查路径。
- 建议负责人和截止时间。

## 踩坑记录

- 不要直接把异常归因给市场变化，先排数据口径。
- 不要忽略促销、节假日和渠道政策。
- 不要只告警不提供排查动作。
