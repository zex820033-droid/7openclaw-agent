# 六大核心任务卡

所有任务使用 `datasets/bootcamp-market-data.csv`。如果数据不足，必须写明假设，不能硬编事实。

## Task 01：TAM / SAM / SOM 测算

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-001 |
| 业务目标 | 判断 B2B AI 数据分析工具在目标地区的 12 个月可拿市场 |
| 输入数据 | market 行数据 |
| 交付物 | TAM/SAM/SOM 表、公式、假设、风险 |
| 验收口径 | 能区分 TAM、SAM、SOM；SOM 不能直接按 TAM 百分比拍脑袋 |

最低输出：

```text
TAM =
SAM =
SOM =
关键公式 =
关键假设 =
最大不确定性 =
季度刷新动作 =
```

## Task 02：用户分层分析

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-002 |
| 业务目标 | 找出最值得经营的用户层 |
| 输入数据 | user 行数据 |

```text
用户层级 =
规模 =
收入贡献 =
毛利贡献 =
价值判断 =
建议动作 =
风险 =
```

## Task 03：细分赛道机会评估

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-003 |
| 业务目标 | 判断优先进入哪个细分赛道 |
| 输入数据 | segment 行数据 |
| 交付物 | 机会评分表、Top 3、验证动作、放弃条件 |
| 验收口径 | 必须解释权重；竞争密度要反向计分 |

默认权重：

| 维度 | 权重 |
| --- | ---: |
| 规模 | 20% |
| 增速 | 20% |
| 利润 | 20% |
| 竞争密度反向 | 15% |
| 能力匹配 | 20% |
| 数据可信度 | 5% |

## Task 04：渠道结构分析

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-004 |
| 业务目标 | 判断渠道预算是否需要调整 |
| 输入数据 | channel 行数据 |
| 交付物 | 渠道结构表、渠道分层、资源配置建议 |
| 验收口径 | 不能只看销售额；必须看毛利、CAC、回款周期和 ROI 风险 |

渠道标签：

- 增长渠道：线索和收入增长强，但需关注利润。
- 利润渠道：毛利和回款质量好。
- 风险渠道：收入高但毛利、回款或 CAC 存疑。
- 观察渠道：样本不足或波动大。

## Task 05：月度市场简报

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-005 |
| 业务目标 | 给经营层一页市场输入 |
| 输入数据 | 全部数据 |
| 交付物 | 一页摘要、机会清单、风险清单、下月动作 |
| 验收口径 | 不做新闻摘要；每条变化都回答“所以我们要做什么” |

必须包含：

- 本月 3-5 个关键变化。
- 用户行为变化。
- 渠道变化。
- 机会清单。
- 风险清单。
- 下月动作。

## Task 06：异常检测

| 字段 | 内容 |
| --- | --- |
| Task ID | BOOT-MKT-006 |
| 业务目标 | 对客单价、转化率、代理 ROI 的突变给出告警 |
| 输入数据 | metric 行数据和 channel 行数据 |
| 交付物 | 告警报告、原因排序、排查动作 |
| 验收口径 | 至少 3 个可能原因；必须先排数据，再排内部，最后排外部 |

最低输出：

```text
告警等级 =
异常指标 =
异常幅度 =
可能原因排序 =
排查路径 =
建议负责人 =
复盘时间 =
```
