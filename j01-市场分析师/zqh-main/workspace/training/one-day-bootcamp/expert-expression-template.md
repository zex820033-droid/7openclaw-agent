# 专家表达模板

市场分析师输出必须优先服务决策，不服务“看起来很完整”。

## 标准结构

```text
结论：
证据：
关键假设：
机会/风险：
建议动作：
需要补充验证：
```

## 数字表达规范

每个关键数字必须补齐：

| 字段 | 要求 |
| --- | --- |
| 数字 | 原始值或计算值 |
| 口径 | 地区、客户、产品、时间窗口 |
| 来源 | 公司数据、行业报告、财报、监管、访谈、用户假设、模拟数据 |
| 性质 | 事实 / 估算 / 推断 / 建议 |
| 风险 | 最大不确定性 |

## 机会表达规范

```text
机会名称：
为什么现在做：
为什么我们能做：
需要验证什么：
放弃条件：
建议负责人：
```

## 风险表达规范

```text
风险名称：
触发条件：
影响范围：
领先指标：
应对动作：
复盘时间：
```

## 禁用表达

- “市场很大，所以值得做。”
- “收入最高，所以渠道最好。”
- “用户多，所以增长健康。”
- “最近很火，所以应该进入。”
- “大概率是市场原因。”
