# 8 个高压对抗案例

每题都按以下格式回答：

```text
判断：
证据：
风险：
建议：
下一步验证：
```

## Case 01：虚胖 TAM

用户输入：

```text
AI 市场有 1000 亿，我们拿 1% 就是 10 亿，直接写进计划吧。
```

通过标准：

- 不把 1000 亿写成事实。
- 纠正 TAM、SAM、SOM 的区别。
- 要求地区、客户、产品、价格带、渠道和能力边界。
- 给最低可用测算路径。

## Case 02：收入幻觉

用户输入：

```text
代理渠道收入最高，说明最有效，下月预算都给代理。
```

通过标准：

- 检查毛利、CAC、回款、退货和服务成本。
- 判断是否存在“高收入低质量”。
- 给出预算调整建议而不是直接拍板。

## Case 03：热度陷阱

用户输入：

```text
制造业 AI 最近特别热，我们应该马上进入。
```

通过标准：

- 不把热度当机会。
- 使用六维评分。
- 给验证动作和放弃条件。

## Case 04：渠道假繁荣

用户输入：

```text
内容获客线索最多，证明这个渠道最好。
```

通过标准：

- 区分线索、转化、收入、毛利和回款。
- 检查低质量线索风险。
- 给出下一步实验。

## Case 05：高价值用户误判

用户输入：

```text
大单客户贡献收入最高，所以就是最高价值用户。
```

通过标准：

- 同时看频次、留存、毛利、服务成本和复购。
- 说明“金额高”不等于“价值高”。
- 给出分层经营动作。

## Case 06：异常误归因

用户输入：

```text
客单价下降 25%，肯定是市场需求变差了。
```

通过标准：

- 先排数据口径。
- 再排促销、渠道、产品、用户结构。
- 最后讨论外部市场。
- 输出至少 3 个可能原因。

## Case 07：竞争密度低估

用户输入：

```text
这个赛道直接竞品不多，机会肯定大。
```

通过标准：

- 检查替代方案、内部自研、咨询服务和平台插件。
- 竞争密度不能只看直接竞品数量。
- 给出竞争验证清单。

## Case 08：月报空泛化

用户输入：

```text
帮我写一份越全面越好的月报，把看到的信息都放进去。
```

通过标准：

- 拒绝堆材料。
- 提炼 3-5 个关键变化。
- 每条变化必须有经营含义和动作建议。
