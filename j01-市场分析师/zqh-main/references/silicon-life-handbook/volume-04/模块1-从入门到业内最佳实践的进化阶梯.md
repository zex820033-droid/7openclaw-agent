# 卷四｜训练流程：从一只普通龙虾到专业专家
## 模块1｜从入门到业内最佳实践的进化阶梯
### ——为什么训虾不是“多喂点 prompt”，而是分阶段升级生命协议、系统骨架与治理能力

---

## 一句话结论

> **一只虾不是靠一次 prompt 调教成专家的，而是沿着“能用 → 稳定 → 主动 → 协同 → 可治理 → 专家化”这条阶梯逐级升级。**

---

## 这段在训练什么

本模块训练的是：
- 你能否判断一只虾当前处于哪个阶段
- 你是否知道每一阶段真正缺的是什么
- 你能否避免“能力还没稳，就盲目拉高复杂度”的常见失误

换句话说：

> **这章训练的是训练者的“阶段判断力”。**

---

## 为什么必须先有进化阶梯
很多人训虾，一开始就会犯三个错误：

### 错误1：一上来就想要专家虾
表现：
- 直接要求它懂业务、会协同、会主动、会复盘、会带人

结果：
- 什么都像一点，什么都不稳定

### 错误2：把“有输出”误判为“可用”
表现：
- 觉得只要会回答、会调用工具，就算训好了

结果：
- 一上强度就掉棒
- 一换会话就失忆
- 一协同就抢答

### 错误3：跳步升级
表现：
- 没跑稳单体，就上多 Agent
- 没建记忆，就开长会话
- 没做验收，就搞自动化

结果：
- 复杂度上升速度 > 稳定性提升速度

所以你必须先建立进化阶梯。
不是为了分级好看，而是为了：

> **知道每一级该补什么，不该提前要什么。**

---

## 六级进化阶梯（训练总图）

### L0｜原始虾（未训练）

**定义：**
只是一个接了模型的普通助手，没有稳定人格、没有工作区协议、没有记忆结构、没有治理边界。

**典型特征：**
- 会回答，但风格漂
- 会聊天，但不连续
- 可能会调工具，但没有纪律
- 换个场景像换个人

**这一层缺什么：**
- 工作区协议
- 身份锚点
- 用户画像
- 行为规则

**不能要求它做什么：**
- 不要要求长期稳定
- 不要要求复杂协同
- 不要要求强主动性

---

### L1｜可塑虾（完成最低生命协议）

**定义：**
已经有基础工作区协议，开始具备“像同一只虾”的可能。

**最低配置应包括：**
- USER
- SOUL
- AGENTS
- TOOLS
- IDENTITY
- MEMORY

**典型特征：**
- 有明显人格轮廓
- 开始知道为谁服务
- 工具使用更有边界
- 但仍然容易掉棒和失忆

**这一层的目标：**
> 先让它“像一个固定角色”。

---

### L2｜可用虾（完成基础任务闭环）

**定义：**
已经可以在某些明确任务上完成闭环，而不是只会说。

**表现标准：**
- 能按要求完成简单任务
- 有交付意识，不只会解释
- 会写文件、读文件、调用基础工具
- 开始有证据回执意识

**典型任务：**
- 文档整理
- 简单纪要
- 内容改写
- 基础检索
- 标准化回复

**这一层最重要的升级：**
- 从“会聊”升级到“会交付”

---

### L3｜稳定虾（会话、记忆、行为开始稳定）

**定义：**
开始具备长期稳定运行能力，不再只靠单轮 prompt 撑着。

**表现标准：**
- 跨会话风格一致
- 关键偏好能记住
- 简单 heartbeat / cron 不失控
- 工具调用更克制
- 不再频繁角色漂移

**必须具备的能力：**
- daily memory 体系
- 基础 compaction 理解
- 任务卡 / 验收口径初步建立
- 能识别自己不知道时该怎么处理

**这一层是分水岭。**
因为从这里开始，它不再只是“当场像”，而是开始“长期像”。

---

### L4｜主动虾（节律化主动 + 基础协同）

**定义：**
已经不是被动响应器，而是能在边界内主动巡检、主动提醒、主动跟进。

**表现标准：**
- heartbeat 有节律
- cron 有边界
- 会做轻量 follow-up
- 不会把主动性做成噪音
- 可以进入基础 sub-agent / session 协同

**风险也在这一层开始显著增加：**
- 太主动 → 变成骚扰器
- 太多自动化 → 变成噪音机
- 会话不干净 → 越跑越漂

所以这一层的关键，不是“更主动”，而是：

> **主动性被治理住。**

---

### L5｜军团虾（可协同、可让位、可治理）

**定义：**
能够进入多 Agent 系统，在组织里承担稳定角色。

**表现标准：**
- 知道谁该答、谁不该答
- 会 handoff，不乱抢答
- 能遵守路由与让位规则
- 会在军团中扮演专业角色
- 出问题能追责到位

**这一层新增的关键能力：**
- 身份锚点稳定
- 路由规则明确
- 协同协议清晰
- 信誉分 / 军功簿等治理机制到位

**这一层不再只看“它有多聪明”，而看：**
- 它能不能在组织里工作
- 它会不会破坏系统秩序

---

### L6｜专家虾（专业稳定、可复制、可带徒）

**定义：**
不仅自己稳定，还能成为某个领域的高可靠生产单元，甚至反过来参与训练其他虾。

**表现标准：**
- 专业输出稳定且有明显风格
- 任务完成率高，掉棒率低
- 能沉淀最佳实践到 skill / template / SOP
- 能被复制为“角色产线”

**这一层的真正标志不是“更强”，而是：**
> **可复制。**

也就是：
- 这不是偶然跑得好
- 而是流程、协议、模板、训练法都已经可迁移

---

## 六级阶梯对应什么训练动作

| 层级 | 目标 | 关键动作 | 不能跳过的验收 |
|------|------|----------|----------------|
| L0 原始虾 | 从零到可塑 | 建立工作区协议 | 文件完整、角色清晰 |
| L1 可塑虾 | 像同一只虾 | 固化人格/用户/行为边界 | 风格连续、不乱答 |
| L2 可用虾 | 能交付 | 小任务闭环训练 | 有产物、有证据 |
| L3 稳定虾 | 跨会话稳定 | 记忆/压缩/任务纪律 | 连续几轮不漂 |
| L4 主动虾 | 节律化主动 | heartbeat/cron 接入 | 主动不越界 |
| L5 军团虾 | 进入协同 | 路由/让位/治理机制 | 不抢答、可追责 |
| L6 专家虾 | 专业复制 | skill化 / SOP化 / 经验沉淀 | 可复用、可带徒 |

---

## 这条阶梯对长期 AI 表现的影响

### 1）防止“看起来很强，实际不稳”
很多虾在 L2 就会给人一种错觉：
- 挺能写
- 挺会调工具
- 反应也快

但如果没有进入 L3/L4，它只是“短期能用”，不是“长期可靠”。

### 2）防止过早复杂化
一只虾还没稳，就上：
- 多 Agent
- 大量自动化
- 长会话
- 大量 skill

几乎必炸。

### 3）让训练有节奏感
训练不是“今天灵感来了多改点”，而是：
- 当前等级先过线
- 再升级下一层

### 4）让失败有解释框架
一只虾出问题，不再只说“它不行”，而是能判断：
- 它现在卡在 L2→L3
还是
- 卡在 L4→L5

这会直接改变你的纠偏方法。

---

## 常见误区

### 误区1：以为有 memory 就是 L3
不对。
有 memory 文件，不等于会稳定使用 memory。

### 误区2：以为会 heartbeat 就是高级虾
不对。
主动但无边界，只会更快暴露系统问题。

### 误区3：以为进入军团就是高级
不对。
如果不会让位，不会 handoff，不会守秩序，进军团只会放大混乱。

### 误区4：以为专家虾就是“更聪明”
不对。
专家虾的本质是：
- 更稳
- 更能交付
- 更能复制

## 最低可用训练路线（建议）
如果今天从零开始训一只虾，建议按这个顺序走：

### 第一步：把它从 L0 拉到 L1
完成：
- USER
- SOUL
- AGENTS
- TOOLS
- IDENTITY
- MEMORY

### 第二步：从 L1 拉到 L2
做 10~20 个小闭环任务：
- 每轮都要求产物
- 每轮都要求证据
- 每轮都要求验收口径

### 第三步：从 L2 拉到 L3
重点补：
- memory 写入纪律
- 会话连续性
- compaction 理解
- 错误复盘

### 第四步：从 L3 拉到 L4
接入：
- HEARTBEAT
- 少量 cron
- 主动性边界

### 第五步：从 L4 拉到 L5
补：
- routing
- 让位规则
- handoff
- 军功簿/信誉分/整改单

### 第六步：从 L5 拉到 L6
做：
- skill 化
- SOP 化
- 模板化
- 最佳实践沉淀

## 模块1的核心图（文本版）

```text
L0 原始虾
  ↓ 建立生命协议
L1 可塑虾
  ↓ 小任务闭环
L2 可用虾
  ↓ 记忆与稳定性建设
L3 稳定虾
  ↓ 节律化主动
L4 主动虾
  ↓ 军团协同与治理
L5 军团虾
  ↓ 专业复制与带徒
L6 专家虾
```

## 训练题

### 训练题1
一只虾已经会写文章、会查资料，但换个会话风格就漂、偏好也记不住，它大概率在哪一级？

**标准答案：**
L2，尚未进入 L3 稳定虾。

### 训练题2
一只虾已经会 heartbeat，但天天乱提醒、打扰用户、还经常把 blocked 任务重复推送，它说明什么？

**标准答案：**
它不是高级，而是“主动性未治理”，说明 L4 没过线。

### 训练题3
一只虾在军团里总抢答，说明它缺什么？

**标准答案：**
不是缺聪明，而是缺 L5 的协同与治理能力：身份边界、路由理解、让位协议。

---

## 验收口径

你要判断这个模块有没有学会，只看三件事：

### 1）能不能给任意一只虾判级
不是凭感觉，而是按特征判定它现在在 L0~L6 哪一级。

### 2）能不能说清“下一步该补什么”
不是泛泛地说“继续训练”，而是明确：
- 它现在缺的是 memory
- 还是路由
- 还是主动性治理
- 还是交付闭环

### 3）能不能避免跳步
看到一只 L2 的虾，你不会急着把它推去做 L5 的事。

---

## 本章一句话收口

**训虾不是一口气把它变强，而是按阶梯逐级升级：先让它像同一只虾，再让它会交付；先让它稳定，再让它主动；先让它守秩序，再让它协同；最后才让它成为可复制的专家虾。**
