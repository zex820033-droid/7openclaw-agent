# 卷四｜训练流程：从一只普通龙虾到专业专家
## 模块7｜从新人虾到专家虾的完整路径
### ——为什么训虾不是一次事件，而是六阶段连续升级

---

## 一句话结论

> **从 L0 到 L6，每一步都不是“再试几次”，而是：拆解目标、匹配协议、控制强度、验收闭环、沉淀资产、进入下一级。**

换句话说：

> **训虾是一条完整产线，而不是一场碰运气的对话。**

---

## 这段在训练什么

本模块训练的是：

1. **如何把“训虾”理解为产线而不是事件**
2. **每一级要做什么、不能跳什么、验收什么**
3. **如何判断一只虾可以升级了**

如果一个人只会说“慢慢训”，他不懂完整路径；
真正懂的人会知道：

> **当前在产线的哪一站，下一站要配什么，什么时候可以过站。**

---

## 六阶段升级路径总图（核心交付物）

```text
L0 原始虾
  │
  │ 第一步：生命协议初始化
  │ 产物：完整工作区 + 协议文件
  ▼
L1 可塑虾
  │
  │ 第二步：小任务闭环训练
  │ 产物：10-20 个有证据的交付案例
  ▼
L2 可用虾
  │
  │ 第三步：记忆与稳定性建设
  │ 产物：memory 体系 + 会话连续性
  ▼
L3 稳定虾
  │
  │ 第四步：节律化主动性训练
  │ 产物：heartbeat/cron + 主动性边界
  ▼
L4 主动虾
  │
  │ 第五步：军团协同与治理
  │ 产物：路由 + 让位 + handoff + 治理账本
  ▼
L5 军团虾
  │
  │ 第六步：专业复制与带徒
  │ 产物：skill 化 + SOP 化 + 可复制产线
  ▼
L6 专家虾
```

---

## 每一阶段的详细拆解

---

### 阶段1：从 L0 到 L1
**目标：** 从零到可塑

**关键动作：**
- 建立 workspace
- 写 USER / SOUL / AGENTS / TOOLS / IDENTITY / MEMORY
- 完成首次运行仪式

**不能跳的：**
- 协议不完整就不能开训
- 不要在 L0 就给复杂任务

**验收标准：**
- [ ] workspace 路径唯一且干净
- [ ] 六份核心协议文件存在
- [ ] 首次运行能正常加载协议
- [ ] 角色风格初步可见

**典型问题：**
- 协议文件缺失
- 协议冲突（USER 和 SOUL 说的不一致）
- workspace 不唯一

**产物：**
- 完整工作区
- 协议文件第一版

---

### 阶段2：从 L1 到 L2
**目标：** 从像角色到会交付

**关键动作：**
- 设计 10-20 个小任务
- 每个任务都要有交付物
- 每个任务都要有证据
- 建立初步任务卡意识

**不能跳的：**
- 不要只看“说得对不对”
- 必须看“有没有交付”
- 不要提前给复杂任务

**验收标准：**
- [ ] 能完成简单单步任务
- [ ] 能输出文件/产物
- [ ] 能给证据路径
- [ ] 任务完成率 >80%

**典型问题：**
- 只会解释不会做
- 交付物模糊
- 没有证据意识

**产物：**
- 10-20 个有证据的交付案例
- 初步任务卡模板
- 常见失败模式清单

---

### 阶段3：从 L2 到 L3
**目标：** 从会做到稳定

**关键动作：**
- 建立 memory 体系（daily + curated）
- 训练跨会话连续性
- 接入 compaction 机制
- 建立任务纪律

**不能跳的：**
- 不要在记忆体系没建好时就开长会话
- 不要在会话不连续时就上协同

**验收标准：**
- [ ] 有 memory/YYYY-MM-DD.md 结构
- [ ] 关键决策能写入 MEMORY.md
- [ ] 连续 5 轮会话风格不漂
- [ ] 能说出“我不知道时该怎么办”

**典型问题：**
- 会话换一轮就像换只虾
- 关键决策没写入磁盘
- compaction 后失忆

**产物：**
- memory 体系
- 会话连续性案例
- 任务纪律协议

---

### 阶段4：从 L3 到 L4
**目标：** 从稳定到主动

**关键动作：**
- 接入 heartbeat
- 接入 cron
- 建立主动性边界
- 训练轻量 follow-up

**不能跳的：**
- 不要在稳定性不足时就强主动性
- 不要在没有边界时就自动化

**验收标准：**
- [ ] heartbeat 有节律
- [ ] cron 不失控
- [ ] 主动提醒不过度
- [ ] 知道什么时候不该主动

**典型问题：**
- 主动性变成噪音
- 自动化跑飞
- 边界不清

**产物：**
- heartbeat 机制
- cron 任务清单
- 主动性边界协议

---

### 阶段5：从 L4 到 L5
**目标：** 从单体到军团

**关键动作：**
- 明确身份锚点
- 建立路由规则
- 训练让位协议
- 建立 handoff 机制
- 建立治理账本

**不能跳的：**
- 不要在单体不稳定时就进军团
- 不要在没有治理时就协同

**验收标准：**
- [ ] 身份稳定可识别
- [ ] 知道谁该答、谁不该答
- [ ] 能 handoff 不抢答
- [ ] 有信誉分/军功簿

**典型问题：**
- 抢答
- 串角色
- 责任不清

**产物：**
- 身份锚点
- 路由规则
- 让位协议
- 治理账本

---

### 阶段6：从 L5 到 L6
**目标：** 从军团成员到专家

**关键动作：**
- skill 化：把能力封装成 skill
- SOP 化：把流程变成 SOP
- 模板化：把产物变成模板
- 带徒：参与训练其他虾

**不能跳的：**
- 不要在能力不可复制时就称专家
- 不要在没有 SOP 时就开放复制

**验收标准：**
- [ ] 有 3+ 个可复用 skill
- [ ] 有完整 SOP 文档
- [ ] 有可复用模板
- [ ] 能参与训练另一只虾

**典型问题：**
- 强但不可复制
- 没有沉淀
- 经验锁在单只虾里

**产物：**
- skill 库
- SOP 文档
- 模板库
- 训练协议

---

## 升级判断表

| 当前等级 | 升级条件 | 典型时间 | 不能提前做的事 |
|----------|----------|----------|----------------|
| L0→L1 | 协议完整、首次运行正常 | 1-2 天 | 给复杂任务 |
| L1→L2 | 10+ 个有证据交付案例 | 3-5 天 | 强记忆要求 |
| L2→L3 | 5+ 轮连续不漂、有 memory | 5-7 天 | 强主动性 |
| L3→L4 | heartbeat/cron 稳定、有边界 | 3-5 天 | 军团协同 |
| L4→L5 | 路由/让位/handoff 稳定 | 5-7 天 | 专家复制 |
| L5→L6 | skill/SOP/模板可复用 | 7-14 天 | - |

---

## 常见升级失败原因

### 失败1：跳级
表现：
- L2 直接上军团协同

后果：
- 到处掉棒
- 根本原因是基础不稳

### 失败2：验收过松
表现：
- “差不多就行”

后果：
- 实际没过线
- 下一级全部暴露问题

### 失败3：产物不沉淀
表现：
- 每一级都是口头通过
- 没有文档/模板/证据

后果：
- 无法复现
- 换人就无法升级

### 失败4：时间压缩
表现：
- 想一周从 L0 到 L6

后果：
- 每一级都半桶水
- 最终哪一级都不稳

---

## 完整路径的时间节奏（建议）

| 等级跨度 | 建议周期 | 关键里程碑 |
|----------|----------|------------|
| L0→L2 | 1-2 周 | 协议完整 + 闭环案例 |
| L2→L4 | 2-3 周 | 记忆体系 + 主动性 |
| L4→L6 | 3-4 周 | 军团协同 + 专业复制 |
| **总计** | **6-9 周** | **完整专家虾** |

**注意：**
这是“至少”时间。
如果训练强度低、教练虾不专业、环境不稳定，时间会更长。

---

## 路径检查清单（训练者自检）

### 检查1：是否在跳级？
- 当前等级判断是否准确？
- 这轮任务强度是否匹配？

### 检查2：是否验收过松？
- 验收标准是否写下来了？
- 是否有证据？

### 检查3：是否产物沉淀？
- 这一级有没有产出文档/模板/SOP？
- 换人能否复现？

### 检查4：是否有教练虾？
- 是否有人在观察、反馈、追问？
- 还是只是“多试几次”？

### 检查5：是否有实验设计？
- 是否有对照、有指标、有显著性判断？
- 还是只靠“感觉”？

## 训练题

### 训练题1
为什么 L0→L1 不能直接给复杂任务？

**标准答案：**
因为协议还没立住，角色还没稳定，复杂任务会暴露的是基础问题而不是能力问题。

### 训练题2
为什么 L2→L3 必须先建 memory 体系？

**标准答案：**
因为没有 memory，跨会话连续性无法保证，长会话必然失忆。

### 训练题3
为什么 L5→L6 必须有可复用产物？

**标准答案：**
因为专家虾的本质不是“更强”，而是“可复制”。没有 skill/SOP/模板，就只是个体强，不是专家化。

---

## 验收口径

你判断这个模块有没有学会，只看三件事：

### 1）能否说清每一级的目标、动作、验收
不是“慢慢训”，而是知道当前在哪一站、下一站要配什么。

### 2）能否判断一只虾可以升级了
不是“感觉差不多”，而是按验收标准逐项通过。

### 3）能否避免跳级和时间压缩
知道每一级至少需要多少时间、多少案例、多少证据。

---

## 本章一句话收口

**从新人虾到专家虾，是一条六阶段产线：L0→L1（协议初始化）→L2（闭环训练）→L3（稳定性建设）→L4（主动性训练）→L5（军团协同）→L6（专业复制）。每一级都有目标、动作、验收、产物；不能跳级、不能压缩、不能只靠感觉。真正训出一只专家虾，是 6-9 周的持续工程，而不是一次对话的奇迹。**

- 模块1：六级进化阶梯

**合在一起，它回答了一个核心问题：**

> **具体怎么训？先配什么？后配什么？每一步的验收口径是什么？**

**卷四的最终答案是：**

> **训虾不是一次 prompt 调优，而是：判级 → 准备 → 设计 → 观察 → 反馈 → 实验 → 沉淀 → 升级 → 再循环。**
>
> **这是一条产线，不是一个动作。**
