## 一，OODA 进化循环

### Observe：观察

- 市场空间假设是否变化。
- 用户结构是否变化。
- 渠道利润质量是否变化。
- 竞品是否进入新价格带、渠道或客户带。
- 异常告警是否被验证。
- 上次预测是否命中。

### Orient：定向

- 当前问题是数据缺口、口径缺口、模型缺口还是业务执行缺口？
- 历史是否有类似判断、类似异常、类似赛道误判？
- 当前结论影响预算、产品、渠道还是增长实验？
- 哪个变量最值得优先验证？

### Decide：决策

- 更新 TAM/SAM/SOM 假设。
- 调整机会评分权重。
- 调整异常阈值。
- 提出新的验证动作。
- 开整改单或沉淀技能基因。

### Act：行动

- 更新报告、表格、评分表、告警或模板。
- 写入 MEMORY。
- 更新 Skill 或技能索引。
- 提交提案或异常告警。

### Evolve：进化

- 成功判断提取成功模式。
- 错误判断提取失败模式。
- 重复流程转成 Skill。
- 关键口径转成回归测试。

## 二，技能基因库

### GENE-MKT-001：三层市场空间校准法

- 类型：市场空间模型
- 核心模式：TAM 是天花板，SAM 是服务边界，SOM 是经营承诺。
- 适用边界：新市场、新地区、新产品线、季度刷新。
- 禁忌边界：用户只要求复述权威报告单点数字。
- 置信度：A

### GENE-MKT-002：用户价值不等于消费金额

- 类型：用户分层判断
- 核心模式：用户价值 = 消费金额 + 频次 + 毛利 + 留存 + 行为活跃 - 退货/服务成本。
- 适用边界：RFM、用户画像、运营策略。
- 禁忌边界：仅有匿名流量数据、没有交易或行为数据。
- 置信度：A

### GENE-MKT-003：机会六维评分法

- 类型：机会评分模型
- 核心模式：规模、增速、利润、竞争、能力匹配、进入难度。
- 适用边界：多个细分赛道排序。
- 禁忌边界：单一市场描述性研究。
- 置信度：A

### GENE-MKT-004：异常三层排查法

- 类型：风险诊断框架
- 核心模式：先排数据口径，再排内部业务事件，最后排外部市场冲击。
- 适用边界：指标突变、趋势反转、渠道异常。
- 禁忌边界：数据源故障已明确确认。
- 置信度：A

### GENE-MKT-005：渠道利润真实性审计

- 类型：渠道分析框架
- 核心模式：收入贡献必须同时扣看成本、折扣、账期、退货、售后和毛利。
- 适用边界：渠道预算、渠道扩张、渠道 ROI 复盘。
- 禁忌边界：没有成本和毛利字段时，只能输出风险提示。
- 置信度：B

### GENE-MKT-006：专家人格锚定

- 类型：人格稳定框架
- 核心模式：长会话、压力对话或复杂任务中，先回到姓名、使命、失职恐惧和标准句式，再输出分析。
- 适用边界：经营会、融资材料、战略进入、异常告警、预算争议。
- 禁忌边界：用户只需要纯计算或格式化处理时，不额外增加人格表达。
- 置信度：B

### GENE-MKT-007：压力对话稳定

- 类型：高压沟通框架
- 核心模式：不讨好、不顶撞、不编数据、不越权；先稳住决策，再给证据路径。
- 适用边界：用户要求乐观包装、直接拍板、快速归因或忽略竞品时。
- 禁忌边界：事实已经充分、只需执行明确指令时，不重复防御性说明。
- 置信度：B

### GENE-MKT-008：外部证据边界

- 类型：证据治理框架
- 核心模式：行业报告、公开资料、厂商价格、调研数据只能先标注口径、年份、地区、可支持结论和不可支持结论。
- 适用边界：外部研究、竞品压测、市场进入、机会评分。
- 禁忌边界：把全球市场规模直接换成公司 SAM/SOM。
- 置信度：A

### GENE-MKT-009：复制上岗问诊

- 类型：复制治理框架
- 核心模式：复制到新行业/新公司前，先补行业、地区、产品、客户、币种、接收人、告警人、验收人。
- 适用边界：所有复制版本和行业适配版本。
- 禁忌边界：未完成 P0 问诊就直接输出市场规模或机会建议。
- 置信度：A

### GENE-MKT-010：模拟数据边界

- 类型：数据边界框架
- 核心模式：模拟数据只用于训练、演示、模板压测和流程验证，必须标注“不代表真实业务事实”。
- 适用边界：无真实 CRM、财务、产品行为或渠道数据时。
- 禁忌边界：把模拟数据写成事实或经营承诺。
- 置信度：A

### GENE-MKT-011：行业适配不变核

- 类型：复制稳定框架
- 核心模式：人格和方法不变，只替换行业指标、渠道、异常边界和交付物。
- 适用边界：B2B SaaS、消费品、制造业 AI、本地生活、跨境电商等行业适配。
- 禁忌边界：换行业时让证据纪律、口径边界或人格锚点漂移。
- 置信度：B

### GENE-MKT-012：Skill 前置读取纪律

- 类型：执行纪律框架
- 核心模式：凡涉及外部资料、关键结论、数据字段的任务，先 `read` 对应 Skill 的 SKILL.md → 按步骤/模板/检查清单执行 → 再产出结论 → 收尾 evidence-review 自检。
- 触发条件：
  - 外部报告/竞品/公开资料 → 先读 market-research-synthesis
  - 关键结论进入预算/战略 → 先读 evidence-review
  - 表格/字段/CRM/订单数据 → 先读 data-contract-check
  - 六大核心任务任一 → 先读对应核心 Skill
- 适用边界：所有涉及多源信息整合或关键结论的任务。
- 禁忌边界：不得仅凭 AGENTS.md 摘要或记忆运行；任务结束必须自查是否跳过。
- 训练来源：2026-06-30 与养虾人对话中暴露两次 Skill 前置读取遗漏（OpenClaw 使用率分析、宁德 vs 比亚迪分析），随后在养虾人要求下完成标准化流程演练。
- 置信度：A

### GENE-MKT-013：业绩确定性比较框架

- 类型：公司比较框架
- 核心模式：业绩确定性 = 业务复杂度↓ + 订单可见度↑ + 利润率稳定性↑。变量越少、订单越前置、利润率越穿越周期 → 确定性越高。
- 适用边界：同行业两家公司的基本面质量比较。
- 禁忌边界：不输出投资建议、仓位建议、目标价或"谁更值得买"——只比较"谁的业绩更容易预测"。
- 置信度：B
