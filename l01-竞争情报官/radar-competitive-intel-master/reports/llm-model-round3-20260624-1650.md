## T6 · 产品拆解：OpenAI vs Anthropic — 模型+Agent策略8维对比

| 维度 | OpenAI | Anthropic | 对OpenClaw的启示 |
|:----|:------|:---------|:----------------|
| **旗舰模型** | GPT-5.5 (推理+代码) | Claude Opus 4.8 (Super-Agent 100%) | Opus 4.8 S0.5(唯一Super-Agent 100%) |
| **Agent产品** | Codex CLI (已退役→Maxxing) + Operator | Claude Code + Claude Tag + Claude Cowork | Anthropic Agent矩阵更完整 |
| **集成入口** | API/SDK + Copilot (Azure) | Claude Code CLI + Slack(Tag) + API | Claude Tag的Slack入口是差异化 |
| **企业能力** | Azure + Microsoft生态 | Claude Enterprise + Team计划 | Anthropic企业产品刚起步但Tag迭代快 |
| **开源策略** | GPT闭源 · Codex不开源 | 闭源(但研究论文多) | 两者都闭源 |
| **定价策略** | $20 Pro + API按tokens | $20 Pro + $100 Max + API按tokens | Opus 4.8 Fast降价3x值得关注 |
| **安全/对齐** | 相对封闭的安全研究 | 开放研究(Frontier Red Team·对齐团队) | Anthropic的安全透明度更高 |
| **对编队的影响** | Codex→企业Agent运行时 | Claude Tag→Slack Agent协作标准 | **两者都在抢占Agent运行时标准。OpenClaw不竞争运行时，竞争**编队编排 **差异化**

### 核心启示

1. **Anthropic是本周最大赢家** — Claude Tag + Claude Code Expertise研究 + Claude Opus 4.8 Super-Agent —— 三天内从模型(Opus)→工具(Code)→协作(Tag)全链路展示。

2. **"领域专业>编码能力"是赛道拐点论据** — Claude Code Expertise研究直接挑战"只有程序员才用AI编程"假设。对OpenClaw是利好——如果任何领域专家都能用Agent，编队编排的市场规模扩大。

3. **模型层竞争的真正赢家是编排层** — Claude Tag + Codex Maxxing都在定义Agent运行时标准，但都在单一模型生态内。跨模型/跨Agent类型的**编队编排**仍是无主之地。
