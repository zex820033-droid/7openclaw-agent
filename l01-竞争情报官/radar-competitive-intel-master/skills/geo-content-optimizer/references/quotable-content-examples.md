# Quotable Content Examples

Pattern cards for turning generic prose into AI-citable content. Use current, verified sources; replace all placeholders before publishing.

## Transformation Patterns

| Content Type | Weak Shape | Quotable Shape | Required Checks |
|--------------|------------|----------------|-----------------|
| Definition | Vague claim about importance | "**[Term]** is [category] that [specific function/scope]. [Dated source] reports [metric], showing [why it matters]." | Precise term, category, source + date |
| Statistic | "Many companies use it" | "[Metric] reached [value] in [year], according to [source]. This means [business implication]." | Number, source, year, context |
| Process | Loose advice | "To [goal]: 1. [action] 2. [action] 3. [action]. Typical effort: [time/range]." | Numbered steps, concrete inputs, time estimate |
| Comparison | "A is flexible, B is easy" | Table comparing [criteria], then "Choose A if..." / "Choose B if..." | Same criteria across options |
| Expert | Unsupported opinion | "[Expert quote]," says [Name], [credential]. [Research/source] supports this with [metric]. | Credential, quote, supporting evidence |
| Q&A | "It depends" | Direct 40-60 word answer first, then 3-4 factors or caveats. | Complete answer, no hidden context |
| List | Unordered keywords | Ranked list with impact, action, and one metric or threshold per item. | Priority order, specific guidance |
| Trend | Generic prediction | "[Trend] is changing [field]. [Source/year] forecasts [metric/timeframe]. Prepare by [3 actions]." | Timeframe, forecast source, action |
| Technical | "X measures UX" | Table: metric, what it measures, good threshold, remediation. | Official thresholds, current metric names |
| Case study | "Traffic improved" | Baseline -> actions -> result table over [timeframe], with one transferable insight. | Baseline, timeframe, reproducible method |

## Copy-Start Blocks

### Definition

```markdown
**[Topic]** is [category] that [function] for [audience/use case]. According to [Source] ([publication date]), [specific statistic]. This matters because [plain-language implication].
```

### Statistic

```markdown
[Metric] changed from [baseline] to [current value] over [timeframe], according to [Source] ([date]). The change indicates [interpretation] for [audience/use case].
```

### Process

```markdown
To [achieve outcome]:
1. [Action with input]
2. [Action with tool/evidence]
3. [Action with decision rule]
4. [Action with validation]

Expected effort: [time/range]. Success indicator: [metric or threshold].
```

### Comparison

```markdown
| Factor | [Option A] | [Option B] |
|--------|------------|------------|
| Best for | [use case] | [use case] |
| Setup effort | [range] | [range] |
| Cost model | [model] | [model] |
| Main limitation | [limitation] | [limitation] |

Choose [A] if [condition]. Choose [B] if [condition].
```

### Case Study

```markdown
**Case Study: [Outcome] in [Timeframe]**
Baseline: [starting metric].
Actions: [action 1], [action 2], [action 3].
Results: [metric] -> [metric] ([delta]); [secondary metric] -> [secondary metric].
Transferable insight: [what others can reuse].
```

## Quick Quotability Test

Content is highly quotable when it:

1. Stands alone without surrounding context.
2. Includes specific numbers or thresholds where relevant.
3. Names a source and publication date for factual claims.
4. Uses precise, unambiguous wording.
5. Would pass expert review.
6. Is structured for scanning.
7. Is current or explicitly labeled historical.
8. Can be verified.
9. Is specific to a use case.
10. Fully answers one question.
