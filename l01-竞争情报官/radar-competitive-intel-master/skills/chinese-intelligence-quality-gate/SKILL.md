# 中文情报质量门禁 (Chinese Intelligence Quality Gate)

## 一、中文金融数据单位换算校验

### 换算表

| 中文原文 | 正确英文 | 常见错误 |
|---------|---------|---------|
| 10亿美元 | $1B | ❌ $10B |
| 100亿美元 | $10B | ❌ $100B |
| 1000亿美元 | $100B | ❌ $1T |
| 1万亿元(人民币) | ≈$138B | ❌ $1T |
| 260亿美元 | $26B | ❌ $260B |
| N亿美元 | $(N/10)B | — |
| N亿元(人民币) | ≈$(N/7.2)B | — |

### 铁律
- 任何含「亿」的中文金融数据 → 强制经过换算表校验再输出英文数字
- snippet来源的融资/估值数字 → 必须标注「⚠️ Bing snippet解读，已过亿/billion换算校验」
- 中文「N亿美元」=(N/10) billion，非N billion
- 必须同时标注 $XB 格式。例：「260亿人民币」→「$26B（260亿人民币）」

## 二、中文源时效三要素标注

每条中文情报源必须同时标注：

```
✅ 正确: "来源: https://www.36kr.com/p/3828877773886338 (36氪, A级, 2026-05-28)"
❌ 错误: "来源: 36氪"
✅ 正确: "日期: 2026-06-08→06-23" (持续更新类内容标注起止日期)
❌ 错误: "最近报道"
```

### 时效一致性检查
- 每条有具体日期？→ 无日期的降级为S3
- 日期是否≤7天？→ 超7天内容不得放入「今日必看」
- 持续更新内容(如教程)标注起止日期范围

## 三、管道诚实度声明

每轮情报扫描开头必须声明：

```
⚠️ 搜索管道声明：
├── 可达管道: [实际可用的搜索引擎/社媒API列表]
├── 不可达管道: [被屏蔽/未接入的工具列表]
├── 已知偏差: [搜索引擎的地域/语言偏向]
└── 替代策略: [当前使用的变通方案]
```

### 管道评估表
| 维度 | 评级 | 说明 |
|------|:----:|------|
| 搜索可达性 | 🟡/🔴 | 具体局限 |
| 社媒覆盖 | 🟡/🔴 | 缺失管道 |
| 新闻时效 | 🟡/🔴 | 可用内容范围 |
| 来源多样性 | 🟡/🔴 | 语言/地域覆盖 |

### 铁律
- 不说话 ≠ 没问题。管道局限必须前置声明，不留给情报消费者猜测。
- 「未找到」和「不存在」是两种不同结论 → 管道局限下永远只说「本轮扫描未找到」。

## 四、来源URL有效性三重门

每条来源必须通过以下三条检查：

```
□ 门1: 标注为具体URL，非「媒体名+日期」
□ 门2: URL可访问（已用web_fetch验证或截图为证）
□ 门3: 如SPA站点无法提供独立文章URL → 诚实标注⚠️并降级信号（S1→S2）
```

### 违规处理
「来源URL全覆盖」声明若含无URL引用 = 质量声称虚假 → 自行记录I-xxx并扣分。
