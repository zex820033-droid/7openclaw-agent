# 平台搜索策略参考

## ⚠️ 时间过滤规范（重要）

**禁止在搜索语句中使用 `after:` / `before:` 参数** — 当前搜索引擎对这类参数支持不稳定，实际执行中常常不生效，只会返回相同的重复结果白白浪费轮次。

**正确的时间控制方式：**
1. 搜索词中加入年月关键词，如 `{关键词} 2026年4月` 或 `{关键词} 最新`，引导搜索引擎优先返回近期内容
2. 拿到搜索结果后，**逐条检查摘要或标题中的日期信息**（通常有"X天前"、"X月X日"、"X小时前"等标注）
3. 在用户指定时间范围内 → 收录；超出范围 → 跳过，不纳入分析

## 搜索效率最佳实践

### OR 组合查询（提升效率）
将同类平台合并为一条搜索，减少轮次：
```
{关键词} site:36kr.com OR site:huxiu.com OR site:ithome.com {年月}
```

### 多角度覆盖（3批 × 4并行 = 12轮）
| 批次 | 内容 |
|------|------|
| 第一批 | 微博、知乎、B站、头条+网易 |
| 第二批 | 腾讯+澎湃、新浪+搜狐、科技媒体、央媒+财新 |
| 第三批 | 小红书、公众号+百家号、负面专项、正面+热点专项 |

### 时间验证流程
1. 搜索完成后，检查每条结果摘要中的日期标注（"X天前"、"X月X日"等）
2. 若摘要中日期不明确，对重要结果使用 `web_fetch` 抓取后查看页面发布时间
3. 确认在用户指定范围内方可收录
4. 超出范围的记录为"已过滤"，最终报告末尾可注明过滤数量

### 搜索量目标
- **有效收录**：筛选后时间范围内条目 **50+ 条**
- **平台覆盖**：**12+ 个平台**有效数据
- **深度抓取**：**10-15 篇**完整正文（质量优先，不贪多）

### 去重规则
- 同一事件在不同平台的报道，保留最详细的 1-2 条
- 标题相同或高度相似的内容合并
- 转载内容标注原始来源
