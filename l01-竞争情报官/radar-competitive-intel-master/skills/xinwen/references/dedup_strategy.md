# 去重策略说明

## 核心思路

同一事件往往被多个信源同时报道，导致信息冗余。去重策略的目标是：
1. 识别同一事件的不同报道
2. 合并为一条情报
3. 保留不同视角的关键差异
4. 标注"X个源报道"提升可信度

## 技术实现

### TF-IDF + 余弦相似度

不依赖额外的NLP库，使用纯Python标准库实现：

1. **分词**：
   - 英文：按空格分词
   - 中文：2-4字滑窗n-gram

2. **TF-IDF向量**：
   - TF（词频）= 词出现次数 / 总词数
   - IDF（逆文档频率）= log(文档总数 / (包含该词的文档数 + 1)) + 1
   - TF-IDF = TF × IDF

3. **余弦相似度**：
   - 计算两个TF-IDF向量的余弦夹角
   - 范围 [0, 1]，越接近1越相似

### 相似度阈值

- 阈值设为 **0.6**
- 0.6以上视为同一事件，进行合并
- 0.6以下视为不同事件，保留各自条目

### 合并规则

1. **主条目选择**：选择热度最高的报道作为主条目
2. **来源合并**：收集所有报道的来源，去重后标注"X个源报道"
3. **URL合并**：保留所有来源的URL，方便用户查看不同视角
4. **多源加分**：
   - 3+源：+15分
   - 2源：+8分
   - 1源：+0分

### 并查集优化

使用并查集（Union-Find）处理传递性合并：
- A与B相似，B与C相似 → A、B、C归为一组
- 避免O(n³)的分组计算

## 局限性

- 基于标题相似度，不涉及正文分析
- 中英文混合标题的去重效果略差
- 对同义词的识别能力有限（如"发布"和"推出"不会自动关联）
- 这些局限在当前轻量级方案中是可接受的trade-off
