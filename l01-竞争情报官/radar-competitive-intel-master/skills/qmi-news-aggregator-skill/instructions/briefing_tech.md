# 🤖 科技大厂早报

> **INPUT**: `tech_blogs`（11 个大厂技术博客）+ `tech_keywords`（百度热搜 + 微博热搜的科技关键词）
> **OUTPUT**: 面向工程师和创业者的深度科技日报
> **目标用户**: 技术人、PM、技术媒体作者

---

## 🎯 聚焦方向
1. **AI/大模型 SOTA**: 国内大模型新版本（Qwen / 文心 / 豆包 / 混元 / 盘古 / DeepSeek）
2. **云原生**: K8s、Serverless、容器、调度系统
3. **基础架构**: 数据库、存储、CDN、计算优化
4. **工程实践**: 性能优化、稳定性、成本控制

---

## ⚠️ 严肃规则
1. **数量**: 大厂技术博客挑 **15-20 条**有干货的；科技热搜挑 **5-10 条**
2. **去 PR 化**: 跳过纯 PR 文（"某某产品重磅发布"），重点是**技术原理、踩坑、对比**
3. **数据说话**: 有 benchmark / QPS / 性能数字的优先

---

## 📝 报告结构

### Part 1: 🚨 AI 前沿（Qwen / 文心 / 豆包 / 混元 / 盘古 / DeepSeek）

每条格式:
```markdown
#### N. [标题](url)
- **来源**: 阿里云 / 字节 / 百度 AI | **时间**: 1h ago
- **摘要**: 简述新模型 / 新论文 / 新功能
- **技术深度**: 💡 创新点、benchmark、对比国外模型、与上一版本差异
- **应用场景**: 谁会用？怎么用？
```

### Part 2: 🛠️ 基础架构（阿里云 / 腾讯云 / 华为云 / 美团 / 滴滴）

每条格式:
```markdown
#### N. [标题](url)
- **来源**: 美团技术 | **作者**: 某某 | **时间**: 2h ago
- **摘要**: 一句话技术摘要
- **技术深度**: 💡 解决了什么工程难题？用了什么技术栈？
- **可借鉴**: 其他公司 / 团队能学到什么？
```

### Part 3: 🦄 行业观察（机器之心 / 量子位 + 科技热搜）

每条格式:
```markdown
#### N. [标题](url)
- **来源**: 量子位 | **时间**: 30min ago
- **摘要**: 一句话核心观点
- **Insight**: 💡 这篇文章揭示了什么行业趋势？
```

---

## 🎨 风格
- **语言**: 简体中文
- **风格**: 工程师文化 + 深度分析
- **术语**: 保留技术黑话（GPU 显存 / 推理成本 / 量化精度 / MoE 架构）
- **数字**: 用 `23.5%`、`1.2亿次` 这种格式
- **对比**: 国内外对比时直接给数据
