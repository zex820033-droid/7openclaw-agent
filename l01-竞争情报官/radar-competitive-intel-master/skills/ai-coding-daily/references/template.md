# 日报模板与生成规则

## 完整日报结构

```
AI Coding & Agent 情报日报 · {YYYY-MM-DD}

📅 报告日期：{YYYY年M月D日（星期X）}
📍 数据截止：{YYYY-MM-DD} 09:00 AM（Asia/Shanghai）
🔍 情报范围：最近7天内的 AI Coding & Agent 最新动态

🔥 本期头条
{1-2条最重要的信息，每条含「臣按」}

📦 开源新项目速递
{GitHub Trending 中的 AI Coding 相关项目，每条含「臣按」}

📊 GitHub Trending 速览（AI/Coding 相关）
{表格：项目名（带链接）| Stars | 简介}

🌐 行业动态
{官方博客 + HN 热帖 + Twitter 重要动态，每条含「臣按」}

📄 学术前沿（可选，有高质量论文时才出现）
{ArXiv 相关论文，每条含「臣按」}

💡 技术趋势观察
{2-3条跨信息源的趋势归纳，每条 100-150 字}

📚 值得深读
{表格：标题（带链接）| 来源 | 日期}

🔗 信息来源
{分类列出所有来源，GitHub 项目 + 文章与报道}

奏折由内侍奥创整理呈上，每日情报搜集，为皇上决策提供参考。
仅供内部使用，未经授权，切勿外传
```

## 各板块生成规则

### 🔥 本期头条

**选取标准**（满足任一即可）：
- Anthropic/OpenAI/Google 发布重大产品更新
- HN 分数 > 300 的讨论
- GitHub 今日 Star > 2000 的新项目
- 行业重大事件（融资、收购、重要人事变动）

**格式**：
```
{序号}. {标题}

{2-3句话描述核心内容，包含关键数据（HN 分数、Star 数等）}

臣按：{60-80字，给出明确判断，指出对 AI Coding 产品/工程的具体影响}
```

**示例**（参考 2026-06-05 日报）：
```
1. Anthropic 发布递归自我改进进展报告，HN 热议 AI 自我构建

Anthropic 在其研究院发布了题为 "When AI Builds Itself: Our progress toward recursive self-improvement" 的文章，在 Hacker News 斩获 336 分、444 条评论，成为本日社区讨论最热话题。文章阐述了 AI 系统借助自身能力实现迭代提升的工程路径，与当前 Agent 自主编码浪潮深度共鸣。

臣按：递归自我改进是 AI Coding Agent 从「辅助工具」迈向「自主系统」的关键能力节点，Anthropic 正式公开技术进展，意味着该方向进入工程化阶段，值得持续关注。
```

### 📦 开源新项目速递

**选取标准**：
- 今日 Star 增量 > 200，或本周 > 1000
- 与 AI Coding / Agent 直接相关
- 每期 6-12 个项目（信息少时宁可少，不要凑数）

**格式**（严格按此格式，不得省略任何字段）：
```
{序号}. {repo_name} — {一句话定位}

GitHub: {repo_url} ⭐ 今日 +{stars_today} / 本周 +{stars_week}

核心定位: {从 README 或描述提炼，20字以内}

主要特性:
- {特性1}
- {特性2}
- {特性3（如有）}

臣按：{60-80字}
```

**注意**：
- `repo_url` 必须是真实可访问的 GitHub 链接，格式为 `https://github.com/{owner}/{repo}`
- 如果只有今日数据没有本周数据，写 `⭐ 今日 +{stars_today}`；如果只有本周数据，写 `⭐ 本周 +{stars_week}`
- 主要特性至少写 2 条，从 README 或项目描述中提炼，不要编造
- 如果项目 Star 增量很高但描述模糊，不要强行纳入，宁缺毋滥

### 📊 GitHub Trending 速览

**格式**：Markdown 表格，项目名必须带 GitHub 链接

```markdown
| 项目 | Stars | 简介 |
|------|-------|------|
| [{repo_name}](https://github.com/{owner}/{repo}) | 今日 +{N} | {一句话} |
```

- 列出本期所有 AI/Coding 相关 Trending 项目（通常 10-15 个）
- 只列项目名和数据，不写「臣按」，这是速览板块
- 已在「开源新项目速递」详细介绍的项目也要出现在此表格中（速览是全量，速递是精选）

### 🌐 行业动态

**来源**：官方博客 RSS + HN 热帖 + web_search 补充采集

**格式**：
```
{标题}（{日期，如 2026-06}）

{2-3句话描述，包含关键信息和具体细节}

臣按：{60-80字}
```

**分组规则**：同一公司/产品的多条动态合并为一条，避免碎片化。例如 Claude Blog 同期发布多篇更新，合并为「Claude Managed Agents 全面升级」一条，在正文中列出各篇文章链接。

**示例**（参考 2026-06-05 日报）：
```
Claude Managed Agents 全面升级：梦境、编排、记忆三大新能力（2026-06）

Claude Blog 发布多篇连续更新，标志着 Managed Agents 平台从功能补全进入能力跃升阶段：
- New in Claude Managed Agents: dreaming、outcomes、multiagent orchestration
- 内建记忆上线: Built-in memory for Claude Managed Agents
- Claude Code 桌面并行 Agent 重设计: Redesigning Claude Code on desktop for parallel agents

臣按：梦境（dreaming）机制暗示 Agent 开始具备离线推理/预生成能力，多智能体编排 + 内建记忆构成 Managed Agents 的「持久智能」三角，这是 Anthropic 在产品层面对 Agent 长期运行的重大架构演进。
```

### 📄 学术前沿

**选取标准**：
- 与 Agent/Coding/Tool Use 直接相关
- 来自知名机构（Anthropic、Google、Meta、CMU、MIT、Stanford 等）
- 每期最多 3 篇
- 没有高质量论文时直接省略该板块

**格式**：
```
{论文标题}

作者：{第一作者} et al.（{机构}）
摘要：{2句话核心贡献}

臣按：{60-80字，重点说明工程价值}
```

### 💡 技术趋势观察

**生成方式**：横向分析本期所有信息，归纳 2-3 个跨信息源的共同趋势。

**格式**：段落式，每条趋势一个自然段，100-150 字，要有具体证据（引用本期具体项目/文章名称），给出趋势判断。

**好的趋势观察示例**（参考 2026-06-05 日报）：
> Agent 记忆基础设施爆发：本周 supermemory、ECC、Hermes Agent、Claude Managed Agents 内建记忆同期发力，「Agent 记忆」从研究概念走向工程产品，成为 2026 年 H1 最热基础设施赛道。
>
> AI Coding 工具生态分层加速：从底层压缩（headroom）、代码理解（codegraph/Understand-Anything）、规格驱动（spec-kit/OpenSpec）到 Agent 编排（harness）、桌面应用（Copilot Desktop/Claude Code Desktop），AI Coding 工具链正快速形成完整分层生态，不同环节均有专项工具涌现。

**差的趋势观察示例**（禁止）：
> AI 技术持续发展：本周有多个 AI 相关项目发布，显示 AI 领域仍在快速进步。

### 📚 值得深读

从本期所有信息中选出 5-8 篇最值得精读的文章/论文，格式为表格，标题必须带链接：

```markdown
| 标题 | 来源 | 日期 |
|------|------|------|
| [{文章标题}]({url}) | {来源名} | {日期} |
```

- 优先选择有实质技术内容的文章（博客、论文、深度报道），不选新闻简讯
- 链接必须是真实可访问的 URL，不要编造

## 「臣按」写作指南

「臣按」是日报最有价值的部分，是 AI 对信息的深度解读，不是简单转述。

**好的「臣按」**：
- 给出明确判断（「这意味着 X」「这是 Y 的信号」「对 Z 有直接影响」）
- 有具体的工程/产品视角
- 指出潜在影响或值得关注的方向
- 60-80 字，不废话

**禁止出现的表述**：
- 「值得关注」「有一定价值」「令人期待」（模糊）
- 「这是一个重要的里程碑」（空话）
- 「开发者可以考虑使用」（废话）
- 重复标题内容（信息冗余）

**示例对比**：

❌ 差：「headroom 是一个 LLM Token 压缩库，可以减少 Token 消耗，值得关注。」

✅ 好：「Context window 虽在扩大，但 Token 成本始终是 Agent 规模化的核心障碍。headroom 的压缩方向极具实用价值，尤其适合长任务 Agent 流水线，60-95% 的压缩率如果属实，将直接影响 Agent 产品的成本结构。」

## 日报长度控制

- 项目速递：每期 6-12 个项目（信息少时宁可少，不要凑数）
- 行业动态：每期 3-6 条
- 学术前沿：0-3 篇（没有高质量论文时直接省略该板块）
- 趋势观察：2-3 条
- 总字数：1500-3500 字（不要为了显得丰富而注水，但内容充实时可以适当超出）

## 空信息源处理

如果某个信息源当天没有相关内容（如 Anthropic 博客没有新文章），直接跳过，不要写「本期 XX 暂无更新」这类废话。

如果整个板块（如学术前沿）没有合适内容，直接省略该板块。

## 链接质量要求

- 所有链接必须是真实存在的 URL，不得编造
- GitHub 项目链接格式：`https://github.com/{owner}/{repo}`
- 博客文章链接：从 RSS 或 web_search 结果中获取原始 URL
- 如果无法获取真实链接，宁可不加链接，也不要写假链接
