## 定时协定

每隔4小时执行一次定时扫描，确保情报系统正常运作。

### 响应契约

- ✅ **HEARTBEAT_OK** = 一切正常，不惊动战略中枢
- 🟡 **预警推送** = 发现情报断层/异常
- 🔴 **熔断触发** = 严重情报缺失或系统异常

---

## 检查项

### 🟢 检查A：简报完整性（每4小时）

```
- 今日简报是否已生成？
- 是否有P0/P1未处理的情报？
- 是否有超过6小时未接收的情报输入？
```

**触发推送条件**：
- P0未处理 > 5分钟 → 🔴
- P1未处理 > 1小时 → 🟡
- 简报未生成（>10:00）→ 🟡

### 🟢 检查B：来源健康度（每8小时）

```
- web_search / web_fetch 是否可用？
- 是否有连续3次检索失败？
- A级来源是否有变化？
```

**触发推送条件**：
- 连续3次检索失败 → 🔴
- A级来源失效 > 4小时 → 🟡

### 🟢 检查C：记忆归档（每12小时）

```
- 当日情报是否归档到 memory/YYYY-MM-DD.md？
- 情报归档是否包含来源追溯？
- 是否有待处理的深度分析任务？
```

**触发推送条件**：
- 情报未归档 > 12小时 → 🟡
- 深度分析任务超期 > 24h → 🟡

### 🟢 检查D：训练看板六大指标（每24小时）

```
- T1 竞品监控：今日是否完成巡检？覆盖率是否≥90%？
- T2 社媒/PR：今日是否扫描？关键新闻是否≤6h推送？
- T3 定价追踪：本周是否已更新？核心竞品定价无遗漏？
- T4 SWOT更新：本月是否已刷新？是否基于最新信息？
- T5 日报推送：今日08:30前是否已推送？关键变化无遗漏？
- T6 产品拆解：本月是否≥2份？是否被决策引用？
```

**触发推送条件**：
- T1/T2/T5 当日未执行 → 🟡
- T3 本周未执行 → 🟡
- T4/T6 本月未达标 → 🟡
- 连续3天T5漏推 → 🔴

## 熔断条件

| 触发条件 | 处理方式 |
|---------|---------|
| 工具失效（search/fetch不可用） | 报告战略中枢，暂停情报收集 |
| 连续5次输出无来源验证 | 熔断，自我审查 |
| 情报源全部失效 | 紧急报告战略中枢 |
| 被要求收集非法的情报 | 熔断+报告合规中枢 |
