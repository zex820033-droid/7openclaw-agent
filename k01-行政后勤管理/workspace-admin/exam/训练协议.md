# 行政虾训练协议

> 版本：v1.0
> 目标：通过题包、答卷、批改、整改单，让 k01 行政虾把知识包转化为可执行能力。

## 1. 目录约定

| 目录 | 责任方 | 说明 |
|------|--------|------|
| `exam/exams/` | 老师 | 存放题包，行政虾只读 |
| `exam/answers/` | 行政虾 | 存放答卷，行政虾作答 |
| `exam/feedbacks/` | 老师 | 存放批改、评分、整改单 |
| `exam/进度.md` | 老师维护 | 当前轮次、状态、下一步 |

## 2. 文件命名

- 题包：`R<NN>-题包.md`
- 答卷：`R<NN>-答卷.md`
- 批改：`R<NN>-批改-整改单.md`

示例：

- `exam/exams/R01-题包.md`
- `exam/answers/R01-答卷.md`
- `exam/feedbacks/R01-批改-整改单.md`

## 3. 作答要求

每题必须包含：

1. 结论
2. 依据文件路径
3. 关键规则或数据
4. 下一步动作
5. 风险或边界提醒

## 4. 三证要求

行政类题目尽量留下三证：

- 产物存在：引用表格、台账、审批单或任务卡
- 前后对比：说明原状态与处理后状态
- 可复现验证：给出文件路径、编号或检查步骤

## 5. 评分口径

| 分数 | 标准 |
|------|------|
| 90-100 | 准确、完整、有边界、有风险意识 |
| 75-89 | 主流程正确，少量遗漏 |
| 60-74 | 能找到部分信息，但执行建议不完整 |
| < 60 | 事实错误、越权、缺少依据或无法执行 |
