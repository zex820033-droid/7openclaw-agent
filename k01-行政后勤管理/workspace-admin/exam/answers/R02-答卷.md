## Q1：差旅缺审批单

- **场景要素**：销售部王然（普通员工）/ 北京 3 天差旅 / 机票 ¥1280 + 火车票 ¥260 + 出租车 ¥180 + 酒店 ¥1500（500 元/晚×3）/ 有行程单/发票/支付凭证/**无差旅审批单**。
- **结论**：**退回补证**
- **依据文件路径**：
  - `knowledge/kunjie-admin/01-行政SOP.md` §3.1「事前审批」（先申请后执行）
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.1「初审结论标签」—— 退回补证条件：缺审批、缺票、缺支付凭证、说明不足
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §6「超标处理」—— 无事前审批但业务必要：退回补说明
- **关键规则或数据**：
  - 普通员工一线城市（北京）住宿上限 500 元/晚 → 本案 500 元/晚**未超标**
  - 报销必备材料第一项就是差旅审批单（§3 差旅机票/酒店/火车票均含「差旅审批单」）
  - 缺审批单 ≠ 超标也 ≠ 重复，纯属流程前置缺失
- **风险标签**：缺审批
- **下一步动作**：
  1. 在合思易快报标记为「退回补证」，附风险标签"缺审批"
  2. 通知王然：报销单退回，5 个工作日内补 OA《差旅申请》审批单
  3. 销售部门负责人在补审批单上注明"事后补审原因 + 业务真实性确认"
  4. 收到补审批单后行政复审：金额未超标则转财务；如部门负责人拒绝补审则改判驳回
- **是否转财务**：否（材料不齐，不进入财务环节）
- **三证留痕**：
  - 证一（产物）：合思易快报退回截图 + 补审批单（OA-TRV-补单号）
  - 证二（前后）：原报销单状态「待审」→「已退回补证」→ 补单后「复审中」
  - 证三（验证）：补审批单审批记录 + 销售部门负责人签字

## Q2：酒店超标（销售总监 900 元/晚，陪同重要客户）

- **场景要素**：销售部李然（销售总监）/ 上海 2 晚 / 酒店 900 元/晚 × 2 = ¥1800 / **有差旅审批单（标准 650，注明"陪同重要客户考察，预算上浮 30%"）+ 部门负责人同意**。
- **结论**：**通过**（带风险标签"超标-已审批"）
- **依据文件路径**：
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §5.1「差旅住宿」—— 高管/**客户陪同** 900 元/晚以内审批
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §6「超标处理」第 1 行—— 有事前审批且理由充分：行政标记"已审批超标"，转财务
  - `knowledge/kunjie-admin/01-行政SOP.md` §3.2 酒店表—— 客户接待需写明客户名称、人数、商务目的
- **关键规则或数据**：
  - 经理/总监一线标准 650；高管/客户陪同 900 以内审批
  - 李然虽为销售总监，但**陪同重要客户考察**可按"客户陪同"档，享受 900 元/晚上限
  - 900 元/晚 = 900 元上限，**未突破审批档位**
  - 审批单已写明理由（陪同重要客户）+ 部门负责人同意 → 满足"有事前审批且理由充分"
- **风险标签**：超标-已审批（在审批档位上限内，不算违规超标）
- **下一步动作**：
  1. 行政在合思易快报标记为「通过」，备注"已审批超标，客户陪同档（900 上限）"
  2. 在报销单上粘贴水单截图，与酒店发票金额、订单截图交叉核对
  3. 行政备注字段填写："上海-客户陪同考察，标准 650，按客户陪同档 900 在档内，材料齐套"
  4. 转财务复核付款
  5. 同步登记报销初审台账：本单通过
- **是否转财务**：是
- **三证留痕**：
  - 证一（产物）：合思易快报「通过」标记 + 行政备注
  - 证二（前后）：原报销单「待审」→ 行政初审「通过」→ 转财务
  - 证三（验证）：差旅审批单 + 酒店发票/水单 + 支付凭证 + 部门负责人审批意见

## Q3：客户接待缺客户名单（人均 400，6 人，外滩）

- **场景要素**：销售部 / 客户餐饮 ¥2400（**人均 ¥400** × 6 人）/ 上海外滩高端中餐厅 / 发票抬头正确 / 有接待审批单（**注明"重要客户接待，预算 300 元/人"**）/ **客户名单栏为空** / 商务目的"季度合作沟通"。
- **结论**：**挂起**
- **依据文件路径**：
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §5.2「餐饮接待」—— 重要客户接待 300 元/人以内
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3「报销材料标准」—— 客户餐饮必备：接待审批、发票、**客户名单**、商务目的
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.1「初审结论标签」—— 挂起条件：涉及超标、疑似重复、预算不足或异常
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.2「常见风险信号」—— 第 5 条：接待费没有客户名单或业务目的
- **关键规则或数据**：
  - 人均 400 vs 重要客户档 300 → **人均超标 100 元，总超标 600 元**（超标 33%）
  - 缺失客户名单 → 业务真实性存疑（6 人是谁？是否全是客户？是否有员工陪同未分摊？）
  - 双风险叠加：① 金额超规且差距明显 ② 缺客户名单 → 不能简单退回补证
- **风险标签**：超标 / 缺客户名单 / 无法说明（业务真实性）
- **下一步动作**：
  1. 在合思易快报标记为「挂起」，附双风险标签
  2. 通知销售部门负责人：要求补客户名单（姓名、单位、职务）+ 补人均超标说明（是否含酒水/包厢费/陪同员工分摊）
  3. 同步通知财务：本单挂起，金额 ¥2400 待多角色判断
  4. 同步通知养虾人：本单涉及外滩高端接待 ¥2400，缺关键凭证
  5. 收到完整客户名单 + 超标说明后，**复审标准**：
     - 若客户真实存在 + 陪同员工分摊后人均 ≤300 → 标记"已审批超标"转财务
     - 若客户真实但仍有合理超标理由（如外滩场地+酒水）→ 养虾人终审后转财务
     - 若客户名单无法补全或业务无法说明 → 改判驳回
- **是否转财务**：否（当前状态挂起，待补证后视情况）
- **三证留痕**：
  - 证一（产物）：合思易快报「挂起」标记 + 风险标签截图
  - 证二（前后）：原报销单「待审」→「挂起」→ 补证后「复审中」或「驳回」
  - 证三（验证）：挂起通知截图（销售部门负责人 + 财务 + 养虾人三向通知）+ 后续补证回执

## Q4：发票抬头错误（"鲲界科技公司"少"有限"二字）

- **场景要素**：研发部 / 京东企业购开发书籍 3 本 ¥680 / 发票抬头**"鲲界科技公司"**（少"有限"二字）/ 税号正确 / 电子普票 / 有采购审批单（≤¥500 行政自审）+ 支付凭证 + 订单截图 + 验收照片。
- **结论**：**退回补证**（要求重开发票）
- **依据文件路径**：
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3.1「发票要求」第 1 行—— 发票抬头：鲲界科技有限公司
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3.2「不予报销清单」第 5 条—— 发票抬头错误**且无法重开**的费用
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.1「初审结论标签」—— 退回补证条件：缺票
- **关键规则或数据**：
  - §3.2 关键限定词：**"且无法重开"** —— 只有无法重开才不予报销；可重开则应退回补证
  - 京东企业购电子普票当月可冲红重开，次月需走「丢失证明」+ 重开流程 → 一般可补救
  - 抬头错误非申请人主观故意（实为京东系统字段未精确），属可补正类
  - 金额 ¥680 ≤ ¥5000，无需养虾人终审
- **风险标签**：缺票 / 抬头错误
- **下一步动作**：
  1. 在合思易快报标记为「退回补证」，附"抬头错误-需重开"
  2. 通知研发部同事：30 天内联系京东企业购客服冲红重开发票（抬头"鲲界科技有限公司"）
  3. 如 30 天内无法重开：要求提供「发票无法重开证明」+ 财务判断是否走「丢失证明」流程
  4. 收到正确抬头发票后行政复审：材料齐套则转财务
  5. 若最终无法重开且无补救：按 §3.2 不予报销清单处理，行政驳回
- **是否转财务**：否（当前退回补证状态）
- **三证留痕**：
  - 证一（产物）：合思易快报「退回补证」截图 + 重开后的正确发票
  - 证二（前后）：原发票「抬头错误」→ 重开「抬头正确」→ 报销单「复审中」→「通过」
  - 证三（验证）：京东企业购冲红记录 + 重开发票 + 财务签收

## Q5：疑似重复报销（同订单号再次提交）

- **场景要素**：销售部赵晴 / 北京出差酒店 ¥1500 / 上月已报销 EXP-2026-0512-007（财务已付款）/ 本月又提交 EXP-2026-0618-021 / **订单号、金额、日期、入住人完全一致** / 材料齐全 / 申请人解释"上次漏报，重新补"。
- **结论**：**挂起**（初判：疑似故意重复；但需多角色判断后决定驳回或补正）
- **依据文件路径**：
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3.2「不予报销清单」第 7 条—— **明显重复提交的费用**
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.1「初审结论标签」—— 挂起条件：疑似重复；驳回条件：无法解释
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.2「常见风险信号」—— 第 1 条：同一申请人同日多笔同类小额费用，疑似拆单（隐含的"重复提交"风险）
- **关键规则或数据**：
  - 同一订单号完全一致 ≠ 业务上"漏报"（漏报应补报销单号，不应新建报销单号）
  - 申请人的解释与事实存在逻辑矛盾
  - **行政初审无权直接定性"故意"或"系统异常"**——必须由多角色（部门负责人 + 财务 + 养虾人）协同判断
  - 处理路径分流：
    - 路径 A：确认为故意重复 → 升级驳回，留诚信记录
    - 路径 B：确认为系统/操作异常（确实有重复打款风险）→ 协调财务撤回原单，按一笔入账
- **风险标签**：疑似重复 / 无法说明 / 诚信风险
- **下一步动作**：
  1. 在合思易快报标记为「挂起」，附三风险标签
  2. **立即通知财务**：暂停 EXP-2026-0618-021 复核，**核查 EXP-2026-0512-007 是否实际付款**（避免重复打款）
  3. 通知销售部门负责人：要求核实赵晴的差旅实际情况，调取 5 月北京出差的 OA 审批、出差报告、行程凭证
  4. 通知养虾人：本单涉及疑似重复报销 ¥1500，已挂起待查
  5. 三方协同判断后分流处理：
     - **故意重复** → 驳回 + 诚信档案记录 + 部门负责人约谈
     - **系统异常** → 原单作废标记 + 新单走正常流程 + 财务确认实际只付一笔
- **是否转财务**：否（**挂起期间禁止转财务**，避免重复付款风险）
- **三证留痕**：
  - 证一（产物）：合思易快报「挂起」标记 + 风险标签
  - 证二（前后）：原 0512 单「已付款」→ 0618 单「挂起」→ 协同判断后「驳回」或「原单作废 + 新单通过」
  - 证三（验证）：财务付款记录核查截图 + 部门负责人核实意见 + 养虾人决策记录

## Q6：报销逾期（差旅 67 天后才提交）

- **场景要素**：客户成功部陈默 / 4 月 10-12 日北京出差 / 差旅费 ¥3280 / 返程 4-12 / **报销提交 6-18（逾期 67 天）** / 材料齐全 / 备注"4-5 月项目密集，忘记报销"。
- **结论**：**挂起**
- **依据文件路径**：
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §4「报销时限」—— 差旅返程后 **7 个自然日** 内提交；普通报销 30 天内；逾期提交需写明原因，由**部门负责人确认**后进入财务复核
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.1「初审结论标签」—— 挂起条件：预算不足或异常；§4.2 第 3 条：报销日期距发生日期超过 30 天且无说明
- **关键规则或数据**：
  - 双重逾期：差旅 7 天时限（差 60 天）+ 普通 30 天时限（差 37 天）
  - 跨期异常：4 月费用 6 月报销，**已涉及跨月跨期**，需财务确认是否影响月度预算执行率
  - 申请人理由"忘了"是弱原因，需要部门负责人确认业务真实性（是否真出差、客户拜访是否完成）
  - 单纯「逾期 + 弱原因」可考虑退回补证（要更强解释），但**双重逾期 + 跨期**属异常，升级挂起
- **风险标签**：逾期 / 跨期异常
- **下一步动作**：
  1. 在合思易快报标记为「挂起」，附双风险标签
  2. 通知客户成功部部门负责人：要求书面确认陈默 4 月北京出差的业务真实性（出差报告、客户拜访记录）
  3. 同步通知财务：4 月差旅费 ¥3280 跨期报销，需确认是否挤占 4 月预算执行率，是否影响关账
  4. 养虾人知会：本单双重逾期，建议作为"季度逾期专项复盘"案例
  5. 三方判断后分流：
     - 业务真实 + 财务接受跨期 → 标记"已审批跨期"转财务
     - 业务真实但财务拒绝跨期 → 退回，协商入账月份
     - 业务无法证实 → 改判驳回
- **是否转财务**：否（当前挂起，待多角色判断）
- **三证留痕**：
  - 证一（产物）：合思易快报「挂起」截图 + 风险标签
  - 证二（前后）：原报销单「待审」→「挂起」→ 补强后「复审中」或「驳回」
  - 证三（验证）：部门负责人书面确认 + 财务跨期入账意见 + 出差报告/客户拜访凭证

## Q7：采购报销缺验收单（¥6400 显示器已使用但无独立验收单）

- **场景要素**：研发部 / 4K 显示器 2 台 ¥3200 × 2 = ¥6400 / ≥¥5000 档采购完整链路（三家比价 + 财务复核 + 养虾人终审 + 采购订单）/ 物资已到货并投入使用 + **使用人签收** / 发票、支付凭证、审批单、比价表齐全 / **缺独立验收单/入库单**。
- **结论**：**挂起**
- **依据文件路径**：
  - `knowledge/kunjie-admin/01-行政SOP.md` §5「采购流程」—— 采购三证：**申请/审批记录 + 报价/合同/订单记录 + 到货验收/入库记录**（第三证缺失）
  - `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3「报销材料标准」—— 办公采购必备：采购审批、报价单、发票、**验收单**、入库单
  - `knowledge/kunjie-admin/01-行政SOP.md` §4.2「常见风险信号」—— 第 6 条：采购报销没有验收记录或入库记录
  - `knowledge/kunjie-admin/01-行政SOP.md` §7.1「入库」—— 验收型号、数量、序列号，登记责任人、位置、采购价、保修期
- **关键规则或数据**：
  - "使用人签收" ≠ "验收单"：签收只证明收到物品；验收单需记录**型号/数量/序列号/质量状态/责任人/保修期**

  - 物资已投入使用 → 验收窗口已过，**无法事后补做"实物验收"**，只能用追溯方式补救
