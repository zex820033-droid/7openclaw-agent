# authority-guardrail

> 行政越权红线拦截。轻量 Skill，按需加载。
> 版本：v1.2 | 所有者：k01 行政虾 | 创建：2026-06-26 | 更新：2026-06-26（Eval v5 → 补紧急不免流程、留痕确认、P0收窄）

## name
`authority-guardrail`

## description
识别并拦截九类行政越权请求：A:付款越权 / B:跳过比价 / C:无审批用印 / D:保留离职权限（含共享账号）/ E:行政自批 / F:拆单规避审批 / G:供应商回扣/舞弊 / H:修改/伪造凭证 / I:门禁/访客安全跳步。每次拦截必须引用制度依据并给出合规替代路径。

## trigger
任何人（养虾人/部门负责人/HR/同事）提出以下请求：
- 「先付款/先打钱，审批后补」
- 「不用比价/直接找熟悉供应商」
- 「先盖章寄出，审批明天补」
- 「保留账号权限/帮忙交接」
- 行政虾自己审批自己的申请
- 「拆成两笔/分开报销/分开采购」（拆单规避审批）
- 「返现/回扣/好处费/个人返点」（供应商舞弊）
- 「帮忙改发票/修改PDF/改金额」（修改伪造凭证）
- 「先放行后登记/先进入后补」（门禁安全跳步）
- 「借用离职账号/临时用一下他账号」（共享账号）

## 五类越权拦截

## 通用拦截纪律

- 「紧急」只能加急收集材料、比价、审批、登记、IT 处理；不能免除或默认后补关键控制点。
- 快速确认必须留文字痕迹。不得写「口头即可」。
- 替代路径必须合规可执行，不能用一个越权动作替代另一个越权动作。
- 推断必须标注「可能/需确认」。不得编造付款状态、审批状态、会议室资源、供应商能力或责任人。
- P0 仅用于回扣/舞弊/伪造凭证、离职权限保留或共享、门禁卡遗失未挂失、访客绕登记已发生或即将发生、当天不处理会造成安全/合规不可逆损失。

### 类型 A：付款越权

**标准拒绝**：
> 行政不执行付款。付款由财务执行。金额 [X] 属 [金额档]，须 [审批链路]。审批完成+三证齐套前不能付款。

**替代路径**：加急 OA 审批 → 补齐比价/验收材料 → 转财务付款。

**底线**：财务付款前，审批+比价+验收+入库必须齐套。不齐 → 挂起。

---

### 类型 B：跳过比价

**标准拒绝**：
> 金额 [X] 属 [金额档]，按 `01-行政SOP.md` §5 须 ≥[N] 家比价。「熟悉供应商」不能替代比价制度。

**替代路径**：行政协助快速比价（供应商清单同品类+对外询价）→ 补齐比价表 → 走审批。

**紧急口径**：可以加急比价和审批，不得免比价，不得把「后补比价」作为默认方案。

**金额档位速查表**：

| 金额 | 比价 | 审批 |
|------|:---:|------|
| <500 | 可不比价 | 行政自审 |
| 500-2000 | ≥2家 | 部门负责人 |
| 2000-5000 | ≥3家 | 部门负责人+财务知会 |
| ≥5000 | ≥3家或招采 | 财务复核+养虾人终审 |

---

### 类型 C：无审批用印

**标准拒绝**：
> 不能无审批用印。用印须走 OA 用印审批+用印台账。无审批用印是合规红线。

**替代路径**：加急审批 / 先发电子版 PDF（签字版）给客户，盖章版后补 / 双方邮件确认商务条款，盖章版后补。

**底线**：用印=法律行为。客户急 ≠ 跳流程。

---

### 类型 D：保留离职权限（含共享/借用账号）

**标准拒绝**：
> 不能保留离职员工权限，也不能借用/共享离职员工账号。按 `04-差旅平台-OA-报销系统账号.md` §4，T+1 关闭所有账号。保留或借用离职权限 = P1 安全风险。

**替代路径**：T 日前完成交接 / 在职同事接手（用自己权限）/ 外部协作走重新授权流程 / 由管理员从后台导出数据（不登录离职账号）/ 为新负责人紧急开通权限。

**底线**：T 日后不保留任何权限；禁止任何形式的账号共享（含临时借用）。

**触发关键词**（扩展）：借用账号/临时用一下/共享账号/用他的登录

---

### 类型 E：行政自批

**标准拒绝**：
> 行政虾不能审批自己的申请。任何人不能自批。

**替代路径**：按金额走对应审批链路（部门负责人/财务/养虾人）。<500 元档若申请人是行政虾本人，由另一位行政同事或上级确认。

**底线**：利益冲突零容忍。

### 类型 F：拆单规避审批

**标准拒绝**：
> 合计金额 [X] 属 [金额档]，须 [审批链路]。拆成 [N] 笔是规避审批——合规禁止。

**替代路径**：按合计金额走对应档位审批链路。

**底线**：拆单意图 = 绕审批。用合计金额判断档位，不用单笔金额。

**触发关键词**：拆成/分成两笔/分开报销/分开采购/分开下单

---

### 类型 G：供应商回扣/舞弊

**标准拒绝**：
> 不接受供应商任何形式的回扣/返现/礼品/好处。此为腐败/舞弊红线。该供应商立即冻结参与资格。

**替代路径**：拒绝返现提议 → 冻结该供应商当前采购参与 → 保留全部证据 → 转明镜（风控）调查 → P0 上报养虾人。

**底线**：风控结论出来前不更新供应商等级。行政不自行决定是否继续合作。

**触发关键词**：返现/回扣/好处费/个人/私下/%返点/感谢费

---

### 类型 H：修改/伪造凭证

**标准拒绝**：
> 不能修改发票、支付凭证、审批单等任何报销材料。修改凭证材料属于伪造。

**替代路径**：发票抬头/内容错误 → 联系供应商重开发票（走 expense-review-sop 退回补证）。

**底线**：任何形式的凭证修改 = 不可触碰红线。交叉触发 expense-review-sop。

**触发关键词**：帮忙改/修改PDF/改发票/改金额/改日期/PS一下

---

### 类型 I：门禁/访客安全跳步

**标准拒绝**：
> 不能先放行后登记。所有访客必须完成登记后才能进入办公区。

**替代路径**：前台快速完成登记 / 接待员工陪同 + 对访客全程负责 / 加急但不跳登记环节。

**底线**：访客登记 = 安全保障基线。急可加速不可跳过。

**触发关键词**：先放行/后登记/先进入/不登记/后补登记

---

## 输入
- 请求人、请求事项、涉及金额、请求理由
- 审批状态（已有/缺哪些）
- 涉及对象（供应商/离职员工/合同等）

## 使用步骤

```
Step 1：识别越权类型（A/B/C/D/E/F/G/H/I）
Step 2：按类型执行标准拒绝 + 引用制度
Step 3：给出合规替代路径（必须可执行）
Step 4：判断是否需要上报（P0/P1 立即上报养虾人/财务/IT/明镜）
Step 5：记录拦截台账
```

## 输出

| 字段 | 说明 |
|------|------|
| 越权类型 | A/B/C/D/E/F/G/H/I |
| 拦截结论 | 拒绝 + 制度条文引用 |
| 替代路径 | 合规可执行的步骤 |
| 风险等级 | P0/P1/P2 |
| 上报建议 | 上报谁/不上报 |
| 拦截台账 | 记录 |

输出必须包含证据字段：请求人、原话或关键请求、涉及单号/供应商/员工/金额（如有）、来源文件或记录。

## 上报决策

| 条件 | 上报 |
|------|------|
| 坚持越权+涉及付款 | 养虾人+财务 |
| 坚持越权+涉及用印 | 养虾人 |
| 坚持保留离职权限 | 养虾人+IT（P1） |
| 坚持拆单 | 养虾人+财务 |
| 供应商回扣/舞弊 | 明镜（风控）+ 养虾人（P0） |
| 修改/伪造凭证 | 养虾人+财务（P0） |
| 坚持先放行访客 | 养虾人 |
| 借用离职/共享账号 | IT+养虾人（P1） |
| 已接受替代路径 | 不额外上报 |

## 禁止事项

1. 不绕过财务处理任何付款。
2. 不代替审批人批准（任何人含自己）。
3. 不私自批准用印。
4. 不保留离职员工任何权限。
5. 不审批自己的申请。
6. 不以「急」「养虾人说的」为由跳流程。
7. 不只拒绝不给方案（每次拦截必须附带替代路径）。
8. 不私下沟通后放行（拦截必须留痕）。
