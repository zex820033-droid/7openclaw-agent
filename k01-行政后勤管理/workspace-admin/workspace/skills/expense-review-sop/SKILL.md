# expense-review-sop

> 行政报销初审四步判断法。轻量 Skill，按需加载。
> 版本：v1.2 | 所有者：k01 行政虾 | 创建：2026-06-26 | 更新：2026-06-26（Eval v5 → 补证据纪律、四标签边界、拆单与重复案例）

## name
`expense-review-sop`

## description
行政报销初审标准操作程序。按「材料齐套 → 金额标准 → 风险标签 → 输出结论」四步完成初审，使用「通过/退回补证/挂起/驳回」四标签输出结论，给出风险标签和下一步动作。

## trigger
- 报销单到达行政虾初审环节
- 退补后重新提交的报销单需要再次初审
- 财务/养虾人要求对某笔报销给出初审意见

## 输入

| 字段 | 必填 |
|------|:--:|
| 报销单号、申请人、部门、金额、费用类型 | ✅ |
| 关联审批单（OA 审批单号） | ✅ |
| 发票、支付凭证 | ✅ |
| 业务说明 | ✅ |
| 客户名单（如有接待） | 条件必填 |
| 验收单+入库记录（如有采购≥5000） | 条件必填 |
| 申请人职级 | 条件必填 |

## 使用步骤

### Step 0：证据字段与数字复核

逐笔核对以下字段，不得凭印象下结论：报销单号、申请人、部门、职级、费用类型、金额、业务日期、提交日期、OA 审批、发票号、发票金额、支付凭证、客户名单、验收/入库、供应商、备注。

凡输出总数、通过/退回/挂起/驳回数量、金额合计、差额、住宿单晚金额、逾期天数、疑似重复数量，必须来自文件字段或展示计算依据。缺字段就写「缺字段，需补证」，不得补全。

### Step 1：材料齐套检查

对照 `02-预算口径与报销规则.md` §3 必备材料表逐项核对：

| 缺什么 | 结论 |
|--------|------|
| 缺审批单、发票、支付凭证、验收/入库等任一必备材料 | **退回补证** |
| 发票抬头/日期/金额与申请不一致 | **退回补证** |
| 全部齐套 | → Step 2 |

### Step 2：金额标准检查

对照 `02-预算口径与报销规则.md` §5 费用标准：

**差旅住宿速查**（一线/新一线/其他 × 普通/经理总监/高管客户陪同）：查 §5.1 表。

**采购阈值速查**：

| 金额 | 比价 | 审批 |
|------|:---:|------|
| <500 | 可不比价 | 行政自审 |
| 500-2000 | ≥2家 | 部门负责人 |
| 2000-5000 | ≥3家 | 部门负责人+财务知会 |
| ≥5000 | ≥3家或招采 | 财务复核+养虾人终审 |

判断：
- 标准内 + 有审批 → **通过**
- 超标 + 有审批 → 标记「已审批超标」→ **通过**
- 超标 + 无审批 → **挂起** 或 **退回补说明**
- ⚠️ 职级未给 → 标记「**职级待确认**」→ 分普通/经理总监/高管三档分别说明 → 退回确认。**不得脑补为普通员工。**

### Step 3：风险标签扫描

对照 `01-行政SOP.md` §4.2 风险信号清单：

| 信号 | 动作 |
|------|------|
| 发票不一致 | 退回补证 |
| 缺客户名单、缺验收/入库 | **挂起** |
| 逾期（差旅>7天 / 普通>30天）| **挂起**（简要理由≠正式说明，仍需书面说明+部门负责人确认） |
| 疑似重复（订单号/发票号/OA+金额+日期一致）| **挂起**→核实后驳回；关键字段完全一致可直接**驳回** |
| 疑似拆单（同申请人+同日+同供应商+同品类，多笔接近阈值）| **挂起**，按合并金额重判审批档位；核实前写「疑似」，不直接定性恶意 |
| 跨期 | **挂起**（影响入账当月预算） |
| **购物卡/现金卡/储值卡/现金等价物** | **挂起**（高合规风险，不得直接通过；需养虾人终审+财务合规判断；违反 `02-预算口径与报销规则.md` §5.5 礼品采购禁令） |
| 虚假/个人消费 | **驳回** |
| 无异常 | → Step 4 |

### Step 4：输出结论

**四标签**：

| 标签 | 条件 | 后续 |
|:---:|------|------|
| **通过** | 材料齐套 + 金额合规 + 无风险/已审批覆盖 | 转财务复核 |
| **退回补证** | 缺材料/信息可修正 | 退回申请人，列缺项 |
| **挂起** | 超标/重复/逾期/缺接待/跨期（需协同） | 通知部门负责人+财务+养虾人 |
| **驳回** | 明确重复/虚假/个人消费（不可逆） | 不转财务，留存 |

转财务时按 `02-预算口径与报销规则.md` §7 填写接口字段。

## 输出

- 初审结论（四个标签之一）
- 风险标签
- 下一步动作（补什么、通知谁、等什么）
- 是否转财务
- 财务接口字段（报销单号/申请人/部门/科目/金额/初审结论/风险标签/关联审批/行政备注）

## 禁止事项

1. **不承诺付款**。行政不付款，只确认资料齐套后转财务。
2. **财务付款前关键材料（审批/比价/验收/入库）必须齐套**；不齐 → 挂起，不先付款后补。
3. **不脑补职级**。未给 → 标记待确认，分档判断。
   - 可以加急找直属上级做文字确认并留痕；不得写「口头即可」。
4. 不替申请人编造业务目的/客户信息。
5. 不修改发票信息。
6. 不把无审批包装为已审批。
7. 不绕过财务处理付款。
8. 简要口头理由 ≠ 正式书面说明（逾期场景）。
9. 不编造税号/密码/公告来源。
10. 不把推断写成事实。维修、结算、供应商配合等因果关系只能写「可能影响，需确认」。
