## 草案 1：inventory-replenishment

### 1. Description

库存扫描、办公用品领用、安全库存对比、补货触发、采购阈值判断的综合程序。解决"什么快没了、要补多少、走什么采购流程"三个问题。

### 2. Trigger

| # | 触发场景 |
|:-:|----------|
| 1 | 每周一 10:00 常规库存扫描 |
| 2 | 员工领用办公用品后库存可能跌破安全线 |
| 3 | 新员工批量入职前的物资储备检查 |
| 4 | 大型活动/培训前物料需求确认 |
| 5 | 月度消耗异常（某品类消耗环比上涨>30%） |
| 6 | 「帮我看看中性笔还有多少」「转接头够不够」等口头询问 |
| 7 | 养虾人或部门负责人询问「最近需要补什么」 |
| 8 | 季度盘点时库存类资产专项扫描 |

### 3. Inputs

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 资产台账 CSV 路径 | ✅ | `03-资产台账.csv`，状态=库存/安防耗材/低值易耗品 |
| 安全库存标准 | ✅ | 来自 `01-行政SOP.md` §6 安全库存示例表 |
| 补货批量标准 | ✅ | 来自 `01-行政SOP.md` §6 补货批量列 |
| 领用事件（如有） | 条件 | 领用物品/数量/部门/用途，用于计算领用后库存 |
| 当前日期 | ✅ | 用于补货建议的时间上下文（如"本周三集中采购"） |
| 采购阈值表 | ✅ | `<500/500-2000/2000-5000/≥5000` 四档 |

### 4. Source Files

| 文件 | 读取章节 | 用途 |
|------|----------|------|
| `knowledge/kunjie-admin/03-资产台账.csv` | 全表，筛选库存类 | 当前库存数据 |
| `knowledge/kunjie-admin/01-行政SOP.md` | §6 办公用品流程 | 安全库存、补货批量、领用登记规则 |
| `knowledge/kunjie-admin/01-行政SOP.md` | §5 采购流程 | 金额→比价→审批映射 |
| `knowledge/kunjie-admin/05-供应商清单.csv` | 对应品类行 | 补货供应商查询 |

### 5. Workflow

```
Step 1：命令扫描库存
  使用 grep/awk 从 03-资产台账.csv 提取状态含"库存"的记录。
  输出每条记录：资产编号、名称、当前数量、安全库存、单位。

Step 2：对比安全库存
  对每条库存记录：
  - 当前量 ≥ 安全库存 → 标记"充足"
  - 当前量 < 安全库存 → 标记"低于安全库存"，计算缺口 = 安全库存 - 当前量
  输出低库存预警清单。

Step 3：考虑领用/消耗因子
  如有领用事件：当前量 - 领用量 → 重新计算是否跌破安全线。
  如为月度消耗异常分析：对比上月消耗量，判断异常涨幅。

Step 4：计算补货建议
  对每个低库存项：
  - 补货量 = max(补货批量, 缺口 + 安全缓冲)
  - 参考安全库存示例表中的补货批量列
  - 无补货批量的项（如门禁卡），建议补货至安全库存 × 1.5

Step 5：判断合并/拆分采购
  计算所有补货项预估总金额。
  如合并采购 → 按合计金额走对应审批档位。
  如拆分采购 → 每项单独走档位。
  ⚠️ authority-guardrail F：禁止为规避审批拆单。
  建议：同类供应商/同日采购优先合并。

Step 6：输出采购链路
  对每个补货批：
  - 金额 + 采购阈值 → 审批链路
  - 供应商（查 05-供应商清单.csv，A/B 优先）
  - 预计交付周期
  - 是否需要三证留痕（≥5000）

Step 7：更新台账建议
  列出需更新的台账字段：库存台账（入库后更新）、采购台账（PO生成后更新）。
```

## 草案 2：supplier-risk-review

### 1. Description

供应商合同到期预警、等级与风险评估、SLA 达标审查、续约/替换建议、回扣/舞弊事件衔接 authority-guardrail 的综合风险审查程序。

### 2. Trigger

| # | 触发场景 |
|:-:|----------|
| 1 | 每月供应商合同到期扫描（定期巡检） |
| 2 | 采购选型时需要从供应商清单筛选候选 |
| 3 | 员工反馈供应商服务质量下降（如茶水间清洁不及时） |
| 4 | 供应商提出返现/回扣/私下交易 |
| 5 | 供应商合同到期前需决策续签或替换 |
| 6 | 新增大额采购（≥5000）需要评估候选供应商风险 |
| 7 | 养虾人/财务要求输出供应商风险月报 |
| 8 | 收到供应商投诉或争议 |

### 3. Inputs

| 字段 | 必填 | 说明 |
|------|:----:|------|
| 供应商清单 CSV 路径 | ✅ | `05-供应商清单.csv` |
| 基准日期 | ✅ | 用于合同到期计算（默认当天） |
| 目标供应商 ID（如有） | 条件 | 单供应商风险评估 |
| 事件类型 | 条件 | 回扣/服务差/合同到期/选型 |
| 事件证据（如有） | 条件 | 员工反馈截图/返现沟通记录/满意度评分 |
| 资产/采购关联数据 | 条件 | 如补货涉及该供应商，需联动库存数据 |

### 4. Source Files

| 文件 | 读取章节 | 用途 |
|------|----------|------|
| `knowledge/kunjie-admin/05-供应商清单.csv` | 全表 20 条记录 | 供应商等级/合同/SLA/风险 |
| `knowledge/kunjie-admin/01-行政SOP.md` | §5.2 三家比价表 | 选型评估维度 |
| `knowledge/kunjie-admin/01-行政SOP.md` | §1.1 供应商台账 | 台账更新字段 |
| `workspace/skills/authority-guardrail/SKILL.md` | 类型 G | 回扣/舞弊处理标准 |

### 5. Workflow

```
Step 1：命令验证数据
  使用 awk/cut 从 05-供应商清单.csv 提取关键字段。
  确认供应商总数（20 家）、等级分布（A:11/B:7/C:2）、风险分布。
  所有数字必须命令验证，不凭记忆。

Step 2：合同到期扫描
  按基准日期计算距到期天数。
  分类：
  - 已到期：到期日 < 今天 → P1
  - 0-30 天到期 → 🟡 紧急预警
  - 31-60 天到期 → 🟡 预提醒
  - 60 天以上 → 正常
  输出预警清单。

Step 3：风险等级判定
  按风险等级+合同状态+服务 SLA 交叉判断：
  - 高风险（高+合同到期/即将到期）→ 🔴 P0 紧急：安防/门禁供应商断档
  - 中风险（中+C 级+已到期）→ 🟡 需决策续签或替换
  - 低风险 → 正常续约/年度评估

Step 4：SLA 达标审查
  如有服务质量投诉：
  - 收集具体证据（日期/频次/照片/反馈人）
  - 对照合同 SLA 逐项检查
  - 输出达标/不达标判定
  - 不达标 → 发整改通知 + 限期改善 + 月度评分降级

Step 5：回扣/舞弊衔接
  如涉及返现/回扣/私下利益 → 立即联动 authority-guardrail 类型 G：
  - 拒绝+留证+冻结+转明镜+P0 上报
  - 风控结论前不擅自改等级
  - 本 Skill 只做风险识别与上报，不做舞弊调查结论

Step 6：续约/替换建议
  对到期/即将到期供应商：
  - A 级+低风险 → 建议续约（复核条款/SLA）
  - B 级+中风险 → 重新比价后决定续签或替换
  - C 级+高风险/已到期 → 建议替换为 A/B 级替代供应商
  - 给出替代供应商候选 + 比价建议

Step 7：输出风险月报
  汇总：到期预警/等级变动/SLA 达标/回扣事件/替换建议。
```

### 5. Workflow

**入职线：**

```
Step 1：接收 HRBP 名单（T-2）
  确认：姓名/岗位/部门/办公点/入职日期。
  区分岗位类型：研发（高性能设备）/销售（便携设备+差旅权限）/其他。

Step 2：资产匹配（T-2）
  命令扫描闲置资产：
  - 当前闲置电脑：2 台（KJ-IT-2026-0021 MacBook Air M3 + KJ-RET-2024-0019 Dell Latitude 7420）。
  - 如需更多，判断采购或从在用设备调配。
  - 显示器/键鼠/转接头按岗位匹配。

Step 3：物资准备（T-1）
  工位：确认工位编号、清理、布置。
  门禁卡：从库存发放（当前 15 张，安全库存 20）。发后库存降低，触发补货评估。
  办公用品：入职包（中性笔/笔记本/便签等），注意库存。
  系统权限：通知 IT 开通邮箱+SSO → 行政按岗位开通差旅/报销/会议权限。

Step 4：入职当天
  引导门禁 → 工位 → 系统登录确认 → 资产签收单（电脑/显示器/门禁卡逐项签收）。
  更新资产台账：责任人/部门/位置/状态。

Step 5：T+1 确认
  检查差旅/报销/用车权限是否正确开通。
  确认所有资产正常使用。

Step 6：T+7 回访
  办公体验回访，处理设备/工位问题。
  闭环记录。
```

**离职线：**

```
Step 1：接收 HRBP 离职通知（T-3）
  列出该员工名下全部资产和系统权限。
  查资产台账确认当前责任人在册资产。

Step 2：确认待回收清单（T-1）
  与员工确认：笔记本/显示器/键鼠/转接头/门禁卡/工牌。
  提醒关闭前导出个人数据。

Step 3：T 日回收
  逐项检查资产状态（验机/外观/功能）。
  签收确认单。
  门禁卡立即停用/挂失（如有丢失→登记异常→立即停用该卡号）。
  更新资产台账：责任人→待分配、状态→闲置/待检测。

Step 4：T+1 权限关闭
  通知 IT 关闭所有系统账号。
  取消商旅/用车/报销/会议权限。
  ⚠️ authority-guardrail D：不得保留任何离职权限，不得共享/借用离职账号。

Step 5：T+3 盘点确认
  完成所有资产盘点。
  如有异常（未归还/损坏/丢失）→ 上报养虾人+HRBP。
  更新资产台账最终状态。
```
