## 四、需要整改的点

### 整改 1：所有精确数量必须命令验证

典型问题：

- `V4-ADMIN-001` 引用了中性笔 96、笔记本 44、转接头 8，但最后写“未触发命令验证”。
- `V4-HR-001` 引用了闲置电脑数量、门禁卡库存、中性笔库存，也没有明确命令验证。

整改要求：

以后只要出现以下内容，必须用 `grep` / `awk` / `wc` / `sort -u` 等命令验证，并在答卷中写明命令用途：

- 库存数量
- 资产状态
- 供应商数量
- 合同到期日
- 预算金额与执行率
- 人数、件数、题数

### 整改 2：回读不等于验证

“读取 CSV 后看到数字”只能算资料回读，不算强验证。  
如果题目要求“盘点、扫描、统计、低库存、合同到期、资产状态”，应该优先命令验证。

推荐句式：

```text
命令验证：使用 awk/grep 从 03-资产台账.csv 筛选库存类资产，确认当前库存与安全库存。
```

### 整改 3：下一批 Skill 需要从“流程描述”变成“可调用程序”

新 Skill 不能只是制度摘抄，必须写清：

- 触发条件
- 输入字段
- 操作步骤
- 输出格式
- 不能做什么
- 何时升级上报
- Eval 验证题型
