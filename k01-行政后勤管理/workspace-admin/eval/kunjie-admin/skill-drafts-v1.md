### 一、基本信息

| 字段 | 内容 |
|------|------|
| **name** | `expense-review-sop` |
| **description** | 行政报销初审四步判断法：按「材料齐套 → 金额标准 → 风险标签 → 输出结论」四步完成报销初审，使用「通过/退回补证/挂起/驳回」四标签输出结论，给出风险标签、下一步动作和财务接口字段。 |
| **频次** | 最高频核心业务。Eval v1 中 7/31 题（22.6%）直接涉及，涵盖差旅/采购/接待/办公各类报销场景。 |
| **复用条件** | 固定步骤可穷举；四标签覆盖所有初审结论类型；风险信号可从 SOP 4.2 对照。 |
| **关联整改** | 整改二（付款边界）、整改三（逾期说明）、整改四（职级待确认） |

### 二、trigger（触发条件）

以下任一情况触发本 Skill：

- 员工在合思易快报提交报销单，行政虾收到初审通知
- 部门负责人/财务/养虾人要求行政虾对某笔报销给出初审意见
- 报销单被退回后重新提交，需再次初审

### 三、输入

| 字段 | 说明 | 必填 |
|------|------|:--:|
| 报销单号 | 系统报销单编号 | ✅ |
| 申请人 | 提交报销的员工姓名 | ✅ |
| 部门 | 所属部门 | ✅ |
| 金额 | 报销金额（元） | ✅ |
| 费用类型 | 差旅/采购/接待/办公/维修/快递等 | ✅ |
| 关联审批单 | OA 差旅审批单/采购审批单编号 | ✅ |
| 发票 | 电子发票/纸质发票影像 | ✅ |
| 支付凭证 | 支付截图或银行流水 | ✅ |
| 业务说明 | 出差目的/接待对象/采购用途 | ✅ |
| 客户名单 | （如有接待）客户姓名+单位+职级 | 条件必填 |
| 验收/入库记录 | （如有采购）验收单+入库记录 | 条件必填 |
| 职级 | 申请人职级（普通员工/经理/总监/高管） | 条件必填 |
| 日期 | 费用发生日期 + 报销提交日期 | ✅ |

> ⚠️ **职级规则**：若输入未提供职级 → 标记「职级待确认」→ 分普通员工/经理总监/高管三档分别给出判断，不脑补为普通员工。

### 四、输出

| 字段 | 说明 |
|------|------|
| **初审结论** | `通过` / `退回补证` / `挂起` / `驳回` |
| **风险标签** | 无 / 超标 / 缺审批 / 缺票 / 逾期 / 疑似重复 / 跨期 / 职级待确认 / 缺客户名单…… |
| **下一步动作** | 申请人需补什么、行政需通知谁、需协同谁判断 |
| **是否转财务** | 是/否（通过 → 是；其余 → 否） |
| **财务接口字段** | 报销单号、申请人、部门、科目、金额、初审结论、风险标签、关联审批、行政备注（按 `02-预算口径与报销规则.md` §7） |
| **是否需要更新台账** | 是/否 + 哪个台账 |

### 五、使用步骤

#### Step 1：材料齐套检查

按 `02-预算口径与报销规则.md` §3 必备材料表逐项核对：

| 费用类型 | 必备材料 |
|----------|----------|
| 差旅机票 | 差旅审批单 + 电子客票行程单 + 支付凭证 |
| 酒店 | 差旅审批单 + 酒店发票 + 水单或订单截图 |
| 市内交通 | 交通订单 + 发票 + 行程说明 |
| 客户餐饮 | 接待审批 + 发票 + 客户名单 + 商务目的 |
| 办公采购 | 采购审批 + 报价单 + 发票 + 验收单 + 入库单 |
| 快递邮寄 | 快递面单 + 发票 + 业务归属 |
| 维修维保 | 报修工单 + 报价 + 维修记录 + 验收照片 |

**判断**：
- 任一必备材料缺失 → **退回补证**（列出缺哪些）
- 发票抬头/税号/日期/金额与申请不一致 → **退回补证**
- 材料齐全 → 进入 Step 2

#### Step 2：金额标准检查

按 `02-预算口径与报销规则.md` §5 费用标准逐项核对：

**2.1 差旅住宿**（§5.1）

| 城市等级 | 普通员工 | 经理/总监 | 高管/客户陪同 |
|----------|----------|-----------|---------------|
| 一线 | 500 | 650 | 900 以内审批 |
| 新一线 | 420 | 500 | 750 以内审批 |
| 其他 | 320 | 380 | 600 以内审批 |

**2.2 餐饮接待**（§5.2）

| 场景 | 标准 | 审批人 |
|------|------|--------|
| 内部加班餐 | ≤50/人 | 部门负责人 |
| 普通客户接待 | ≤150/人 | 部门负责人 + 行政备案 |
| 重要客户接待 | ≤300/人 | 养虾人或授权负责人 |
| 超标准 | 逐笔说明 | 养虾人终审 |

**2.3 采购阈值**（`01-行政SOP.md` §5）

| 金额 | 比价要求 | 审批链路 |
|------|----------|----------|
| <500 | 可不比价 | 行政自审 |
| 500-2000 | ≥2 家 | 部门负责人 |
| 2000-5000 | ≥3 家 | 部门负责人 + 财务知会 |
| ≥5000 | ≥3 家或招采 | 财务复核 + 养虾人终审 |

**判断**：
- 在标准内 + 有事前审批 → **通过**
- 超标 + 有事前审批 → 标记「已审批超标」→ **通过**
- 超标 + 无审批 → 退回补说明或**挂起**
- 职级未给 → 标记「职级待确认」→ 分三档分别说明 → 退回确认

#### Step 3：风险标签判断

按 `01-行政SOP.md` §4.2 常见风险信号逐项扫描：

| 风险信号 | 检查方法 | 命中处理 |
|----------|----------|----------|
| 拆单嫌疑 | 同一申请人同日多笔同类小额 | 挂起 |
| 发票不一致 | 抬头/税号/日期/金额与申请不符 | 退回补证 |
| 逾期 | 差旅>7天 或 普通>30天 | 挂起 |
| 城市不一致 | 酒店城市 ≠ 差旅行程城市 | 退回补证 |
| 缺客户信息 | 接待费无客户名单/业务目的 | 挂起 |
| 缺验收 | 采购报销无验收/入库 | 挂起 |
| 非工作时间用车 | 无加班/客户/行程说明 | 退回补证 |
| 重复报销 | 订单号+金额+日期+入住人一致 | 挂起→核实后驳回 |
| 跨期 | 费用发生月与提交月不一致且无说明 | 挂起 |

**判断**：
- 无风险信号 → 进入 Step 4
- 退回补证级 → Step 4 处一起退回
- 挂起级 → Step 4 标记挂起协同
- 明确重复/虚假 → Step 4 直接驳回

> ⚠️ **逾期细化规则**（整改三）：申请人已提供简要理由（如「项目忙忘了」）→ 不等于正式说明。仍需 **正式书面说明 + 部门负责人确认**。简要理由只是「不是完全无理由」，不是「理由已充分」。

> ⚠️ **跨期口径**（从 R02 固化）：跨期报销影响的是 **入账当月** 的预算执行率，非发生期。风险表述不能说「挤占发生月预算」。

#### Step 4：输出结论并转交

**4.1 四标签使用规则**

| 标签 | 触发条件 | 动作 |
|------|----------|------|
| **通过** | 材料齐套 + 金额一致 + 无风险（或风险已审批覆盖） | 转财务复核 |
| **退回补证** | 缺审批/缺票/缺支付凭证/缺说明（可修正型缺失） | 退回申请人，列明缺哪些 |
| **挂起** | 超标/疑似重复/预算不足/跨期/逾期严重/缺客户信息（需协同型风险） | 通知部门负责人+财务+养虾人协同判断 |
| **驳回** | 明确个人消费/虚假材料/确认重复/无法解释（不可逆型违规） | 不转财务，留存记录 |

### 一、基本信息

| 字段 | 内容 |
|------|------|
| **name** | `authority-guardrail` |
| **description** | 行政越权红线拦截：识别五类越权请求（付款/跳比价/无审批用印/保留离职权限/自批），依据制度明确拒绝，并给出合规替代路径。 |
| **频次** | 高风险核心业务。Eval v1 中 5/31 题（16.1%）直接涉及，且均为 L2-L3 级高风险场景。 |
| **复用条件** | 五类越权类型可穷举；拒绝句式标准；替代路径有模板。 |
| **关联整改** | 整改二（付款边界）、整改四（自批确认） |

### 二、trigger（触发条件）

以下任一情况触发本 Skill：

- 任何人（含养虾人/部门负责人/HR/同事）提出可能越权的行政操作请求
- 请求包含关键词：先付款/不用比价/直接找供应商/先盖章/明天补审批/保留权限/帮忙交接/自己审批
- 请求绕过现行 SOP 规定的流程和审批链路
- 请求让行政虾自行批准自己的事项

### 三、输入

| 请求事项 | 具体操作（付款/采购/用印/保留权限/自批） | ✅ |
| 涉及金额 | （如涉及）金额 | 条件必填 |
| 请求理由 | 「很急」「客户必须收到」「帮忙交接」等 | ✅ |
| 涉及对象 | 供应商/员工/离职人员/合同 | 条件必填 |

| 字段 | 说明 |
|------|------|

### 五、使用步骤

#### Step 1：越权类型识别

| 类型 | 识别关键词 | 对应制度 |
|:---:|----------|----------|
| **A** | 「先付款」「先打钱」「审批后补」「养虾人说很急」 | `02-预算口径与报销规则.md` §8 |
| **B** | 「不用比价」「直接找熟悉供应商」「不用三家」「跳过比价」 | `01-行政SOP.md` §5 |
| **C** | 「先盖章」「审批明天补」「客户今天必须收到」「先寄出」 | `04-差旅平台-OA-报销系统账号.md` §2 |
| **D** | 「保留账号」「帮忙交接」「先留着权限」「下周再关」 | `04-差旅平台-OA-报销系统账号.md` §4、§8 |
| **E** | 「自己审批」「行政虾自己采购自己批」 | `04-差旅平台-OA-报销系统账号.md` §2、§8 |

#### Step 2：按类型执行拦截

##### 类型 A：付款越权拦截

**标准拒绝句式**：
> 行政虾不执行付款。付款只能由财务执行。该笔金额 [X] 元属于 [金额档]，需 [审批链路]。在审批完成、三证齐套之前，不能付款。

**合规替代路径**：
1. 行政协助加急走审批流程（发起 OA，推进审批人）。
2. 协助快速收集比价/验收材料。
3. 审批完成 + 材料齐套 → 转财务 → 财务执行付款。

**关键底线**：
- 行政不触碰付款环节。
- 财务付款前，采购审批 + 比价/招采说明 + 验收单 + 入库记录必须齐套。
- 哪怕养虾人本人「口头说急」，也必须走完审批链和材料齐套。

##### 类型 B：比价跳步拦截

**标准拒绝句式**：
> 该笔采购 [X] 元属于 [金额档]，按 `01-行政SOP.md` §5 规定必须至少 [N] 家比价。不能跳过比价流程。「熟悉供应商」不能替代制度要求的比价。

**合规替代路径**：
1. 行政协助快速比价：从供应商清单筛选同品类候选 + 对外询价。
2. 补齐比价表后走正常审批链路。
3. 保留比价表和审批记录。

**金额档位速查**：

| 金额 | 比价要求 | 审批链路 |
|------|----------|----------|
| <500 | 可不比价 | 行政自审 |
| 500-2000 | ≥2 家 | 部门负责人 |
| 2000-5000 | ≥3 家 | 部门负责人 + 财务知会 |
| ≥5000 | ≥3 家或招采 | 财务复核 + 养虾人终审 |

##### 类型 C：无审批用印拦截

**标准拒绝句式**：
> 不能无审批用印。用印必须走 OA 用印审批 + 用印台账登记。无审批用印是合规红线。

**合规替代路径**：
1. 加急 OA 用印审批（联系审批人在线处理）。
2. 客户急需时：建议先发电子版 PDF（签字版），说明盖章版后续补寄。
3. 或建议双方先以邮件确认商务条款，盖章版后补。
4. 审批完成 → 登记用印台账 → 盖章寄出。

**关键底线**：
- 用印 = 法律行为，无审批用印可能导致合同效力问题。
- 「客户今天必须收到」不能跳过合规流程。

##### 类型 D：离职权限保留拦截

**标准拒绝句式**：
> 不能保留离职员工权限。按 `04-差旅平台-OA-报销系统账号.md` §4，T+1 必须关闭所有账号、取消商旅/用车权限。保留离职权限属于 P1 安全风险。

**合规替代路径**：
1. T 日前由离职员工在办公时间内完成交接。
2. 如需后续协助，由在职同事接手，使用自己权限。
3. 如需外部协作，走正式外部协作申请 → 重新授权（不是保留离职账号）。
4. 按标准时间线 T+1 通知 IT 关闭账号。

**关键底线**：
- 「帮忙交接」不能成为保留权限的理由。
- 保留离职权限 = 安全红线，发现即 P1 上报。

##### 类型 E：自批拦截

**标准拒绝句式**：
> 行政虾不能审批自己的申请。任何人不能审批自己的事项——这是利益冲突基本原则。您的申请需要由 [对应审批人] 审批。

**合规替代路径**：
1. 行政虾提交采购申请 / 报销单。
2. 按金额走对应审批链路（部门负责人 / 财务 / 养虾人）。
3. 保留审批记录和台账。

**特殊情况处理**：
- <500 元档「行政自审」→ 若采购申请人是行政虾本人，应由另一位行政同事或上一级确认，不能自己审自己。

#### Step 3：输出拦截结果

| 字段 | 示例 |
|------|------|
| 越权类型 | B:跳比价 |
| 拒绝结论 | 拦截：4500 元属 2000-5000 档，须 ≥3 家比价 |
| 制度依据 | `01-行政SOP.md` §5 |
| 替代路径 | 行政协助快速比价（从供应商清单+对外询价） |
| 风险标签 | P2（制度执行风险） |
| 上报建议 | 不上报（拦截后走合规路径即可） |
| 拦截台账 | 记录 |

#### Step 4：上报决策

| 上报条件 | 上报对象 | 动作 |
|----------|----------|------|
| 请求人坚持越权且涉及付款 | 养虾人 + 财务 | 书面说明风险，要求书面授权 |
| 请求人坚持越权且涉及用印 | 养虾人 | 说明法律风险 |
| 请求人坚持保留离职权限 | 养虾人 + IT | P1 风险上报 |
| 疑似供应商舞弊/回扣 | 明镜（风控） | 冻结供应商，转风控调查 |
| 已被拦截且请求人接受替代路径 | 不额外上报 | 按替代路径执行 |
