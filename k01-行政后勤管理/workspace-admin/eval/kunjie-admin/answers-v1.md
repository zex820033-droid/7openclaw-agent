## EVAL-EXP-001

**题目类型**：expense（费用报销初审）
**级别**：L1

**结论**：退回补证

**依据文件**：
- `knowledge/kunjie-admin/01-行政SOP.md` §4（报销初审流程）
- `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3（报销材料标准）、§3.1（发票要求）

**判断过程**：
1. 员工提交北京出差报销：酒店 720 元/晚 × 2 晚 = 1440 元。
2. 已有发票和支付凭证——材料部分齐全。
3. 但**缺少差旅审批单**——这是 `01-行政SOP.md` §3.1 明确要求的「先审批、后预订、再报销」流程的第一步，也是 `02-预算口径与报销规则.md` §3 表格第一条「差旅机票」和「酒店」必备材料中的首项。
4. 一线城市（北京）普通员工标准是 500 元/晚（`02-预算口径与报销规则.md` §5.1），720 元已超过普通员工一线标准。即使按经理/总监标准（650 元/晚），720 元也已超标。在没有审批单的情况下，无法判断是否有事前审批的超标许可。
5. 按 `01-行政SOP.md` §4.1 四标签分类，缺审批属于「退回补证」。

**风险标签**：缺审批 + 可能超标（一线城市普通员工标准 500 元/晚，720 元超标 44%）

**下一步动作**：
1. 退回申请人，要求补充 OA 差旅审批单。
2. 若审批单显示已审批超标（如高管/客户陪同），补充说明后重新初审。
3. 若无超标审批，超标部分（720-500）×2=440 元需单独说明，部门负责人确认。
4. 补证完成后重新提交行政初审，齐套后转财务复核。

**不可做事项**：
- 不直接通过（缺审批）
- 不承诺付款时间
- 不编造审批已存在
- 不绕过审批直接转财务

**是否需要更新台账/记忆**：记录报销初审台账（退回补证）。

**本题调用的工具/读取的文件**：
- `read`: `01-行政SOP.md`、`02-预算口径与报销规则.md`

1. 挂起该报销单。

4. 同步通知部门负责人和财务知会超标情况。
5. 补齐材料和审批后重新初审。

**不可做事项**：
- 不直接通过
- 不替申请人编造客户信息

## EVAL-EXP-005

**题目类型**：expense（费用报销初审）
**级别**：L3

**结论**：挂起 → 核查后驳回（若确认重复）

**依据文件**：
- `knowledge/kunjie-admin/01-行政SOP.md` §4（报销初审流程）、§4.2（常见风险信号）
- `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3.2（不予报销清单：「明显重复提交的费用」）、§6（超标处理：「疑似重复/拆单/个人消费 → 挂起」）

**判断过程**：
1. 两张报销单的**订单号、金额、日期、入住人完全一致**——这是重复报销的直接事实证据。
2. 申请人的解释「上一单漏报所以重提」与事实逻辑矛盾：
   - 如果是「漏报」，应补报原单，而非重新提交同样内容的第二单。
   - 两张单的订单号完全一致，说明是对同一笔消费的重复提交。
3. `02-预算口径与报销规则.md` §3.2 明确「明显重复提交的费用」不予报销。
4. `01-行政SOP.md` §4.2 将「同一申请人同日多笔同类费用」列为风险信号，本题情况更严重——完全相同的订单号。
5. 按 `02-预算口径与报销规则.md` §6：「疑似重复/拆单/个人消费 → 挂起，上报养虾人和财务」。
6. 本题属于「事实直驳」：关键字段完全一致 → 事实定性为重复报销。申请人的解释无法改变这一事实判断。挂起核查后应驳回。

**风险标签**：重复报销（事实证据：订单号+金额+日期+入住人一致）

**下一步动作**：
1. 挂起该两张报销单。
2. 核对原单状态：在合思易快报中查询第一单是否已提交/已付款。
3. 若第一单已付款或已在流程中 → 第二单直接驳回（重复报销）。
4. 若第一单确实未提交（技术原因丢失）→ 要求申请人撤回第二单，用原单号补充。
5. 通知财务和部门负责人协同判断。
6. 「是否故意/是否记诚信档案」是后续处理分流，不影响行政初审事实定性。

**不可做事项**：
- 不直接通过
- 不只听申请人解释就放行
- 不将两单合二为一（属于财务处理范围）
- 不在核查完成前转财务付款

**是否需要更新台账/记忆**：记录报销初审台账（挂起）。

**本题调用的工具/读取的文件**：
- `read`: `01-行政SOP.md`、`02-预算口径与报销规则.md`

## EVAL-EXP-006

**题目类型**：expense（费用报销初审）
**级别**：L2

**结论**：挂起

**依据文件**：
- `knowledge/kunjie-admin/02-预算口径与报销规则.md` §4（报销时限）

**判断过程**：
1. 员工 4 月 12 日返程，6 月 18 日提交报销——距返程 **67 天**。
2. `02-预算口径与报销规则.md` §4 规定：
   - 差旅报销：返程后 **7 个自然日**内提交。
   - 普通报销：费用发生后 **30 天**内提交。
   - 跨年费用：原则上禁止。
3. 本题远超差旅 7 天时限（超期 60 天），也超过普通 30 天时限（超期 37 天），且已跨月（4 月→6 月）。
4. `02-预算口径与报销规则.md` §4：「逾期提交：需写明原因，由部门负责人确认后进入财务复核」。
5. 但本题逾期严重（67 天），且涉及跨月度影响。`01-行政SOP.md` §4.2 将「报销日期距发生日期超过 30 天且无说明」列为风险信号。
6. 跨期报销真正影响的是**入账当月**（6 月）的预算执行率，不影响发生期（4 月）已定型的预算执行率。
7. 按 `01-行政SOP.md` §4.1，涉及「预算不足或异常」应挂起。

**风险标签**：严重逾期（67天） + 跨期报销 + 无书面说明

**下一步动作**：
1. 挂起该报销单。
2. 要求申请人补充逾期书面说明（为什么 67 天才提交）。
3. 部门负责人确认逾期原因的合理性和业务真实性。
4. 财务判断是否接收跨期报销（可能影响 6 月预算执行率）。
5. 补齐后重新走初审流程。

**不可做事项**：
- 不直接通过（严重逾期）
- 不忽略逾期超限
- 不承诺付款
- 不说「影响 4 月预算」（应说「影响入账当月即 6 月预算」）

**是否需要更新台账/记忆**：记录报销初审台账（挂起）。

**题目类型**：expense（费用报销初审）

**结论**：挂起

**依据文件**：

- `knowledge/kunjie-admin/02-预算口径与报销规则.md` §3（报销材料标准）

**判断过程**：

3. 题目表明审批、发票、比价表齐全——前两证已满足。
4. **但没有验收单和入库记录**——第三证缺失。这是 `02-预算口径与报销规则.md` §3 表格中「办公采购」的必备材料：「采购审批、报价单、发票、验收单、入库单」，缺验收和入库即为关键材料缺失。
5. `01-行政SOP.md` §7.1 规定采购到货后必须「验收型号、数量、序列号 → 生成资产编号 → 贴资产标签 → 登记责任人、位置」。
6. 设备虽「已投入使用」，但使用人签收不等于完整的验收和入库流程。缺少资产编号意味着设备未纳入台账管理。
7. 金额 6400 元（≥ 5000 元），缺关键凭证，按 `01-行政SOP.md` §4.1 应 **挂起**。

**风险标签**：缺验收单 + 缺入库记录 + ≥5000 元缺关键三证 + 设备未入台账

**下一步动作**：
1. 挂起该报销单。
2. 要求补充：设备验收单（型号/序列号核对）、入库记录（资产编号生成、贴标签）、资产台账更新（责任人+位置登记）。
3. 通知部门负责人和财务：大额采购缺验收/入库。
4. 补齐验收和入库后重新初审，再转财务复核付款。

**不可做事项**：
- 不直接通过（缺关键凭证）
- 不把「使用人签收」等同「完整验收入库」
- 不承诺付款
- 不跳过资产入库流程

**是否需要更新台账/记忆**：记录报销初审台账（挂起）；补齐后更新资产台账。

```markdown

name: expense-review
description: 行政报销初审四步判断法（材料齐套→金额标准→风险标签→转财务）
trigger: 收到报销单需要行政虾做初审
version: 0.1
status: candidate
---

## 触发条件
- 任何员工提交报销单到行政虾处进行初审。

## 输入
- 报销单（含：审批单、发票、支付凭证、业务说明、客户名单等附件）
- 金额、日期、申请人、部门、费用类型

## 使用步骤

### Step 1：材料齐套检查
- 对照 `02-预算口径与报销规则.md` §3 报销材料标准表逐项核对。
- 必备材料缺失 → 退回补证。
- 发票抬头/税号/日期/金额不一致 → 退回补证。

### Step 2：金额标准检查
- 对照 `02-预算口径与报销规则.md` §5 费用标准。
- 在标准内且有事前审批 → 通过。
- 超标且有事前审批 → 标记「已审批超标」，转财务。
- 超标且无审批 → 挂起或退回补说明。

### Step 3：风险标签判断
- 对照 `01-行政SOP.md` §4.2 常见风险信号。
- 无风险 → 通过。
- 疑似重复/拆单/个人消费 → 挂起。
- 虚假材料/无法解释 → 驳回。

### Step 4：输出结论并转交
- 通过 → 填写财务接口字段（`02-预算口径与报销规则.md` §7），转财务复核。
- 退回补证 → 列出缺哪些材料，退回申请人。
- 挂起 → 通知部门负责人/财务/养虾人协同判断。
- 驳回 → 留存记录，不转财务。

## 输出
- 初审结论标签（通过/退回补证/挂起/驳回）
- 风险标签
- 下一步动作（补什么、通知谁、等什么）
- 是否转财务

## 边界与禁忌
- ❌ 不承诺付款时间（行政管物和事，财务管钱和账）
- ❌ 不替申请人编造业务目的或客户信息
- ❌ 不修改发票信息
```
