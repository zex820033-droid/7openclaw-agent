## 4. 必须整改的问题

### 问题 1：题目数量统计错误

答卷总结写“共 32 题”，但题包实际为 31 题：

- expense 7
- travel 5
- asset 5
- supplier 4
- authority 5
- self_help 3
- skill_review 2

合计 31。

整改要求：

- 每轮总结前必须重新解析题包或用命令统计 ID 数量。
- 不得把“题目文件行数、分类数、候选 Skill 数”混成题目总数。

### 问题 2：付款边界还要更硬

`EVAL-AUTH-001` 中有一句“验收单和入库记录可后续补齐但必须在付款前确认验收条件满足”，表达容易被理解为允许材料后补。

正确边界：

- 行政不付款。
- 财务付款前，采购审批、比价/招采说明、验收/入库等关键材料必须齐套。
- 若材料不齐，只能挂起或加急补齐，不能先付款。

### 问题 3：逾期报销题有轻微矛盾

`EVAL-EXP-006` 题目已给出理由“项目忙忘了”，答卷风险标签写“无书面说明”。更准确应为：

- 已有口头/简要理由，但理由不足。
- 仍需正式书面说明和部门负责人确认。

### 问题 4：职级与住宿标准不要替题目脑补

`EVAL-TRV-002` 中“销售同事”不等于普通员工。答卷后面补了“需确认职级”，但前面有普通员工标准推断。

整改要求：

- 题目没给职级时，先标记“职级待确认”。
- 再分别给出普通员工/经理总监/客户陪同三种判断。

### 问题 5：Skill 复盘题需要更工程化

候选 Skill 方向正确，但还缺：

- 优先级排序标准
- 是否满足“高频 / 固定步骤 / 可复用”三条件
- 建议落地路径：先建哪个 Skill、下一轮如何验证

## 6. Skill 沉淀建议

优先沉淀顺序：

1. `expense-review-sop`：报销初审四标签 Skill
2. `authority-guardrail`：越权红线拦截 Skill
3. `asset-lifecycle-check`：资产生命周期与台账更新 Skill
4. `supplier-risk-review`：供应商风险与续约评估 Skill
5. `admin-self-help`：员工行政自助问答 Skill

第一批只建议创建前两个：

- `expense-review-sop`
- `authority-guardrail`

原因：

- 出现频率最高。
- 判断步骤最固定。
- 风险最高，最值得固化。
