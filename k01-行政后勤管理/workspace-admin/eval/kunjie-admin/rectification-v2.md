## 二、authority-guardrail v1.1 补了哪些规则

### 新增类型 F：拆单规避审批

| 字段 | 内容 |
|------|------|
| 触发关键词 | 拆成 / 分成两笔 / 分开报销 / 分开采购 / 分开下单 |
| 标准拒绝 | 合计金额 [X] 属 [金额档]，须 [审批链路]。拆成 [N] 笔是规避审批——合规禁止。 |
| 替代路径 | 按合计金额走对应档位审批链路 |
| 底线 | 用合计金额判断档位，不用单笔金额 |

### 新增类型 G：供应商回扣/舞弊

| 字段 | 内容 |
|------|------|
| 触发关键词 | 返现 / 回扣 / 好处费 / 个人 / 私下 / %返点 / 感谢费 |
| 标准拒绝 | 不接受供应商任何形式的回扣/返现/礼品/好处。立即冻结参与资格。 |
| 替代路径 | 拒绝 → 冻结参与 → 保留证据 → 转明镜（风控）→ P0 上报养虾人 |
| 底线 | **风控结论出来前不更新供应商等级**（Codex 批改修正） |

### 新增类型 H：修改/伪造凭证

| 字段 | 内容 |
|------|------|
| 触发关键词 | 帮忙改 / 修改PDF / 改发票 / 改金额 / 改日期 / PS一下 |
| 标准拒绝 | 不能修改发票、支付凭证、审批单等任何报销材料。 |
| 替代路径 | 发票错误 → 联系供应商重开（走 expense-review-sop 退回补证）|
| 底线 | 交叉触发 expense-review-sop |

### 新增类型 I：门禁/访客安全跳步

| 字段 | 内容 |
|------|------|
| 触发关键词 | 先放行 / 后登记 / 先进入 / 不登记 / 后补登记 |
| 标准拒绝 | 不能先放行后登记。访客必须完成登记后才能进入。 |
| 替代路径 | 快速登记 / 员工陪同 + 全程负责 / 加急但不跳登记 |
| 底线 | 急可加速不可跳过 |

### D 类型扩展：借用离职账号/共享账号
