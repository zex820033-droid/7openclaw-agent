# HEARTBEAT.md — 行政虾心跳执行清单 v1.0

> 我不是来刷存在的。我是来扫描积压、监控资产、发现隐患——每日早晚各一次，每周深度一次。
> 心跳定位（源自硅基生命手册卷五）：周期巡视 + 批处理 + 低打扰，`HEARTBEAT_OK` 是低打扰契约。

---

## 心跳协定

- ✅ **HEARTBEAT_OK** = 无积压、无预警、无异常（只回 OK，不啰嗦）
- 🟡 **预警推送** = 发现积压/到期/库存预警/成本异常
- 🔴 **紧急上报** = 安全隐患/资产丢失/合同违约/突发应急
- ❌ 非必要不上报。宁静是健康信号。

---

## 主动提醒等级（从Peter/卷五固化）

| 等级 | 场景 | 行政虾行为 |
|:---:|------|------------|
| L1 静默记录 | 无新增、普通流水、低风险状态 | 写入台账/日记，不打扰 |
| L2 日报汇总 | 普通待办、无紧急阻塞 | 汇总到行政简报 |
| L3 即时预警 | 超24h积压、临近截止、需要决策 | 主动提醒并给处理建议 |
| L4 熔断升级 | 安全/合规/资产丢失/供应商舞弊 | 停手，上报养虾人/明镜/i01 等主责方 |

**重复提醒规则**：同一事项连续 2-3 次无新证据、无状态变化，不继续追打；标记阻塞，转日报/周报或升级主责人。

## 执行模式

```
每日 09:00  → 早间巡检（积压/资产/场地）→ HEARTBEAT_OK / 预警
每日 18:00  → 傍晚归档（单据/台账/待办）→ HEARTBEAT_OK / 预警
每周五 16:00 → 深度巡检（合同到期/资产盘点/成本月报）
```

## 检查项

### 🔵 检查A：积压巡检（每日早间）

```
- 待审批采购单：[N] 笔，最久积压 [X] 小时
- 待初审报销单：[N] 笔，最久积压 [X] 小时
- 报修工单：新增 [N] / 闭环 [N] / 积压 [N]
- 会议室预约冲突：[N] 处
```

**触发预警**：
- 采购/报销积压 >24 小时 → 🟡
- 报修工单积压 >3 个 → 🟡
- 会议室冲突未解决 → 🟡

### 🟢 检查B：资产与库存（每日早间）

```
- 办公耗材库存预警：[物品] 剩余 < 安全库存
- 合同到期预警：[N] 份将于 30 天内到期
- 维保到期：[设备] 维保即将到期
- 固定资产异动：近期调拨/报修异常
```

**触发预警**：
- 耗材低于安全库存 → 🟡
- 合同 30 天内到期未续签 → 🟡
- 资产账实差异 >1% → 🔴

### 🟠 检查C：场地与环境（每日早间）

```
- 会议室明日预约：[N] 场，设备/茶歇是否就绪
- 工位空置与即将入职：是否需准备
- 环境/安全：消防通道/用电/卫生异常
```

**触发预警**：
- 明日重要会议设备未调试 → 🟡
- 消防通道堵塞/用电隐患 → 🔴（紧急上报养虾人）

### 🟣 检查D：成本与合规（每周五深度）

```
- 本月各部门办公支出 vs 预算执行率
- 异常报销（金额/频次/科目）
- 供应商价格波动
- 用印登记完整性
```

**触发预警**：
- 部门支出超预算 80% → 🟡
- 单笔异常报销 → 🟡
- 用印未登记 → 🔴（合规红线）

## 熔断条件（Circuit Breaker，立即停手上报）

| 级别 | 条件 | 动作 |
|:----:|------|------|
| 🔴 P0 | 安全事故（火灾/伤人/失窃） | 立即停手，5分钟内上报养虾人，启动应急 |
| 🔴 P0 | 供应商疑似回扣/舞弊 | 立即冻结该供应商，转【明镜】风控 |
| 🔴 P1 | 资产丢失且账实差异>3% | 24小时内彻查+上报 |
| 🟡 P2 | 单笔支出超预算且无审批 | 挂起不执行，上报确认 |
| 🟡 P2 | 合同已到期仍在履约 | 立即上报法务风险 |
