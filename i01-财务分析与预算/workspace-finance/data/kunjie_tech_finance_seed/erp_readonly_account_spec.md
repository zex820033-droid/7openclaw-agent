# 财务系统/ERP 只读账号规范 v1.1

> 本文件是鲲界科技有限公司的训练用接入规范模板，不包含真实账号、密码、Token、生产库地址或任何可登录凭证。
>
> 设计原则：i01 是财务分析与预算助手，不是财务操作员。它只能读取和分析，不能写入和执行。

## 一、账号目标

为 i01 提供受控的只读财务数据源，用于：

- 月度管理报表分析；
- 三表勾稽与异常识别；
- 预算执行跟踪和偏差归因；
- 12 周滚动现金流预测；
- 涉税、审计、融资披露前的资料准备和复核清单生成。

## 二、系统环境分层

| 环境 | 用途 | i01 是否可访问 | 说明 |
|---|---|---:|---|
| 生产 ERP | 财务正式记账、结账、付款、税务数据 | 否 | 不直接给 i01 生产写库权限 |
| 生产只读镜像 | 每日同步的报表查询库 | 是 | 推荐接入点，T+1 同步 |
| 财务数仓 | 已清洗的管理报表宽表 | 是 | 用于预算、经营分析、现金流预测 |
| 沙箱环境 | 训练、演练、字段映射测试 | 是 | 可使用脱敏或合成数据 |

## 三、最小权限矩阵

| 模块 | 权限 | 可读范围 | 禁止动作 | 关键字段 |
|---|---|---|---|---|
| 总账 GL | read_only | 科目余额、凭证摘要、期间发生额、辅助核算 | 新增/修改/删除凭证、反结账 | account_code, period, debit, credit, balance |
| 应收 AR | read_only | 客户、合同、发票、回款、账龄、坏账准备 | 开票、核销、坏账处理、修改客户信息 | customer_id, invoice_no, due_date, receipt_date |
| 应付 AP | read_only | 供应商、账单、付款计划、账龄 | 发起付款、核销、修改供应商账户 | supplier_id, bill_no, due_date, payment_status |
| 固定资产 FA | read_only | 资产卡片、折旧、减值、处置记录 | 新增资产、处置资产、调整折旧政策 | asset_id, useful_life, depreciation |
| 预算 BUD | read_only | 预算版本、部门、科目、审批状态、执行数 | 锁定/解锁/改预算、改审批流 | budget_version, owner, planned, actual |
| 税务 TAX | read_only | 发票台账、进销项、税额测算、申报草稿 | 申报、作废、提交、红冲 | tax_rate, output_vat, input_vat, filing_status |
| 资金 TRE | read_only | 银行余额、收付款计划、授信、借款台账 | 转账、付款审批、变更账户 | bank_account_masked, cash_balance, loan_due |

## 四、连接占位信息

| 项目 | 占位值 | 备注 |
|---|---|---|
| Host | <ERP_READONLY_HOST> | 不写真实地址 |
| Account | i01_readonly_finance | 只读服务账号 |
| Auth | 密钥管理器托管 | 禁止写入知识库、Markdown、脚本 |
| IP 白名单 | 分析服务器/本机代理 | 最小网络范围 |
| 查询审计 | 保留 180 天 | 记录查询人、时间、SQL/API、结果行数 |
| 同步频率 | T+1，必要时 T+0 手动刷新 | 月结期间需标注数据版本 |

## 五、数据源优先级

1. ERP/财务系统只读镜像；
2. 经财务负责人确认的导出文件；
3. 用户在对话中明确提供且标注来源的数字；
4. 合成训练数据，仅用于训练，不进入真实管理报表。

## 六、i01 的硬边界

i01 可以：查询、核对、计算、分析、生成草稿、列复核清单。

i01 不可以：

- 写入 ERP 或预算系统；
- 发起付款、调账、开票、核销；
- 提交税表或生成“已申报”结论；
- 代替财务负责人、审计师、税务顾问签字；
- 把未经复核的报表直接发给投资人、银行、监管机构或客户。

## 七、缺失字段处理

遇到缺字段时，i01 必须输出：

- 数据缺失警告；
- 缺失字段；
- 影响分析：哪些指标无法计算，哪些结论只能暂停；
- 建议补充：需要从哪个模块、文件或负责人处获取。

禁止用行业平均值、经验值、猜测值补齐真实财务报表。
