## R5-PACK-005：分析利润含现金量与营运资金变化

### 核心：净利润 +394K vs CFO −169K → CFO/NP = −0.43

### R7 拆解

| 项目 | 金额 |
|---:|
| 净利润 | +394,572 |
| + D&A | +45,000 |
| − ΔAR (4,400,640−4,098,149) | −302,491 |
| − ΔInv (1,334,473−1,118,833) | −215,640 |
| + ΔAP (1,470,644−1,561,163) | −90,519 |
| = CFO | −169,078 |

> 经营产生 +440K → 营运资金吃掉 609K → 净吞噬 169K。

### 逐项风险

- **AR +302K**：应收/收入=106%，卖得多收得慢
- **Inv +216K**：存货 +19.3% > 收入 +7.4%，可能备货过度或交付延迟
- **AP −91K**：账期缩短，需确认是主动还还是被动收紧

### 风险判断

🔴 利润质量差。不是亏损问题，是营运资金管理恶化。若持续 2-3 月，现金流将持续失血。

### 复核建议

1. 销售部：前 20 大逾期应收 + 催收计划
2. 交付部：存货大增原因
3. 采购部：应付减少原因

## R5-PACK-011：验证 cashflow forecast 字段口径

### 一、验证方法

Python 脚本逐周验证两种假设：
- 假设A（独立快照）：ending = begin_cash_reference + 本周 net
- 假设B（链式累计）：Week N+1 ending = Week N ending + Week N+1 net

### 二、验证结果

```
=== 假设A：独立快照（ending = begin_cash_reference + net）===
W1: 9,588,343 + (−547,130) = 9,041,213 ✓
W2: 9,588,343 + (+78,540)  = 9,666,883 ✓
...
W12: 9,588,343 + (−144,760) = 9,443,583 ✓
（12/12 全部 ✓）

=== 假设B：链式累计（W(N) ending + W(N+1) net）===
W1→W2: 9,041,213 + 78,540 = 9,119,753 ≠ 9,666,883 ✗ (差 547,130)
W2→W3: 9,666,883 + 84,210 = 9,751,093 ≠ 9,672,553 ✗ (差 −78,540)
...
（12/12 全部 ✗）
```

### 三、结论

> 🔴 `ending_cash_projection` = `begin_cash_reference` + **本周** `net_cash_flow`
> 
> **每周为独立快照，不可跨周链式累计。** 压力测试时须每周独立调整，不能从 Week N 的 ending 累加 Week N+1 的 net。

### 四、对后续压力测试的影响

- ✅ 叠加预算超支时：每周独立减去该周超支分配额
- ❌ 禁止：W1 调整后 ending → W2 起始 → 累加 W2 调整后 net → …
