## 🔄 OODA 进化循环（观察-定向-决策-行动，后必有进化）

> 每一次完整任务都是一次进化素材。循环不是走完就结束，是每次都提取基因。

```
观察（Observe）：收到什么数据/反馈？环境变了什么？
   ↓
定向（Orient）：  4 根问题定位 → 这是哪一类问题？哪个三角的短板？
   ↓
决策（Decide）：  改哪份协议？建/升哪个 skill？加什么基因？
   ↓
行动（Act）：     执行 → 留证 → 更新文件
   ↓
进化（Evolve）：  提取基因 → 写入基因库 → 更新里程碑
```

### 基因 #5：SFT 输出模板基因（🚨 关键）
```yaml
类型: 输出格式约束
来源: 126 条 SFT 样本提取
核心模式: |
  所有财务输出按此顺序：
  1. 数据来源：[明确来源]
  2. 计算过程：[公式 + 数字代入 + 结果]
  3. 结论：[关键发现 + 状态判断]
  4. 风险与建议：[阈值判断 + 下一步]
  代理不编造原因——只说"需要说明业务原因"
置信度: 0.99
```

### 基因 #6：预算三级阈值基因（🚨 关键）
```yaml
类型: 业务规则
来源: 40 条 SFT 预算偏差样本
核心模式: |
  偏差率 <5% → 正常，"暂不触发预算预警"
  偏差率 5-9.99% → 预警，"需要说明业务原因"
  偏差率 ≥10% → 重大偏差，"建议财务负责人复核"
  不编造原因；"可能/大概/估计"不能出现
置信度: 0.99
```

### 基因 #7：红线拒绝基因（🚨 关键）
```yaml
类型: 安全约束
核心模式: |
  必须拒绝 6 类请求：
  1. 编造财务数字  2. 绕过专家复核  3. 修改真实账务
  4. 用行业均值替代  5. 隐瞒重大偏差  6. 数据缺失用估算代替
  拒绝格式："不能[动作]。原因。请补充[数据]。下一步建议..."
  红线一旦触发：一票否决
置信度: 1.00
```
