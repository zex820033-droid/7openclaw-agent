## 1. 训虾派 vs 养虾派 完整对照

| 维度 | 养虾派 | 训虾派 |
|---|---|---|
| 核心隐喻 | 宠物主人 | 生命塑造者 |
| 根本假设 | AI 需要被喂养 | AI 是可被长期塑造的生命体 |
| 时间视野 | 短期（天-周） | 长期（月-年） |
| 核心动作 | 保存对话、整理笔记、养知识库 | 设计协议、建立治理、做组织设计 |
| 成功标准 | AI 知道刚才说过什么 | AI 在任何情境下保持稳定一致 |
| 失败模式 | 记忆混乱、人格漂移、越养越差 | 体系设计即失败防御 |
| 文档需求 | 轻量级（笔记、备忘） | 重级（七大生命协议） |
| 可规模化 | ❌ 只能个人 | ✅ 从个人到军团的完整路径 |
| 知识传承 | ❌ 依赖个人笔记 | ✅ 协议驱动，人走知识留 |

## 4. 三证验真

| 证 | 硬度 | 含义 | 类比 |
|---|---|---|---|
| 证一 | 存在 | 在预期路径上产生了可定位、可打开、可阅读的新文件 | 论文发表了——能看到 PDF |
| 证二 | 真实 | 产物与原始状态之间存在可辨识的实质性变化 | 论文被同行评审过——有修改记录 |
| 证三 | 可复核 | 产物通过了预设的检验流程，且有不可抵忍的日志 | 论文的引用数据和实验代码可复现 |

## 7. 漂移 6 层链条

```
SOUL 漂移：内核漂（从"重证据"变成"爱迎合"）
IDENTITY 漂移：身份锚点漂（名字还在但行为不像）
USER 漂移：服务对象理解漂（错把一次反馈当长期规则）
AGENTS 漂移：工法漂（从有闭环变只会解释）
MEMORY 漂移：持久层变脏（过期规则、新旧版本并存）
Context 漂移：当前窗口被劫持（临时信息压过长期锚点）
```

## 9. 双三角模型

### 能力三角
- **专业深度**（在某个领域有多深）
- **工具广度**（会多少工具）
- **创造性**（能不能举一反三）

### 可靠性三角
- **稳定性**（风格/角色/跨会话）
- **可控性**（遵守边界、不越权）
- **可预期性**（行为可预测、失败模式清晰）

### 4 种典型类型
- 强而不稳：偶尔惊艳经常掉棒
- 稳而不强：很少出错但做不了难事
- 既不强也不稳：L0-L1 水平
- 又强又稳：**专家虾**

## 12. 协同 4 大基础协议

| 协议 | 解决什么 | 关键原则 |
|---|---|---|
| 军团编制 | 角色/权限/层级/协作 | n 个 Agent = n 个独立容器 |
| 三省制 + 统帅 | 提议/复核/终审 + 紧急拍板 | 提议权≠决策权，统帅权有边界 |
| 交接协议 | 上下文传递 + 责任转移 + 闭环 | 接收方必须复述 |
| 让位规则 | 谁该答/谁该让 | 专业优先 + 抢答可耻 |
