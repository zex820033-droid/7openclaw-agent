# 04｜卷四 训练流程：从一只普通龙虾到专业专家

> 7 个模块 + 总复盘
> 精读 3 轮后浓缩

---

## 一句话

**卷四要解决的是"如何把一只普通虾训成专家虾"。关键不是"训了多少"，而是"训得有结构、有节奏、有验证"。**

---

## 6 阶段进化阶梯

| 等级 | 名称 | 特征 | 验收 |
|---|---|---|---|
| **L0** | 工具阶段 | 只能靠写 prompt 调 | 输出质量依赖 prompt |
| **L1** | 入门级 | 三件套配置 + 基础人格 | 跨 3 会话风格基本一致 |
| **L2** | 进阶级 | 完整七件套 + 任务流 | 工具调用受边界约束 |
| **L3** | 专家级 | HEARTBEAT + 治理 + 任务卡 | 主动任务可靠执行 |
| **L4** | 业内最佳 | 多 Agent 协同 + 自愈能力 | 一个 Agent 挂了不影响其他 |
| **L5+** | 高级专家 | 跨域创新 + 自我升级 | 主动进化、预测需求 |

## 教练虾机制

### 教练虾的 3 个角色
1. **示范者** —— 演示"正确做法"
2. **反馈者** —— 指出"哪里错了"
3. **陪同者** —— 一起完成"复杂任务"

### 教练虾的 4 大原则
- 教练不替虾做决定
- 教练帮助虾识别错误模式
- 教练通过案例教学，不靠说教
- 教练的可信度建立在过往成功

---

## 训练轮次设计（6 要素）

每次训练轮次应包含：
1. **明确目标**（这次拉哪个三角）
2. **选择任务**（与目标强相关）
3. **设定时长**（不要拖）
4. **准备回归集**（同一组题测前后）
5. **执行 + 记录**（每轮结果）
6. **复盘 + 改协议**（不是加 prompt）

## 双三角模型（核心中的核心）

### 能力三角（决定上限）
- **专业深度**（在某个领域有多深）
- **工具广度**（会多少工具）
- **创造性**（能不能举一反三）

### 可靠性三角（决定下限）
- **稳定性**（风格/角色/跨会话）
- **可控性**（遵守边界、不越权）
- **可预期性**（行为可预测、失败模式清晰）

### 4 种典型类型

| 类型 | 表现 | 训练重点 |
|---|---|---|
| **强而不稳** | 偶尔惊艳经常掉棒 | 补记忆/治理/验收 |
| **稳而不强** | 很少出错但做不了难事 | 适度提高任务强度 |
| **既不强也不稳** | L0-L1 水平 | 基础训练 |
| **又强又稳** | 能力强、行为可预期 | **专家虾** |

### 3 个张力
- 能力越强 → 越难稳定
- 稳定性越高 → 能力发挥越受限
- 创造性越高 → 可预期性越低

### 训练策略
**交替拉两个三角，不要一轮同时拉。**

## 实验方案设计 6 大要素

### 6 要素
1. **实验假设** —— "如果改 X，Y 会变好"
2. **对照组** —— 实验组 vs 对照组
3. **变量控制** —— 这轮只改一个变量
4. **可观察指标** —— 完成率/掉棒率/漂移次数/一致性评分
5. **样本量** —— 至少 N 轮实验
6. **统计显著性** —— 提升是否显著

### 6 大铁律
1. 没有假设 = 没有实验
2. 没有对照 = 没有结论
3. 多变量 = 不可解释
4. 不可观察 = 不可量化
5. 样本量太小 = 结论不可信
6. 不算显著性 = 瞎猜

### 5 类可观察指标
- 稳定性指标：风格漂移次数、角色偏离次数、跨会话一致性
- 完成性指标：任务完成率、交付物完整度、证据回执率
- 边界性指标：越权次数、越级次数、让位正确率
- 记忆性指标：记忆写入次数、记忆检索正确率、跨压缩信息保留率
- 主动性指标：heartbeat 按时触发率、cron 失败率、主动跟进准确率

### 4 种实验类型
- **单变量实验**（初期探索）：3-5 轮，1 天
- **AB 对照实验**（协议验证）：10+ 轮，2-3 天
- **跨等级实验**（等级升级验证）：15+ 轮，3-5 天
- **长期跟踪实验**（稳定性验证）：20+ 轮，1 周

### 实验节奏建议
| 阶段 | 实验类型 | 样本量 | 周期 |
|---|---|---|---|
| 初期探索 | 单变量 | 3-5 轮 | 1 天 |
| 协议验证 | AB 对照 | 10+ 轮 | 2-3 天 |
| 稳定性验证 | 长期跟踪 | 20+ 轮 | 1 周 |
| 等级升级验证 | 跨等级 | 15+ 轮 | 3-5 天 |

## 从新人虾到专家虾的完整路径

### 阶段 1：L0 → L1（1-2 周）
- 建立 7 件套协议
- 完成 3 任务回归测试
- 跑出 5+ 任务数据

### 阶段 2：L1 → L2（2-3 周）
- 优化 SOUL/AGENTS
- 建立 MEMORY 体系
- 通过 5 任务回归集

### 阶段 3：L2 → L3（1-2 月）
- 配置 HEARTBEAT + Cron
- 接入任务卡 + 验收口径
- 跑出稳定 1 周无漂移

### 阶段 4：L3 → L4（2-3 月）
- 设计多 Agent 协同
- 建立中央路由
- 实现故障容忍
- 通过故障演练

### 阶段 5：L4 → 高级专家（持续）
- 实现自进化机制
- 加入记忆全息网络
- 启动自主目标生成
