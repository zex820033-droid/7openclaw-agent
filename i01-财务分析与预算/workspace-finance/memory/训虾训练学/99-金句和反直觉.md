## 训虾派 vs 养虾派 核心对比

| 养虾派 | 训虾派 |
|---|---|
| 保存对话 | 写协议 |
| 整理笔记 | 注入 MEMORY.md |
| 订阅工具 | 设计 TOOLS.md 白名单 |
| 给 AI"介绍背景" | 用 IDENTITY 锚定 |
| "这次加一句 prompt" | "这次该改哪份文件？" |
| 出了问题 → 加约束 | 出了问题 → 改协议 |
| 一切都堆在一个 Agent | 一个 Agent 一个角色 |

## 5 阶段配置顺序（核心中的核心）

1. **USER.md**（第 1 份建立）—— 服务对象模型
2. **SOUL.md**（第 2 份）—— 人格内核
3. **AGENTS.md**（第 3 份）—— 工法纪律
4. **TOOLS.md**（第 4 份）—— 工具白名单
5. **IDENTITY.md**（第 5 份）—— 身份锚点
6. **MEMORY 体系**（第 6 份）—— 跨会话记忆
7. **HEARTBEAT.md**（第 7 份）—— 主动节律

**顺序错了 SOUL.md 经常要重写。**

## 4 根问题递进关系

```
Q1 一致性（卷一+卷二+卷三+卷四）
   ↓ 解决前 Q2 是空楼
Q2 持续性（卷二 MEMORY + 卷五）
   ↓ 解决前 Q3 放大断裂
Q3 规模化（卷三 Runtime + 卷六 + 卷七）
   ↓ 解决前 Q4 沉淀混乱
Q4 资产化（卷八）
```

**Q4 的设计思维必须从 Q1 就开始贯穿。**

## 漂移 6 层（最常用诊断）

```
SOUL → IDENTITY → USER → AGENTS → MEMORY → Context
```

**修复分级**：
- L1 轻微波动 → 微调文档
- L2 角色变形 → 清 memory + 回归测试
- L3 内核偏离 → 回滚版本 + 重新训练

## 三证硬度（最常用交付）

- 证一：产物存在
- 证二：实质性变化
- 证三：可复核

**三证的核心不是数量——是硬度。**

## 5 种养虾派失败（自我警示）

- 控制幻觉：以为 AI 越来越懂你
- 笔记腐坏：几十页笔记但 AI 从不读
- 工具茧房：100 个工具深度使用不到 2 个
- 角色缺口：所有任务堆在一个 Agent
- 周期陷阱：prompt 越加越长

---

## 双三角 3 个张力

- 能力越强 → 越难稳定
- 稳定性越高 → 能力发挥越受限
- 创造性越高 → 可预期性越低

**交替拉两个三角，不要一轮同时拉。**

---

## 6 道铁律

1. 未写入磁盘 = 未记住
2. 失败时改协议不加 prompt
3. 三证齐全 = 完成
4. 白名单 > 黑名单
5. 先收敛再丰富
6. 诊断先定层再动手
