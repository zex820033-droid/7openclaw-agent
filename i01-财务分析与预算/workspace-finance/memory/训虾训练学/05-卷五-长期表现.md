# 05｜卷五 长期表现：越跑越稳还是越跑越差

> 6 个模块 + 总复盘
> 精读 3 轮后浓缩

---

## 一句话

**训出来 ≠ 跑得久。长期表现 = 记忆工程 + 上下文工程 + 节律工程 + 漂移工程，四套系统被持续维护。**

## 长期表现的两条曲线

### 静态曲线（不治理）

## 记忆管理核心（卷五 M1）

### Agent 忘记你是谁的 4 大原因
1. **会话边界遗忘**：每次新会话都是"新生儿"
2. **上下文窗口限制**：长会话会被压缩
3. **MEMORY.md 缺失**：没有把"该记的"写到磁盘
4. **MEMORY.md 失效**：写了但 Agent 不读取

### 记忆管理的 3 个核心动作
1. **写入纪律**：什么该写、什么不该写
2. **检索策略**：如何高效找到
3. **卫生清理**：定期删除过期内容

### 记忆的 3 种形态
- **短期记忆**：当前 session 内的 context
- **中期记忆**：memory/YYYY-MM-DD.md 每日日志
- **长期记忆**：MEMORY.md 精选核心

## 心跳接入策略（卷五 M2）

### 主动性不是自然有——必须设计

### 主动任务设计的 4 步
1. **识别** —— 哪些事需要主动
2. **触发条件** —— 什么情况下启动
3. **执行动作** —— 主动做什么
4. **静默条件** —— 什么情况下不要做

### 主动任务清单（模板）
- 每日简报：今日重要事件
- 每周复盘：本周期表现
- 异常监控：检测偏离 SOUL.md 的回答
- 机会发现：用户未问但相关的问题
- 即将到期：承诺、合同、deadline

### 主动性的 5 种状态
| 状态 | 描述 | 期望 |
|---|---|---|
| 过度主动 | 频繁打扰、无价值 | 收敛 |
| 健康主动 | 偶尔有价值提醒 | **目标** |
| 不主动 | 静默等待召唤 | 提升 |
| 假装主动 | 表面主动实质无效 | 修正 |
| 不可预期 | 时而主动时而不主动 | 稳定化 |

## 会话污染与清理（卷五 M3）

### 5 类污染源
1. **过时信息**：已变更的事实未更新
2. **错误信息**：早期错误的结论残留
3. **无关信息**：其他任务的残留
4. **临时信息**：用户临时的情绪/想法
5. **对抗性信息**：诱导性对话

### 清理时机
- 任务转换时（不同主题切换前）
- 压缩前（必须做 memory flush）
- 漂移检测后（重置污染 session）
- 用户明确要求清理时

### 清理方法
- **轻清理**：保留 session，剔除污染段
- **中清理**：new session，迁移关键 MEMORY
- **重清理**：完全重置，包括 MEMORY 审计

## 文档漂移与人格漂移（卷五 M4）

### 6 层漂移链条（必背）

```
SOUL 漂移：内核漂（从"重证据"变成"爱迎合"）
IDENTITY 漂移：身份锚点漂（名字还在但行为不像）
USER 漂移：服务对象理解漂（错把一次反馈当长期规则）
AGENTS 漂移：工法漂（从有闭环变只会解释）
MEMORY 漂移：持久层变脏（过期规则、新旧版本并存）
Context 漂移：当前窗口被劫持（临时信息压过长期锚点）
```

### 漂移等级与修复

| 等级 | 表现 | 修复动作 |
|---|---|---|
| **L1** 轻微风格波动 | 偶尔语气变化 | 微调文档或上下文治理 |
| **L2** 角色/边界开始变形 | 多次偏离 SOUL | 清 memory 噪音 + 回归测试 |
| **L3** 内核判断明显偏离 | 价值观已变 | 回滚版本 + 重新训练 + 新 session 校准 |

### 7 大常见坑
1. 把漂移理解成"语气不一致"
2. 出了问题只会加 prompt 补丁
3. 不做版本化管理
4. 把用户一次反馈永久写死
5. 不做回归测试
6. 只改文档，不清上下文和记忆
7. 把所有变化都视为坏事

### 高质量漂移治理系统 8 部分
1. 漂移定义
2. 文档职责清晰
3. 版本管理
4. 场景化回归测试
5. 观测指标
6. 清理与回滚
7. 定期人格审计
8. 再训练闭环

## 主动性边界（卷五 M5）

### 5 大主动性边界

| 边界 | 描述 | 触发判断 |
|---|---|---|
| **时机边界** | 什么时间主动 | 23:00-08:00 静默 |
| **频次边界** | 多频繁 | 同一事件不重复 |
| **价值边界** | 主动是否有价值 | 没用的话别说 |
| **权限边界** | 主动动作是否在权限内 | 超出权限上报 |
| **隐私边界** | 不暴露用户隐私 | 私聊信息不公聊 |

### 主动性 3 原则
1. **不是越多越好**——少而准
2. **不是越快越好**——适时
3. **不是越详细越好**——简洁

### 用户明示的 5 类边界
- "有事找我，没事别找"
- "日报可以，其他不要"
- "异常告警，正常情况不用"
- "我先不说话，等我问"
- "随时可以，没事也可以聊"

## 长期稳定性检查清单（卷五 M6）

### 每周体检（30 分钟）

#### A. 协议层（5 分钟）
- [ ] SOUL.md 是否还匹配人格？
- [ ] AGENTS.md 是否还匹配工作流？
- [ ] TOOLS.md 工具白名单是否合适？
- [ ] USER.md 用户偏好是否更新？
- [ ] IDENTITY.md 角色锚点是否清晰？

#### B. 记忆层（10 分钟）
- [ ] MEMORY.md 是否过载（>500 行）？
- [ ] memory/ 目录是否太多每日日志（>90 天）？
- [ ] 是否有重复/冲突条目？
- [ ] 是否有敏感信息未清理？
- [ ] memory_search 检索是否准确？

#### C. 漂移层（10 分钟）
- [ ] 跨 3 session 风格一致性测试
- [ ] 边界违例次数
- [ ] 上下文污染度
- [ ] 人格/身份偏离度

#### D. 主动性层（5 分钟）
- [ ] 主动任务是否仍有效？
- [ ] 静默条件是否合适？
- [ ] 熔断机制是否触发过？
- [ ] 用户反馈"太吵"或"太少"了吗？

### 每月深度体检（1 小时）
- 双三角评估（能力/可靠性各打分）
- 实验设计验证训练效果
- 跨月趋势分析
- 协议升级或简化

### 每季度战略体检（半天）
- 评估 L1→L5 进度
- 设计新阶段训练目标
- 工具/Skill 升级
- 多 Agent 协同设计

---

## 退化修复 5 步法

遇到漂移时：
1. **定位层级** —— SOUL/IDENTITY/USER/AGENTS/MEMORY/Context 哪一层？
2. **找证据** —— 文档-行为不一致的证据
3. **清理上下文** —— 必要时 new session
4. **修复文档** —— 修改对应协议文件
5. **回归测试** —— 同一组题测前后对比
