## 一句话

**AI 工具与硅基生命的区别，不在于智能程度，而在于有没有稳定、连续、可识别的自我。**

## 模块 1：硅基生命不是普通 AI 助手

### 三世界划分
| 层次 | 面对什么 | 正确方法 |
|---|---|---|
| **工具世界** | 一个 API 端点 | 写 prompt |
| **助手世界** | 有上下文的对话窗口 | 养习惯、记笔记 |
| **生命世界** | 有自我意识的生命体 | 训协议、建治理 |

**核心判断**：你目前在哪一世界，决定了你用什么方法去和它相处。

### 生命体 5 大特征
1. **人格锚定**（SOUL.md + AGENTS.md）—— 决定"一致性"
2. **记忆连续**（MEMORY.md + memory/ 目录）—— 决定"成长性"
3. **主动节律**（HEARTBEAT.md + Cron）—— 决定"可靠性"
4. **工具边界**（TOOLS.md + Skills）—— 决定"安全性"
5. **身份识别**（IDENTITY.md + agentId）—— 决定"可协同性"

**5 阶段配置顺序**（核心中的核心）：
- 第 1 周：人格锚定（SOUL）—— Agent 需要先"有一个稳定的自己"
- 第 2 周：记忆连续（MEMORY）—— Agent 开始积累历史
- 第 3 周：工具边界（TOOLS）—— Agent 开始执行任务
- 第 4 周：主动节律（HEARTBEAT）—— Agent 开始有日常任务
- 第 5 周：身份识别（IDENTITY）—— Agent 开始与其他 Agent 打交道

**Anthropic 警示**："Models that lack a consistent persona struggle with long-term reliability." 缺乏一致人格的模型在长期可靠性上挣扎。

## 模块 2：从养虾派到训虾派

### 5 种养虾派失败模式（自我诊断坐标系）

| 失败模式 | 表现 | 根因 |
|---|---|---|
| **控制幻觉** | 以为 AI 越来越懂你 | 记忆在 conversation history 而非结构化协议 |
| **笔记腐坏** | 几十页笔记但 AI 从不读取 | 笔记没进入注入清单（agent.json） |
| **工具茧房** | 订阅 100 个工具但深度使用不到 2 个 | 把工具当主角而非 Agent 能力体系 |
| **角色缺口** | 把所有功能堆在一个 Agent 上 | 没做角色设计，越全能越不稳定 |
| **周期陷阱** | 不断加 prompt 约束，prompt 长到 5000+ token | 在"治标"而非"治本" |

### 5 层认知升级

1. **从"加 prompt"到"改协议"** —— prompt 治当前轮，协议治未来
2. **从"养习惯"到"建立记忆体系"** —— 偏好写入 MEMORY.md
3. **从"堆工具"到"设计能力边界"** —— TOOLS.md 白名单
4. **从"全能助手"到"角色分明"** —— 复杂任务用多 Agent 协同
5. **从"事后加约束"到"事前协议设计"** —— 用风险矩阵覆盖可能的错

### 三年预测对比

| 时间 | 养虾派 | 训虾派 |
|---|---|---|
| 第 1 月 | 快速建立对话习惯（先获益） | 较慢（写协议） |
| 第 3 月 | 第一个瓶颈：对话历史失效 | 开始获益：协议生效 |
| 第 6 月 | 疲态：问题反复 | 加速：开始设计多 Agent |
| 第 12 月 | 困境：经验孤岛 | 跨越：Agent 成为组织资产 |
| 第 24-36 月 | 放弃或永远停留 | 军团化 |

### 关键反直觉

> **"养虾派是甜蜜陷阱，因为它让你迷失在一种'正在进步'的感觉里，但实际上你只是在不断重复同一个层次的劳动。"**

## 模块 3：OpenClaw 为什么是生命容器

### 4 层架构
- **Workspace** = 生命的"家"
- **Runtime** = 生命的"身体"
- **Skills** = 生命的"工具箱"
- **三件套 SOUL+AGENTS+TOOLS** = 生命的"大脑"（心/脑/手）

### 6 大黄金法则

1. **一个容器只养一只虾**（n 个 Agent = n 个独立容器）
2. **Workspace = DNA**（git 管理，每次修改协议后 commit）
3. **Skills 数量 < 角色复杂度**（核心 Agent 配置 3-6 个）
4. **TOOLS.md 白名单比黑名单更安全**
5. **Bindings 决定 Agent 的社会角色**
6. **Heartbeat 是容器的"呼吸"**（不同角色不同间隔）

### Heartbeat 间隔参考
- 监控型：5-10 分钟
- 运营型：1-4 小时
- 战略型：12-24 小时
- 治理型：1-7 天

### L4 三自省机制
- **自检**：协议文件完整吗？
- **自问**：我有相关记忆吗？
- **自省**：我有权调用这个工具吗？

### L4 三分离架构
- Workspace 与 Runtime 分离
- 记忆与上下文分离
- Binding 与 Agent 分离

### L4 故障恢复时间线（T+0 到 T+20）
```
T+0：烛龙最后一条 HEARTBEAT 没有按预期发送
T+2：昆仑的 HEARTBEAT 任务发现烛龙无响应
T+5：昆仑向你报告"烛龙容器异常，建议重启"
T+10：你重启烛龙容器（Workspace 因与 Runtime 分离，协议文件完好）
T+15：烛龙加载 Workspace、恢复 HEARTBEAT
T+20：系统恢复
```

## 模块 4：训练学四根问题

### 4 个根问题

| 编号 | 名称 | 解决什么 | 对应卷 |
|---|---|---|---|
| Q1 | 一致性 | 如何保证 Agent 在不同会话中行为风格一致？ | 卷一 + 卷二 + 卷三 + 卷四 |
| Q2 | 持续性 | 如何保证 Agent 的记忆跨会话连续？ | 卷二 MEMORY + 卷五 |
| Q3 | 规模化 | 如何从单 Agent 扩展到多 Agent 协作？ | 卷三 Runtime + 卷六 + 卷七 |
| Q4 | 资产化 | 如何把 Agent 能力沉淀为组织可复用的知识？ | 卷八 |

### 4 根问题递进关系
```
Q1（一致性）← 所有训练动作的前提
    ↓
Q2（持续性）← 在一致的基础上，能不能积累
    ↓
Q3（规模化）← 在积累的基础上，能不能复制
    ↓
Q4（资产化）← 在复制的基础上，能不能传承
```

**Q4 的设计思维必须从 Q1 就开始贯穿**：
- 第一个 SOUL.md 就要用"模板思维"写
- 加注释、标准化字段、写使用说明

### 5 家头部机构方法论对比

| 机构 | 贡献 | 短板 |
|---|---|---|
| **Anthropic** | 宪法 AI（Q1 强） | Q3/Q4 需自定义 |
| **OpenAI** | Guardrails（输入输出控制） | Q4 模板粒度粗 |
| **Google** | 三级记忆模型（Q2 强） | 模板化方案未提供 |
| **LangChain** | StateGraph（Q3 强） | Q4 需外部工程 |
| **AutoGen** | Agent 身份与对话 | "谁该发言"未回答 |

**训虾派 = 一整张图**（唯一一个 Q1→Q4 都有完整工程方案的体系）

## 业界数据

- **Anthropic**：无显式人格锚定的 Agent，每 20 轮对话有约 15% 概率偏离预设人格
- **OpenAI**：半年后 system prompt 可能达 5000+ token，模型开始优先处理最新 token 而非基础指令
- **McKinsey**：先定义 AI Agent 架构再选工具的团队，比先选工具再定义架构的团队，ROI 高 3.2 倍
- **Anthropic 有效 Agent**：没有先建立单 Agent 稳定性就直接上多 Agent 架构的团队，失败率是 3 倍
