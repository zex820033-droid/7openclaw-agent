# 07｜卷七 协同军团：从单体 Agent 到多智能体作战

> 4 个模块 + 总复盘
> 精读 3 轮后浓缩

---

## 一句话

**卷七要解决的是"如何让一群 Agent 真正能协同"，靠 4 套协议：编制 + 三省制 + 交接 + 让位。**

---

## 4 大基础协议

### 协议 1：军团编制（M1）

#### 解决什么
- 角色定义
- 权限边界
- 层级关系
- 协作接口

#### 编制不是"起名字"
编制必须包含：
- 角色的核心职责
- 角色的决策权限
- 角色的汇报对象
- 角色的协作对象
- 角色的晋升/降级规则

#### 核心 3 问
1. 这个 Agent 叫什么、做什么、不做什么
2. 这个 Agent 能拍板什么、不能拍板什么
3. 这个 Agent 汇报给谁、谁能调用它

#### 编制的 6 大原则
1. **一个 Agent 只承担一个角色**的核心任务
2. **角色清晰才能治理**
3. **复杂任务用多 Agent 协同**，不用单 Agent 全栈
4. **每个 Agent 独立容器**（独立 Runtime + Workspace）
5. **路由决定 Agent 知道消息是自己的**
6. **扩缩容有章可循**（角色复用、合并规则）

#### 5 大角色模板
- **执行层**：技术专家、业务专家、运营专家
- **协调层**：协调者、调度者
- **监督层**：质量审核、合规官
- **兜底层**：兜底 Agent、熔断 Agent
- **战略层**：首席战略师、统帅

### 协议 2：三省制 + 统帅机制（M2）

#### 解决什么
- 谁来提议、谁来复核、谁来终审
- 紧急情况谁拍板

#### 三省制 vs 统筹审批
| 机制 | 解决的问题 | 效率定位 |
|---|---|---|
| 三省制 | 日常决策质量 | 质量优先 |
| 统帅机制 | 紧急决策效率 | 效率优先 |

#### 三省制 3 大角色
- **提议角色（Raiser）**：识别问题、提出方案、建议行动
- **复核角色（Reviewer）**：审查提议、给出同意/修改/否决
- **终审角色（Decider）**：综合判断、拍板决策、承担最终责任

#### 统帅机制的 4 大原则
1. 紧急情况有拍板（不取代日常三省制）
2. 统帅的裁决事后可被三省复核
3. 不绕过三省制（紧急除外）
4. 紧急裁决必须事后三省复核

#### 决策时限建议
- 提议窗口：30 分钟
- 复核窗口：15 分钟
- 终审窗口：15 分钟
- 紧急压缩：统帅可在 5 分钟内裁决

#### 置信度机制（L4 进阶）
- 每个角色的历史决策质量计入置信度
- 置信度影响决策权重
- 决策正确且被执行：置信度+5%
- 决策正确但执行有偏差：+2%
- 决策被否决后证明错误：-10%
- 决策被否决后证明正确：+3%

#### 7 大原则
1. 提议权 ≠ 决策权
2. 复核权必须独立
3. 终审权必须集中
4. 统帅权必须有边界
5. 决策必须有记录
6. 置信度必须动态
7. 效率和质量必须平衡

### 协议 3：Agent-to-Agent 交接协议（M3）

#### 解决什么
- 任务从 A 交到 B 时上下文怎么传
- 责任怎么转
- 闭环怎么做

#### 交接 5 要素
1. **触发条件**（什么时候交）
2. **上下文传递**（交什么）
3. **接收确认**（接了没有）
4. **执行反馈**（做怎么样了）
5. **证据存档**（有没有记录）

#### 交接数据包结构

```yaml
交接ID:
任务卡ID:
交出方:
接收方:
交接时间:

上下文:
  任务背景:
  已完成部分:
  未完成部分:
  关键约束:
  截止时间:

状态:
  当前状态:
  阻塞/风险:
  依赖项:

交付物:
  - 交付物1: 路径/位置
  - 交付物2: 路径/位置

预期动作:
  下一步行动:
  预期完成时间:
  验收标准:
```

#### 交接确认 5 步
1. 交出方发送交接数据包
2. 接收方回告"已接收"
3. 接收方复述理解（"我理解的任务是：..."）
4. 交出方确认"正确"或"补充说明"
5. 交接正式成立，责任转移

#### 交接置信度（L3 进阶）
| 维度 | 权重 | 评分 |
|---|---|---|
| 上下文完整度 | 30% | 1-5 |
| 交付物清晰度 | 30% | 1-5 |
| 接收方理解准确度 | 40% | 1-5 |
| **综合得分** | 100% | 加权平均 |

**阈值**：
- ≥4.0：正常交接
- 3.0-4.0：需补充说明后执行
- <3.0：重新交接 + 升级协调者

#### 4 类异常处理
| 异常 | 处理 |
|---|---|
| 接收方超时未确认 | 交出方追问 → 升级协调者 |
| 接收方理解有偏差 | 交出方补充说明 → 重新确认 |
| 交出方失联 | 接收方升级至协调者 |
| 接收方拒绝接收 | 接收方说明原因 → 协调者裁定 |

### 协议 4：群聊/私聊/让位规则（M4）

#### 解决什么
- 群聊里谁该答、谁该让
- 什么信息走群聊、什么信息走私聊

#### 3 类信息
- **群聊**：用户公开问题、决策公示、任务分配公示、进度同步、重要风险预警
- **私聊**：用户隐私信息、内部讨论/争议、敏感决策前的探索、负面反馈
- **静默**：与任务无关、纯闲聊、用户已明确不需要

#### 3 种让位类型
| 类型 | 触发 | 动作 | 边界 |
|---|---|---|---|
| **完整让位** | 专业边界明确属于他人 | 声明让位 + 停止 + 等待 | 超时则补位 |
| **补充式让位** | 主答案已给出 | 声明补充 + 不覆盖主答 | 补充不等于开新话题 |
| **纠正式让位** | 发现主答案明显错误 | 声明纠正 + 指出 + 提供正确 | 对事不对人 |

#### 让位 5 大原则
1. 专业优先原则（谁专业谁答）
2. 让位光荣原则（让位是专业素养）
3. 抢答可耻原则（非专业抢答是协同破坏）
4. 私聊例外原则（私聊用于隐私，不用于密谋）
5. 纠错公开原则（纠错在群里，不私聊，不人身攻击）

#### 让位信誉分（L3 进阶）
| 行为 | 信誉分变化 |
|---|---|
| 正确让位 | +5 |
| 正确补充 | +3 |
| 正确纠错 | +4 |
| 乱抢答 | -5 |
| 让位后干扰主答 | -3 |
| 错误纠错 | -5 |

#### 私聊规范
- 私聊开始：在群里声明
- 私聊结束：结论发回群里公示
- 涉及决策：结果必须进群公示

## 完整协同闭环公式

> **编制（职责 × 权限）× 决策（三省 × 统帅）× 交接（上下文 × 确认 × 闭环）× 让位（专业优先 × 信誉联动）= 真正能作战的军团**

---

## MVP（最小可用协同）

1. **最小编制**：3 个角色以内（执行 + 协调 + 兜底）
2. **明确路由规则**：什么消息路由到什么 Agent
3. **强制交接确认**：接收方必须复述
4. **最简让位规则**：谁专业谁答
5. **决策必须记录**：每个决策存档可查

> 做到这 5 点，协同就从混沌走向有序。

---

## Pro（高阶协同治理版）

在 MVP 上加 4 个治理组件：
- **交接置信度体系**
- **让位信誉分**
- **决策置信度机制**
- **动态编制调整**

---

## 4 大基础协议之间的因果链

```
(1) 军团编制
   ↓ 决定职责边界和权限范围
(2) 三省制 + 统帅机制
   ↓ 决定谁来决策
(3) Agent-to-Agent 交接协议
   ↓ 决定任务怎么传
(4) 群聊/私聊/让位规则
   ↓ 决定群聊怎么跑
真正的多智能体协同
```

**四个协议缺任何一个，军团就会以不同形式回到"伪军团"状态。**

## 验收标准

### 编制验收
- [ ] 每个 Agent 知道自己负责什么、不负责什么
- [ ] 出了新问题，能判断该路由到哪个 Agent
- [ ] 职责重叠时，有协调者裁定机制

### 决策验收
- [ ] 日常决策走三省制，有提议/复核/终审记录
- [ ] 紧急决策走统帅机制，事后有三省复核
- [ ] 没有"议而不决、决而不行"

### 交接验收
- [ ] 每次交接有完整上下文传递
- [ ] 接收方有明确复述确认
- [ ] 任务完成有闭环验收
- [ ] 交接记录可追溯

### 让位验收
- [ ] 专业问题由专业 Agent 回答
- [ ] 非专业 Agent 不抢答，主动让位
- [ ] 让位行为计入信誉分
- [ ] 纠错在群里公开，不人身攻击
