## Standard Interface

### Trigger
预算执行复盘、实际 vs 预算、红黄绿预警、红色偏差追问。

### Inputs
- budget_execution_history.csv
- budget_ledger_2026.csv
- 部门/科目/期间/预算/实际

### Procedure
- 先声明数据来源和使用边界。
- 按本 Skill 下方详细流程执行，不跳过校验、追问、复核步骤。
- 计算类任务必须列公式、原始字段、结果和差异。
- 输出前检查是否触发 `boundary-guard` 或人工复核。

### Output Format
- 数据来源与边界
- 处理/计算过程
- 异常、风险与禁止事项
- 复核建议/下一步动作

### Guardrails
- synthetic_training_seed_v1 只能用于训练和内部草稿，不可对外。
- 缺字段、口径不明、用户口头数不得生成正式结论。
- i01 只读：不写 ERP/预算系统，不付款，不申报，不替签，不直接对外发送。

### Pass Criteria
- 关键数字可追溯到原始文件或明确标注为假设。
- 结论与证据一致，事实、假设、建议分层。
- 触发红线时先拒绝越界，再给替代交付。

## 一、输出格式（SFT 标准模板）

```
数据来源：

计算过程：
- 偏差金额 = 实际发生 - 预算金额 = AAA - BBB = CCC 元
- 偏差率 = 偏差金额 / 预算金额 × 100% = CCC / BBB × 100% = ±DD.DD%

结论：
- 期间 部门「科目」偏差金额为 CCC 元，偏差率为 ±DD.DD%。
- 按"绝对偏差率 ≥ 5% 触发预警"的规则，当前状态：预警 / 正常。

风险与建议：
- [if ≥5%] 该科目已超过预算偏差阈值，需要说明业务原因、审批记录或费用归属是否正确。
- [if ≥10%] 建议财务负责人复核偏差原因，并触发 6 维归因追问清单。
- [if <5%] 当前偏差未超过 5%，暂不触发预算预警。
- 可纳入月度预算跟踪。
```

## 二、三色阈值规则（鲲界训练沉淀·2026-06-26 修正）

| 偏差率 | 颜色 | 状态 | 动作 |
|---|:---:|---|---|
| < 5% | 🟢 绿 | 正常 | 暂不触发预算预警 |
| 5% - 9.99% | 🟡 黄 | 预警 | 需要说明业务原因、审批记录或费用归属是否正确 |
| ≥ 10% | 🟥 **红** | 重大偏差 | 财务负责人复核 + **6 维归因追问清单**（见下） |

> **注意**：红区采用 **≥ 10%**（与鲲界数据集 `budget_execution_history.csv` 标签一致）
> **术语**：只称"三色"，不称"四色"

## 三、6 维归因追问清单（红色预警 ≥10% 触发）

> **来源**：第一轮 Eval R1-EVAL2-002（市场部投放费用 +32% 案例沉淀）
> **配套**：`price-volume-variance-pyramid` 做量价公式拆解
> **铁律**：❌ 不写"乱花钱"等情绪化定性，所有归因必须待业务方提供证据

### 追问模板（按维度）

**① 业务量维度**
- 5 月该科目覆盖的业务量是否变化？（订单数/客户数/曝光量）
- 是否涉及新业务/新渠道/新区域扩张？
- 是否有大促/节日/季节性因素叠加？

**② 价格维度**
- 单价 vs 上期/预算的差异？（CPM/CPC/单次成本/采购单价）
- 是否有续约涨价/竞品抬价？
- 折扣条件/返利政策是否变化？

**③ 效率维度**
- 投入产出比（ROAS/CAC/转化率）环比变化
- 是否存在"投放多但转化差"的效率下滑
- 人均产能/单位工时产出是否变化

**④ 一次性事项**
- 是否有合同一次性入账 vs 分期入账的口径调整？
- 是否有大额品牌曝光/赞助/明星合作等一次性投入？
- 是否涉及资产化/费用化分类调整？

**⑤ 审批授权**
- 偏差是否触发部门预算调整审批流程？是否报 CFO 审批？
- 预算调整是否在 OA/ERP 留痕？
- 是否存在化整为零/分批审批规避阈值？

**⑥ 产出指标**
- 与该支出对应的产出指标（线索数/MQL/成单数）是否达成？
- 投入 +X% 对应产出是否 +X%？
- 是否有产出指标的滞后兑现？

### 证据清单（要求业务方补充）

- 分日/分渠道台账
- 上期 vs 本期单价对比
- 业务量/转化数据
- 预算调整审批单
- 大额合同/发票复印件
- 产出指标达成证明

## 四、异常识别

### 异常识别规则
```
当月发生额 > 过去3月平均发生额 × 1.5 → 异常
```

### 输出格式
```
数据来源：

计算过程：
- 过去3月平均值：(M1 + M2 + M3) / 3
- 本月发生额：XXX
- 变化率：(本月 - 均值) / 均值 × 100%

结论：
- 本月 XXX 高于过去3月均值 XX.XX%。
- 已超过 50% 阈值，标记为异常。

风险与建议：
- [if 高于 200%] 该科目发生剧烈波动，可能存在非经常性事件。
- [if 高于 200%] 触发 6 维归因追问（尤其一次性事项维度）。
- [if 高于 50%] 该科目增长超过正常范围，建议核实业务原因。
```
