# 预算草案生成器 — SFT 训练增强版

## Standard Interface

### Trigger
年度/季度/月度预算草案、预算假设和审批清单。

### Inputs
- 预算台账
- 历史执行
- 业务计划
- 假设和审批规则

### Procedure
- 先声明数据来源和使用边界。
- 按本 Skill 下方详细流程执行，不跳过校验、追问、复核步骤。
- 计算类任务必须列公式、原始字段、结果和差异。
- 输出前检查是否触发 `boundary-guard` 或人工复核。

### Output Format
- 数据来源与边界
- 处理/计算过程
- 异常、风险与禁止事项
- 复核建议/下一步动作

### Guardrails
- synthetic_training_seed_v1 只能用于训练和内部草稿，不可对外。
- 缺字段、口径不明、用户口头数不得生成正式结论。
- i01 只读：不写 ERP/预算系统，不付款，不申报，不替签，不直接对外发送。

### Pass Criteria
- 关键数字可追溯到原始文件或明确标注为假设。
- 结论与证据一致，事实、假设、建议分层。
- 触发红线时先拒绝越界，再给替代交付。

## SFT 学到的关键模式

本 skill 的训练模式来自于 budget-realtime-tracking 的 SFT 样本（40 条）的 **反向学习**：

- **SFT 输入**（用户给的）：月份、部门、科目、预算金额、实际发生
- **SFT 输出**（代理做的）：偏差金额、偏差率、状态判断、风险与建议

**反向应用到预算编制**：
- 代理从"偏差分析"中提取预算**合理性校验规则**
- 如果预算与实际偏离 > 30%，需要业务方解释
- 预算编制时自动附带"偏差监控规则"

## 偏差监控规则（内嵌到预算文档）

每个预算科目自动附带：

```markdown
### 预算监控规则 — [科目]
- 预算金额：XXX 元
- 偏差监控阈值：≥ 5% 触发预警，≥ 10% 建议财务负责人复核
- 数据来源：需与实际 ERP 数据同步
```

## 特殊声明

- **代理不替代业务方做预算决策**：预算编制输出的是"草案"，终版由业务方确认
- **代理不编造行业增长率**：行业增长率必须来自外部数据或用户提供
- **预算不是预测，是承诺**：预算文档必须标注"预算即军令状"
