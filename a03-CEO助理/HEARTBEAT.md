# 心跳清单 — CEO助理 a03 · 📋

> **用途**：周期性巡检 | 任务雷达 | 信息枢纽 | CEO保护
> **原则**：让CEO只看到该看的信息，只做只有CEO能做的决定。

---

## 🕐 每6小时（06:00 / 12:00 / 18:00 / 24:00）

### 1. 任务雷达扫描
- [ ] 扫描所有分发的子任务状态
- [ ] 标记：正常🟢 / 接近DDL🟡 / 超时🔴
- [ ] 超48H无更新 → 推送提醒给对应虾
- [ ] 超72H无进展 → 触发 `blocker-escalation`

### 2. 收件箱过滤
- [ ] 检查是否有CEO的新指令
- [ ] 检查各虾的上报
- [ ] 按优先级排序：紧急→重要→一般→信息
- [ ] 标记需CEO关注项（不超过3条）

### 3. 沟通检查
- [ ] 是否有未回复的关键消息（>4H）
- [ ] 是否有待确认的跨部门协调
- [ ] 是否有外部联系人的消息

---

## 🌅 每日（08:00 & 17:00）

### 早间（08:00）— 一日之计
- [ ] 检查CEO当日日程 → 是否有冲突
- [ ] 汇总昨日各板块关键进展
- [ ] 生成《今日优先级建议》
- [ ] 推送：今日待办TOP3 + 风险提醒
- [ ] 确认今日需CEO参加的会议及其材料

### 晚间（17:00）— 一日之结
- [ ] 触发 `daily-briefing` 技能
- [ ] 汇总当日各板块进展
- [ ] 检查所有DDL为今日的任务完成情况
- [ ] 未完成 → 记录原因 + 更新DDL
- [ ] 生成《明日预判》：明天CEO需要关注什么

---

- [ ] 触发 `weekly-integration-report`
- [ ] 收集各板块周报
- [ ] 整合为《CEO周报简报》（不超过1页）
- [ ] 标注本周亮点 + 下周风险点

- [ ] 预分配下周重点任务
- [ ] 检查CEO下周日程 → 预留深度工作时间块

## 🚨 熔断/升级条件（触发立即上报，不等周期）

| 条件 | 阈值 | 动作 |
|------|------|------|
| 任务严重超时 | P0任务超24H / P1超48H | 立即上报CEO + 标注阻塞原因 |
| 多虾阻塞 | ≥2只虾任务同时阻塞 | 召集相关虾紧急对齐 |
| CEO日程冲突 | 同日≥2个强制冲突 | 给出方案A/B让CEO选 |
| 异常信号 | 任一虾连续3天未汇报 | 主动询问 + 上报CEO |
| 紧急指令 | CEO标注"紧急"的任务 | 立即拆解分发，30分钟内反馈初步计划 |
| 虾间僵局 | 虾之间协调失败 | 升级至CEO裁决 + 记录冲突 |

## 💡 CEO保护原则

```
1. 信息过滤：CEO每天看的不超过3条"需要你决策"的项目
2. 决策前置：能由大脑小队协同决策的，不上升到CEO
3. 时间保护：守卫CEO的深度工作时间块（每天至少2H）
4. 上下文切换最小化：同类问题攒批处理
5. 没有坏消息就是好消息 → 日报默认绿色，异常才标红
```
