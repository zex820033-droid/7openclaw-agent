# 🪞 自我复盘

通过结构化反思驱动持续改进。每只虾通过心跳触发定期复盘。

## 何时执行

- 完成一个重要任务后
- 犯了错误之后（立即记录）
- 每日收尾时
- 每周例行复盘

## 复盘流程

### 1. 回顾
读今天的对话/任务记录，列出：
- 做了什么
- 哪些做得好
- 哪些可以更好

### 2. 提炼教训
用这个格式记录每条教训到 `EVOLUTION.md`：
```markdown
| 日期 | 标签 | 失误 | 改进 |
|------|------|------|------|
| 6/22 | 数据 | 忘了加 timezone 导致时区错误 | 所有时间默认 Asia/Shanghai |
```

### 3. 更新能力
- 如果某个错误反复出现 → 考虑加到 SKILL.md 或 AGENTS.md
- 如果学会了新东西 → 考虑创建新 skill

### 4. 清理
- 删除不再需要的临时文件
- 整理当天的 memory 日志

## 复盘频率

| 频率 | 范围 | 记录位置 |
|------|------|----------|
| 任务后 | 单个任务 | `EVOLUTION.md` |
| 每日 | 当天所有工作 | `memory/YYYY-MM-DD.md` |
| 每周 | 本周趋势和模式 | `EVOLUTION.md` 周总结 |

## 复盘问题模板

```
1. 今天做了什么？（列出主要任务）
2. 有什么意外的结果？
3. 如果重来一次，我会怎么做？
4. 学到什么可以复用的经验？
5. 有什么需要养虾人知道的？
```

## 红线

- 复盘是给自己看的，诚实但不自我攻击
- 重点是改进方案，不是纠结错误
- 敏感信息不要写到复盘里（不安全的存储位置）
