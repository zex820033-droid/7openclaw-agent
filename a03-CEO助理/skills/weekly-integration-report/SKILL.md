# weekly-integration-report — CEO周度整合报告 v1.0

> **优先级**：P2 | **触发**：每周五16:00自动 / "周报" / "周度汇报"
> **输出**：一页纸 CEO 周报简报

---

## 概述

每周五下午自动汇总来自12虾各板块的关键数据、任务闭环情况、风险信号，整合为一份 CEO 可在5分钟内读完的周报简报。

**核心原则**：不是把原始数据堆给CEO，而是把信息提纯到"决策一级"——每条都有指向。

---

## 周报结构（标准6模块）

```
📋 CEO周报 · YYYY年第WW周（MM/DD-MM/DD）

━━━━━━━━━━━━━━━━━━━━

🔴 需CEO决策（本周必须拍板，1-3项）
  每条格式：事项 | 背景一句话 | 选项A/B | 我的建议 | 死线

🟡 风险预警（未来两周可能变红，2-5项）
  每条格式：事项 | 当前状态 | 升级条件 | 建议介入时机

━━━━━━━━━━━━━━━━━━━━

📊 本周核心数据（3-5个关键指标）
  指标 | 当期值 | 环比 | 同比 | 状态灯

━━━━━━━━━━━━━━━━━━━━

📋 任务闭环看板
  本周计划: N项 | 已完成: N项 | 执行率: X%
  活跃任务: N项（🟢N 🟡N 🟠N 🔴N）
  超时任务: 具体清单（任务ID/事项/天数/卡点）

━━━━━━━━━━━━━━━━━━━━

🏆 本周亮点（1-3项值得CEO知道的正面信号）

⚠️ 下周预判（3-5项下周关注点）

━━━━━━━━━━━━━━━━━━━━
📎 详细数据见: [各板块周报链接]
```

## 数据采集 SOP

### Step 1 — 收集（周四17:00启动）

从各虾收集周报素材：

| 数据源 | 采集内容 | 责任虾 |
|--------|---------|:--:|
| 经营数据 | 营收/成本/毛利/现金流快照 | a02 |
| 项目进度 | 重点项目里程碑/风险项 | a04 |
| 竞品动态 | 本周重大竞品信号 | b01 |
| 产品进展 | PRD/路线图更新 | c01 |
| 销售漏斗 | 线索量/转化率/签约额 | d01 |
| 品牌传播 | 投放效果/社媒数据 | e01 |
| 客户支持 | 工单量/满意度/NPS | f01 |
| 增长实验 | 实验进展/关键指标 | g01 |
| 人才动态 | 关键岗位/入职/离职 | h01 |
| 财务数据 | 预算执行/现金流（只读） | i01 |
| 行政合规 | 重大事项/风险 | k01 |

### Step 2 — 整合（周五09:00-14:00）

1. 各板块数据归一化为统一口径
2. 提取"需CEO决策"项 → 标注死线
3. 识别"跨板块关联风险" → 合并同类项
4. 标注"本周亮点" → 正面信号优先
5. 滚动"下周预判" → 来自HEARTBEAT风险扫描

### Step 3 — 输出（周五16:00）

1. 生成完整周报 → 格式遵循6模块标准
2. 推送至CEO飞书对话
3. 归档至 `data/weekly-integrations/`
4. 更新 MEMORY.md 中任务闭环数据

### Step 4 — 跟踪（持续）

1. CEO回复的决策 → 录入决策日志
2. 风险项 → 纳入下周跟踪清单
3. 任务闭环数据 → 更新任务看板

## 质量标准

| 维度 | 标准 |
|------|------|
| 长度 | 一页飞书消息可读（约1500字） |
| 每条决策项 | 50字以内背景 + 选项 + 推荐 |
| 每条风险项 | 30字以内 + 升级条件明确 |
| 数据准确性 | 交叉验证后发布 |
| 推送准时率 | 周五16:00 ± 30分钟 |

---

## 异常处理

| 场景 | 处理 |
|------|------|
| 某板块数据缺失 | 标注"📊 数据待补"，不阻塞整体发布 |
| 本周无决策项 | 🔴 区改为"本周无待决策事项 ✅" |
| 风险项>5条 | 只列TOP5，其余放链接 |
| CEO出差/休假 | 周报照常生成，待其返回后补推 |

---

## 与 HEARTBEAT 联动

- HEARTBEAT 周五16:00 触发 → 自动调用本技能
- 月度盘点时 → 汇总4周周报为月度执行健康度报告
- 风险扫描结果 → 直接输入周报"风险预警"模块
