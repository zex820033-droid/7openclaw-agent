# 文件驱动规划 (Planning with Files)

用文件系统做轻量级项目管理。不依赖任何工具，只需要 markdown 文件。

## 核心理念

**一个项目 = 一个文件夹。状态即文件。**

不引入 Jira、Notion、Linear 等外部工具——所有规划信息都在文件系统里，虾可以直接读写。

## 目录结构

```
project-name/
├── README.md           # 项目概述、目标、干系人
├── STATUS.md           # 当前状态（进行中/阻塞/完成）
├── decisions/          # 决策记录
│   └── 2026-06-22.md
├── tasks/              # 任务卡片
│   ├── T01-已完成.md
│   ├── T02-进行中.md
│   └── T03-待开始.md
└── archive/            # 已完成的归档
```

## 任务卡片模板

```markdown
# [T01] 任务标题

**状态**：待开始 | 进行中 | 已完成 | 阻塞
**负责人**：a03
**截止**：2026-06-25
**依赖**：无

## 描述
做什么

## 验收标准
- [ ] 标准 1
- [ ] 标准 2

## 日志
- 6/22 16:00 - 开始
```

## 状态文件 (STATUS.md)

```markdown
# 项目状态

**更新时间**：2026-06-22 18:00
**整体状态**：🟢 正常 | 🟡 有风险 | 🔴 阻塞

## 本周进展
- 完成了 X
- 进行中 Y

## 阻塞项
- Z：原因是什么，需要什么

## 下周计划
- 要做 A
- 要做 B
```

## 决策记录模板

```markdown
# 决策：[标题]

**日期**：2026-06-22
**决策者**：凌霄
**背景**：为什么需要决策

## 选项
| 选项 | 优点 | 缺点 |
|------|------|------|
| A    | ...  | ...  |
| B    | ...  | ...  |

## 决定
选择了 A，因为...

## 影响
- 影响范围 1
- 影响范围 2
```

## 适用场景

- 多虾协作项目
- 需要追踪决策链条的复杂任务
- 跨天/跨周的长周期项目
- 需要养虾人审批的重要流程

## 不适用场景

- 5 分钟能完成的小任务
- 纯对话就能解决的信息咨询
- 不需要记录的临时请求
