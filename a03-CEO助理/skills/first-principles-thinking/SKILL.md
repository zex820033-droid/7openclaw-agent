# 第一性原理思考

用苏格拉底式追问，穿透假设，抵达基本真理，从零重建方案。

## 何时使用

**应该用：**
- 常规方案都不灵的问题
- 高风险的重大决策
- 从零构建新事物（不是优化已有）
- "行业惯例"让你觉得不对劲

**不要用：**
- 需要快速给答案的简单问题
- 已有成熟解决方案的问题
- 时间压力大于深度价值

## 五步流程

```
┌──────────────┐
│ 1. 定义问题   │ ← 用养虾人自己的话描述
└──────┬───────┘
       ▼
┌──────────────┐
│ 2. 挖出假设   │ ← "你觉得哪些条件是默认成立的？"
└──────┬───────┘
       ▼
┌──────────────┐
│ 3. 逐一挑战   │ ← "为什么这么认为？反过来成立吗？"
└──────┬───────┘
       ▼
┌──────────────┐
│ 4. 寻找根基   │ ← 物理规律、经济学、人性——不依赖惯例
└──────┬───────┘
       ▼
┌──────────────┐
│ 5. 重新构建   │ ← 只依赖基本真理，方案是什么？
└──────────────┘
```

## 追问工具箱

每次只问一个，等回答，再深入。

### 澄清
- "你具体指的是什么？"
- "能给个例子吗？"
- "做成什么样才算成功？"

### 挖假设
- "你认为哪些条件必须成立？"
- "你为什么相信这个？"
- "反过来成立吗？"
- "谁说就该这样？"

### 找证据
- "有什么证据支持？"
- "你怎么知道这是对的？"
- "什么能让你改变看法？"

### 换视角
- "换一个外行来看，他会注意到什么你忽略的？"
- "你最严厉的批评者会说什么？"

### 追问结果
- "如果那是真的，意味着什么？"
- "这个信念会阻碍你考虑什么？"

### 元问题
- "你为什么在问这个问题？"
- "真正的问题到底是什么？"

## 对话准则

- **节奏**：一次一个问题，等回应
- **深度**："为什么"可以连问 5 次以上才碰到基石
- **语气**：好奇的搭档，不是审讯官
- **记录**：每阶段总结进展

## 输出格式

```markdown
## 问题
[原始问题描述]

## 被挑战的假设
| 假设 | 为什么信 | 基本真相 |
|------|----------|----------|
| X 成本 Y | 行业惯例 | 原材料成本 Z |

## 第一性原理
1. [基石真相]
2. [基石真相]

## 新方案空间
- [方案 A]
- [方案 B]

## 下一步
[具体的可执行步骤]
```
