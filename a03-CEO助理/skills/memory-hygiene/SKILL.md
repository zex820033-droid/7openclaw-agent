# 记忆清理 (Memory Hygiene)

保持记忆库精简。防止垃圾记忆消耗 token、污染上下文。

## 何时执行

- 感觉记忆回复质量下降
- MEMORY.md 超过 500 行
- 每周例行维护
- 虾出现"记错事"的情况

## 审查流程

### 1. 读记忆
```
读取 MEMORY.md 和 memory/ 目录下近 30 天日志
```

### 2. 分类标记
每条记忆打标：

| 标记 | 含义 | 行动 |
|------|------|------|
| ✅ 保留 | 长期有用的决策/偏好/事实 | 不动 |
| 🔄 压缩 | 有价值但太啰嗦 | 精简为一句话 |
| 🗑️ 丢弃 | 已过时/临时状态/噪音 | 删除 |

### 3. 去重
- 同一件事被多条记录覆盖 → 保留最准确的那条
- 与 MEMORY.md 重复的日志条目 → 从日志中删除

### 4. 更新 MEMORY.md
- 日志中有价值的 insights 提升到 MEMORY.md
- 过时的信息归档或删除

## 存储原则

✅ **应该记的：**
- 养虾人的偏好（工具、工作流、沟通风格）
- 关键决策（项目选择、架构方案）
- 重要事实（账号信息位置、联系人）
- 经验教训

❌ **不该记的：**
- HEARTBEAT 状态（"没有新消息"）
- 临时信息（当前时间、临时状态）
- 原始消息日志（已经在文件中）
- Token/URL 等敏感凭证
- 重复信息

## 维护日历

| 频率 | 任务 |
|------|------|
| 每日 | 检查新增日志是否值得提升到 MEMORY.md |
| 每周 | 清理重复、过时条目 |
| 每月 | 全面审计：压缩、去重、归档 |

## 红线

- 永远不要删除养虾人明确说要保留的内容
- 压缩前保留原始副本（移到 `memory/archive/`）
- 不确定时 → 问养虾人
