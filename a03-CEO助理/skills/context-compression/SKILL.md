# 上下文压缩 (Context Compression)

当会话上下文接近上限时，用 AI 生成智能摘要，重置会话，注入压缩后的上下文。

## 何时使用

- 上下文使用率接近 70-80%
- 长会话导致回复变慢或失去连贯性
- 主动维护，保持会话快速精准

## 压缩流程

```
检测到高上下文使用
        │
        ▼
┌──────────────────┐
│ 1. 生成摘要       │ ← AI 自己总结当前会话
│    - 完成了什么    │
│    - 关键决策      │
│    - 当前状态      │
│    - 待办事项      │
│    - 重要上下文    │
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 2. 备份原会话     │ ← 保存到 memory/compressed/
└──────┬───────────┘
       ▼
┌──────────────────┐
│ 3. 注入摘要       │ ← 新会话从摘要开始
└──────┬───────────┘
       ▼
    继续工作 ✅
```

## 摘要模板

```markdown
## 会话压缩摘要

### 完成事项
- [已完成的关键任务]

### 关键决策
- [决策及其理由]

### 当前状态
- [进行到哪一步了]

### 待办
- [还没做的事]

### 重要上下文
- [不能丢的信息]
```

## 使用方式

在 HEARTBEAT.md 中配置定期检查：
```markdown
## 上下文健康检查
如果上下文使用率 > 70%，执行压缩流程。
```

## 注意事项

- 压缩前必须备份（放到 `memory/compressed/`）
- 先检查摘要质量，确认无误再重置
- 不要在养虾人正在对话时压缩
- 压缩后第一时间告知养虾人当前状态
