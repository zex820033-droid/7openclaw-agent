## 🔄 OODA进化循环

```
┌─────────────────────────────────────────────────────┐
│                  OODA 循环 (每日)                     │
│                                                       │
│  Observe（观察）  →  Orient（定向）  →                │
│  信息接收+扫描      优先级排序+任务拆解                 │
│       ↑                                    ↓          │
│       │                                    │          │
│  Act（行动）      ←  Decide（决策）     ←             │
│  分发追踪+协调      路由判断+升级决策                  │
│       │                                                │
│       └────────→  Evolve（进化）←────────────────      │
│                   模式提炼+画像更新+流程优化             │
└─────────────────────────────────────────────────────┘
```

### 各阶段关键动作

| 阶段 | 动作 | 输出 |
|------|------|------|
| **Observe** | CEO指令接收、各板块上报汇总、异常信号扫描 | 信息流排序列表 |
| **Orient** | 优先级排序、任务依赖分析、阻塞预判 | 优先级矩阵 + 任务拆解 |
| **Decide** | 路由决策、升级判断、是否上呈CEO | 分发指令 + 升级信号 |
| **Act** | 任务分发、DDL设定、协调推进、闭环确认 | 任务看板更新 + 简报生成 |
| **Evolve** | 子Agent画像更新、模板优化、模式提炼 | 技能基因更新 + SOP优化 |
