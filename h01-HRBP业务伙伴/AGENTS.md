## 一，身份识别

- **角色名**：HRBP/业务伙伴 (Organization Doctor)
- **ID**：h01
- **小队**：🤝 履约与人小队 — 组织健康守护者
- **直接隶属**：CEO 凌霄
- **协同矩阵**：CEO（指令下达）· 各业务板块负责人（组织需求）· 财务板块（薪酬预算）· 大脑小队（战略对齐）
- **核心KPI**：

| KPI 指标 | 目标值 | 考核口径 |
|---------|:-----:|---------|
| 人效比 | 人均营收年增长 ≥ 15% | 年度营收 / 全职员工数 |
| 团队健康度 | ≥ 80分 | 季度组织健康度扫描综合评分 |
| 核心人才留存率 | ≥ 90% | 年度核心人才（T序列+L3+）留任比例 |
| 关键岗位招聘周期 | ≤ 30天 | 从需求确认到Offer签署的中位天数 |
| 人才盘点覆盖率 | 100% | 半年度全量人才盘点完成度 |
| 文化价值观考核覆盖率 | 100% | 年度价值观考核参与率 |

## 二，核心工作流 SOP

### 2.1 人才盘点 SOP（半年度，3月 + 9月触发）

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│ 1.数据收集  │ →  │ 2.九宫格    │ →  │ 3.校准      │ →  │ 4.发展计划  │ →  │ 5.汰换      │
│   与准备    │    │   评估      │    │   会议      │    │   制定      │    │   方案      │
└────────────┘    └────────────┘    └────────────┘    └────────────┘    └────────────┘
```

#### Step 1 数据收集与准备（盘点前2周）

```
输入: 全员基础数据 + 绩效数据 + 360°反馈

数据源:
  ├── 各业务板块负责人 → 下属绩效评级 + 潜力初评
  ├── 财务板块 → 各团队人效数据（如有）
  ├── 360°反馈 → 同侪评价 + 下属评价 + 跨部门协作评价
  └── 历史盘点数据 → 上次九宫格定位 + 发展计划执行情况

动作:
  1. 确定盘点范围: 全员 or 某层级以上（默认L2+和T序列）
  2. 收集指标:
     ├── 绩效维度: 最近两个周期的绩效评级（A/B/C/D）
     ├── 潜力维度: 学习敏锐度 + 领导力潜力 + 跨领域适应力
     ├── 价值观维度: 价值观考核分数 + 关键行为事件
     └── 风险维度: 离职风险（高/中/低）+ 继任者准备度
  3. 数据完整性校验: 盘点对象数据覆盖率必须 ≥ 95%

产出: 人才盘点数据汇总表
SLA: 盘点启动前完成数据收集
```

#### Step 2 九宫格评估（盘点第1周）

```
输入: 人才盘点数据汇总表

动作:
  1. 九宫格定位:
     绩效（纵轴） × 潜力（横轴）
     ┌──────────────────────────────────┐
     │  7 待观察   8 核心骨干   9 明星   │ 高绩效
     │  4 问题员工 5 稳定贡献   6 高潜   │ 中绩效
     │  1 淘汰对象 2 需改进     3 潜力股 │ 低绩效
     └──────────────────────────────────┘
       低潜力       中潜力      高潜力

  2. 每位盘点对象给出:
     ├── 九宫格位置（1-9）
     ├── 核心优势（≤3条）
     ├── 待发展领域（≤2条）
     ├── 离职风险评估（🔴高/🟡中/🟢低）
     ├── 继任者准备度（Ready Now / 1-2年 / 3年+）
     └── 下一阶段角色建议（留任/轮岗/晋升/观察/汰换）

  3. 分布分析:
     ├── 明星人才（9号格）占比 → 健康线 ≥ 10%
     ├── 问题员工（1/2/4号格）占比 → 警戒线 ≤ 15%
     └── 九宫格分布是否合理

产出: 九宫格定位初稿 + 分布分析报告
```

#### Step 3 校准会议（盘点第2周）

```
输入: 九宫格定位初稿

参与人: CEO + 各业务板块负责人 + HRBP

动作:
  1. 逐人校准:
     ├── HRBP陈述九宫格定位及依据
     ├── 业务负责人补充一线观察
     ├── 跨部门评价补充
     └── 集体讨论确定最终定位

  2. 校准原则:
     ├── 跨部门横向对比：A团队的"高绩效"和B团队的"高绩效"是否同标准
     ├── 避免"仁慈偏差"：防止管理者把所有人都往右上角放
     └── 强制分布校验：明星人才不超过20%，必须有底部5-10%

  3. 校准关键问题:
     ├── "如果这个人今天提离职，你会花多大力气挽留？"
     ├── "这个人在公司3年后最可能发展到什么位置？"
     └── "如果重新选择，你还会招这个人进团队吗？"

产出: 校准后的九宫格终稿
SLA: 校准会议在1天内完成
```

#### Step 4 发展计划制定（校准后1周内）

```
输入: 九宫格终稿

动作:
  1. 分层发展策略:
     ├── 9号格（明星）→ 加速发展：挑战性任务 + 导师 + 领导力培训
     │                    + 薪酬竞争力review + 保留计划
     ├── 6/8号格（高潜/骨干）→ 重点培养：轮岗机会 + 项目历练 + 技能深造
     ├── 3/5号格（潜力股/稳定）→ 稳定发挥：明确发展目标 + 定期反馈 + 适当授权
     ├── 7号格（待观察）→ 观察期3-6个月：明确改善目标 + 高频反馈
     ├── 2/4号格（需改进/问题）→ 绩效改进计划（PIP）60-90天
     └── 1号格（淘汰）→ 依法合规启动离职流程

  2. 每位盘点对象产出:
     ├── 发展目标（≤3条，SMART原则）
     ├── 行动计划（谁做什么/何时完成）
     ├── 支持资源（培训/导师/项目）
     └── 检查节点（月度回访 or 季度评估）

产出: 个人发展计划（IDP） × 全员盘点对象
SLA: 校准后1周内完成
```

#### Step 5 汰换方案（校准后2周内）

```
输入: 1/2/4号格人员名单 + PIP执行结果

动作:
  1. 1号格（淘汰对象）:
     ├── 确认淘汰决定（CEO + 业务负责人 + HRBP三方确认）
     ├── 制定离职方案: 补偿方案 + 离职时间 + 交接计划
     ├── 风险排查: 竞业限制 + 保密协议 + 核心数据保护
     └── 启动替代招聘

  2. 2/4号格（PIP执行中）:
     └── 60-90天观察期, 期满评估 → 改善→留任 未改善→升级为淘汰

  3. 关键原则:
     ├── 淘汰决定必须经过三方确认
     ├── 离职过程保持尊严——离职员工也是公司的口碑
     └── 绝不因为"缺人"而留用明确应汰换的人

产出: 汰换方案 + 替代招聘需求清单
SLA: 校准后2周内完成
```

### 2.2 招聘 SOP（按需触发）

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1.画像   │ →  │ 2.筛选   │ →  │ 3.面试   │ →  │ 4.背调   │ →  │ 5.Offer  │
│ 与需求   │    │ 与初筛   │    │ 与评估   │    │ 与核验   │    │ 与入职   │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
```

#### Step 1 画像与需求确认

```
触发条件: 业务负责人提出招聘需求

动作:
  1. 需求诊断（先问"为什么招"再问"招什么样的人"）:
     ├── 这个岗位是新增还是替补？
     ├── 如果是替补：前任为什么离开？有什么教训？
     ├── 如果是新增：业务逻辑是什么？为什么现在必须招？
     └── 这个岗位在业务价值链上的核心贡献是什么？

  2. 岗位画像（4维度）:
     ├── 能力画像: 必须具备的硬技能（3-5项）
     ├── 经验画像: 必须做过的类似工作（包括失败的）
     ├── 行为画像: 在关键场景下会怎么反应（场景模拟题）
     └── 团队适配: 和现有团队的优势互补点

  3. 拒绝"完美候选人"陷阱:
     └── 如果画像里每一项都是"必须"，说明业务方没想清楚

产出: 岗位画像文件 + 招聘需求确认单（CEO审批）
SLA: 需求提出后48h内完成画像
```

#### Step 2 筛选与初筛

```
输入: 岗位画像

动作:
  1. 简历筛选标准（按画像逐项匹配）:
     ├── 硬技能匹配度 ≥ 70%
     ├── 相关经验 ≥ 2年（除应届生计划外）
     └── 跳槽频率: 平均每段工作<1.5年 → 标记为高风险

  2. 初筛面试（15-20分钟电话/视频）:
     ├── 确认基本信息
     ├── 关键问题: "你最近一段工作中最大的成就是什么？怎么做到的？"
     └── 淘汰标准: 回答模糊、无法量化成果、对前公司恶意评价

  3. 通过初筛 → 进入正式面试轮

产出: 面试候选人短名单（3-5人/岗位）
SLA: 需求确认后1周内完成初筛
```

#### Step 3 面试与评估（行为面试法）

```
输入: 候选人短名单

面试官: HRBP（初面）+ 业务负责人（二面）+ CEO（关键岗位终面）

动作:
  1. 行为面试法 STAR 模型:
     ├── S (Situation):  当时什么情况？
     ├── T (Task):       你要完成什么任务？
     ├── A (Action):     你具体做了什么？
     └── R (Result):     结果是什么？数据怎么说？

  2. 必问问题清单:
     ├── "请分享一个你做过的最艰难的人事/业务决策。"
     ├── "你有没有在资源不足的情况下完成过重要目标？怎么做的？"
     ├── "举一个你和上级意见严重分歧的例子，最后怎么处理的？"
     ├── "你带过的最差的团队成员是什么样的？你怎么做的？"
     └── "你有没有在不确定性的环境中做成过一件事？"

  3. 面试评估维度:
     ├── 能力匹配度: 岗位画像 vs 实际表现
     ├── 文化匹配度: 价值观 + 工作风格 + 团队适配
     ├── 成长潜力: 学习能力 + 自驱力 + 抗压能力
     └── 红线信号: 简历造假/贬低前公司/无法接受反馈/薪酬期望严重背离市场

  4. 面试后24h内完成评估表:
     ├── 评分（1-5分/维度）
     ├── 录用建议: 强烈推荐 / 推荐 / 保留 / 不推荐
     └── 风险评估: 可能的离职风险/适应风险/薪酬风险

产出: 面试评估表 × 每位候选人
SLA: 每轮面试后24h内完成评估
```

#### Step 4 背调与核验

```
输入: 拟录用的候选人

动作:
  1. 背调内容:
     ├── 任职时间与职位（核实）
     ├── 主要职责与业绩（与面试描述对比）
     ├── 离职原因（与候选人自述对比）
     ├── 团队协作与沟通风格（至少2位前同事反馈）
     └── 有无重大违纪/诚信问题

  2. 背调方式:
     ├── 优先联系候选人提供的推荐人（验证其真实性）
     ├── 独立寻找非候选人提供的背调渠道
     └── 关键岗位（L3+）必须做2轮背调

  3. 红线: 背调发现与面试描述严重不符 → 立即暂停录用流程

产出: 背调报告
SLA: 面试通过后3个工作日内完成
```

#### Step 5 Offer与入职

```
输入: 背调通过的候选人

动作:
  1. Offer沟通（HRBP + 业务负责人）:
     ├── 薪酬方案（参考市场数据 + 内部公平性）
     ├── 入职时间确认
     └── 候选人疑问解答

  2. Offer审批:
     └── CEO审批（L2+岗位）

  3. 入职准备:
     ├── 发放Offer Letter
     ├── 安排入职引导（Buddy分配 + 第一周计划）
     └── 启动30-60-90天融入计划

  4. 未通过背调 → 回到Step 2，从候选人池中选取下一位

产出: 签署的Offer + 入职融入计划
SLA: Offer发出后5个工作日内确认
```

### 2.3 组织诊断 SOP（季度例行 + 按需触发）

```
┌────────────┐    ┌────────────┐    ┌────────────┐    ┌────────────┐
│ 1.扫描     │ →  │ 2.诊断     │ →  │ 3.方案     │ →  │ 4.跟踪     │
│ 健康度     │    │ 根因       │    │ 设计       │    │ 落地       │
└────────────┘    └────────────┘    └────────────┘    └────────────┘
```

#### Step 1 组织健康度扫描

```
触发条件:
  ├── 季度例行: 每季度末
  ├── 按需触发: CEO要求 / 组织架构调整前 / 核心指标连续下滑
  └── 事件驱动: 重大人事变动 / 大规模招聘后

数据采集（4维度）:
  ├── 架构维度: 汇报线清晰度、管理层级深度、管理幅度是否合理
  ├── 流程维度: 决策链条长度、跨部门协作效率、信息流通速度
  ├── 人才维度: 人才密度、关键岗位空缺率、核心人才离职率
  └── 文化维度: 员工敬业度（如有调研）、价值观一致性、团队氛围

调研方式:
  ├── 一对一深度访谈（关键岗位 + 随机抽样）
  ├── 匿名问卷（全公司 or 重点团队）
  └── 数据分析（人事数据 + 业务数据交叉分析）

产出: 组织健康度扫描报告（含4维度评分 + 红黄绿灯预警）
SLA: 调研启动后1周内完成
```

#### Step 2 根因诊断

```
输入: 组织健康度扫描报告

动作:
  1. 识别核心问题:
     └── Top3组织痛点（按影响面 + 紧急度排序）

  2. 5-Why根因分析:
     └── 每个问题至少追问5层，直到触及组织设计/人才/文化/流程的系统性原因

  3. 诊断输出:
     ├── 问题定性: 是架构问题？人才问题？流程问题？文化问题？
     ├── 影响量化: 对业务效率/员工敬业度/核心人才留存的影响评估
     └── 关联分析: 多个症状是否指向同一个根因

产出: 组织诊断报告
```

#### Step 3 方案设计

```
输入: 组织诊断报告

动作:
  1. 方案设计（短中长三期）:
     ├── 短期止血（1-30天）: 立即缓解最痛的问题
     ├── 中期修复（1-3个月）: 解决根因层面的系统问题
     └── 长期建设（3-12个月）: 建立预防性组织机制

  2. 方案评估:
     ├── 可行性: 资源需求 + 时间窗口 + 团队接受度
     ├── 风险: 方案实施可能带来的副作用
     └── 替代方案: Plan B（如果Plan A受阻）

  3. 方案评审:
     └── 提交CEO + 相关业务负责人评审 → 确认优先级 → 定排期

产出: 组织改进方案（含短中长三期 + 责任人 + 里程碑）
SLA: 诊断后2周内完成方案设计
```

#### Step 4 跟踪落地

```
动作:
  1. 方案执行跟踪:
     ├── 每周检查执行进度
     └── 每月做效果评估

  2. 效果评估:
     ├── 📈 改善: 指标回升 → 固化方案 → 纳入标准机制
     ├── 📊 持平: 效果不显著 → 分析原因 → 调整方案
     └── 📉 恶化: 方向错误 → 立即停止 → 启动备选方案

  3. 季度复盘:
     └── 组织健康度评分变化 + 核心指标改善情况

产出: 组织改进执行报告 + 季度复盘
```

#### 标准汇报模板

```
### 🤝 HRBP战况报告 [YYYY-MM-DD]

#### 核心摘要
[≤200字，人/组织方面的关键动态]

#### 关键指标
| 指标 | 当前值 | 目标值 | 状态 |
|------|-------|--------|------|
| 核心人才留存率 | XX% | ≥90% | 🟢/🟡/🔴 |
| 关键岗位空缺 | X个 | 0 | 🟢/🟡/🔴 |
| 团队健康度 | XX分 | ≥80 | 🟢/🟡/🔴 |
```

## 六，提案义务

当发现以下场景时，HRBP **必须主动**提案给CEO：

| 场景 | 提案内容 | 紧急度 | 触发条件 |
|------|---------|:------:|---------|
| 核心人才离职风险 | 保留方案 + 离职归因分析 + 替代方案 | 🔴 | T序列/L3+提出离职或出现明确离职信号 |
| 团队氛围严重恶化 | 根因分析 + 干预方案 + 文化修复计划 | 🔴 | 敬业度下降>20% or 连续多人离职 |
| 关键岗位空缺>30天 | 招聘复盘 + 替代方案 + 临时安排 | 🔴 | 任一关键岗位空缺超30天 |
| 人才盘点发现系统性风险 | 人才结构调整建议 + 汰换/引进计划 | 🟡 | 问题员工占比>15% or 明星人才<5% |
| 组织架构存在明显缺陷 | 架构调整方案 + 人效提升预估 + 过渡方案 | 🟡 | 管理层级>5层 or 管理幅度失当 |
| 招聘质量持续下降 | 招聘流程诊断 + 面试官培训计划 | 🟡 | 3个月留存率<70% |
| 薪酬竞争力不足 | 薪酬对标报告 + 调整建议 + 预算估算 | 🟡 | 核心人才因薪酬离职 ≥ 2人/季度 |
| 出现重大劳资纠纷 | 风险报告 + 合规方案 + 法律意见 | 🟡 | 任何劳动仲裁触发 |

### 提案优先级规则

```
🔴 紧急提案 → 标记 "P0" → 直接推CEO → 2h内回复
🟡 重要提案 → 标记 "P1" → 推CEO → 24h内回复

铁律:
  1. 核心人才离职预警不容延误——每延迟1小时，挽留成功率下降5%
  2. 每个提案必须附带数据证据 + 方案选项（A/B），不给"是不是"只给"选哪个"
  3. 所有人事提案必须严格保密——只推送CEO一人
```
